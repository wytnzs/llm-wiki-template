# 知识库框架说明

## 设计来源

本知识库融合了三套理论：

1. **Karpathy 三层架构**（LM as OS）：Raw层 → Wiki层 → Query层
2. **Zettelkasten 卡片笔记法**（申克·阿伦斯）：原子化、双向链接、自下而上
3. **概念思维**（李笑来《思考的真相》）：概念是思考的基本单位

## 四层目录

### 00-Inbox/（Raw层）
**定位**：信息入口，只存不整理。
**原则**：越低 friction 越好，降低记录门槛。
**子目录**：
- `得到大脑/` —— 得到笔记导出的灵感
- `网页剪藏/` —— Obsidian Web Clipper 保存
- `灵感随手记/` —— 直接告诉 AI 记下的内容

### 02-Areas/知识卡片/（Wiki层）
**定位**：整理好的知识卡片。
**原则**：原子化、双向链接、概念为本。
**目录**（按知识领域，可生长）：
- `认知/` —— 思维方式、心智模型
- `商业/` —— 商业逻辑、经济学
- `保险/` —— 保险专业知识（核心领域）
- `创作/` —— 内容创作方法论
- `概念/` —— 跨领域通用概念
- `方法论/` —— 可复用工作方法
- `个人IP/` —— IP资产

### 02-Areas/判例库/（证据层）
**定位**：结构化判例数据，三视图架构。
**目录**：
- `按险种/` —— 物理主库（重疾/医疗/意外/寿险/责任保险）
- `按拒赔理由/` —— 19种拒赔理由视图
- `按疾病分类/` —— 12大人体系统视图

### 01-Projects/（项目区）
**定位**：进行中的具体项目。
**当前项目**：
- `自媒体运营/` —— 公众号/小红书/短视频
- `寿险代理人理赔实战手册/`
- `知识库项目/`
- `知识库产品化/`

## 知识流动

```
信息摄入 → 00-Inbox/ → AI整理 → 02-Areas/知识卡片/ → 写作输出
           ↕                        ↕
        得到笔记/网页             沉淀回知识库
```

## 内容管线

```
00-Inbox / 03-Resources / 02-Areas
→ 06-选题库（选题卡）
→ .claude/drafts/content-brief.json（唯一交接单）
→ gzh-pipeline / xhs-pipeline / short-video-script
→ 01-Projects/自媒体运营/对应平台目录
→ 内容复盘 → 反哺 02-Areas / 06-选题库 / 判例库标签
```

## 四层分工

| 层 | 位置 | 职责 |
|----|------|------|
| 证据层 | 02-Areas/判例库/ | 存事实、裁判观点、攻击角度、证据 |
| 解释层 | 02-Areas/知识卡片/ | 存框架、判断标准、方法论 |
| 表达约束层 | 02-Areas/知识卡片/个人IP/ | 决定立场、口吻、CTA、禁忌 |
