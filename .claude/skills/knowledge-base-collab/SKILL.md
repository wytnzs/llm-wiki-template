# 知识库协作 Skill

## 入口

先阅读根目录 `README.md`，获取知识库目的、实际目录映射和已启用模块。README 是唯一必需入口。

不要假设用户一定使用某套目录；推荐路径只在 README 没有说明时作为默认值。

## 基本流程

```text
输入 -> 整理 -> 连接 -> 使用 -> 复盘
```

## 五种内容角色

| 角色 | 判断标准 | 默认位置 |
|---|---|---|
| 输入 | 尚未处理、需要保留原貌 | `00-Inbox/` |
| 项目 | 只服务当前目标 | `01-Projects/` |
| 资产 | 能跨项目复用 | `02-Areas/` |
| 资源 | 外部材料，不代表用户判断 | `03-Resources/` |
| 归档 | 已完成或暂时不用 | `04-Archive/` |

目录可以改名或合并，以 README 的映射为准。

目录只表达内容角色，不表达全部来源、主题和项目关系。不要因为一份知识涉及多个主题就复制文件或新增一级目录。

## 统一术语

- 主题：需要持续回答的问题；
- 聚合：围绕问题筛选资料的动作；
- 主题地图（主题页）：保存当前综合判断、证据、反例和边界；
- 关系导航：进入不同主题页的入口；
- 知识库地图：启用后的动态状态报告。

## 触发流程

### 记一下

保存用户原始表达，不擅自扩写，不强制立即制成知识卡片。

### 整理收件箱

逐项判断：保留原始输入、进入项目、提炼资产、转为外部资源或归档。没有明确价值时先报告，不静默删除。

### 提炼知识卡片

只有内容能够跨项目复用并帮助判断、决策或行动时才建卡。新卡至少包含 `title` 和 `type`，优先补充有意义的 `description`、可核验来源和适用边界。来源不确定时进入待确认队列，不得编造。

### 查一下

先读 README 确定资产位置，再按“主题页 -> 知识卡片/案例 -> 原始材料”下钻。回答时区分用户判断、外部资料和案例事实，并说明适用边界。

### 复盘

追溯问题来自输入、知识、项目、模板还是 Skill；把修正写回源头。

## 最小关系与 AI 权限

知识资产日常只优先维护：`source`、`related_theme_maps`、`related_projects`。支持、反例和例子按需使用；已有旧关系字段继续兼容，不批量机械迁移。

- 原始材料只读，AI 可以分类和建立来源链接，不改写正文；
- 案例事实、合同条款、病历以及法律、医疗等关键结论只能提议修改；
- 知识卡片和主题页可以由 AI 起草，关键判断先进入 `00-Inbox/AI待确认.md`；
- 模板、索引、状态统计、断链检查和 `知识库变更日志.md` 可以按规则自动维护。

## 可选模块

- README 未声明知识库地图时，不调用地图 Skill；
- README 未声明案例库时，不创建案例目录；
- README 未声明选题库时，不创建选题卡或 `06-选题库/`；
- 选题库已启用时，也必须由用户确认候选选题后再入库。

## 原则

1. 标准定义角色，不强制复杂目录；
2. 原始材料、知识资产、项目成品和操作流程分开；
3. Skill 只保存方法，不保存业务知识；
4. 不把个案结论写成普遍承诺；
5. 不上传私人资料，不静默删除或批量迁移。
