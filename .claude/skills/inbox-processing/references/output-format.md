# Inbox 加工输出格式

```markdown
# Inbox 加工建议

## 原始输入
- 文件：
- 摘要：
- 来源状态：已确认 / 待核验 / 无来源 / 本人观察
- 敏感性：普通 / 需脱敏 / 必须暂停

## 路由决定
- 主要去向：继续保留 / 进入项目 / 知识卡片 / 外部资料来源 / 个人输入来源 / 系统历史来源 / 更新主题 / 待清理隔离 / 暂停
- 目标位置：
- 判断依据：
- 置信度：高 / 中 / 低

## 资产草稿
- 类型：
- 标题：
- 核心内容：
- 来源：
- 适用场景：
- 边界与反例：
- 状态：draft / growing / stable

## 人工确认
- [ ] 是否采用该去向？
- [ ] 是否允许创建目标文件？
- [ ] 是否允许移动原文件？移动到哪里？
- [ ] 哪些事实或敏感信息需要先核验、脱敏？

## 执行后报告
- 新建或更新：
- 原文件状态：
- 尚未处理：
```

没有必要生成资产时，删除“资产草稿”部分。遇到安全闸门时，只输出风险、暂停原因和所需确认。
