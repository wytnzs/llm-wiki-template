# Inbox 路由判断表

| 主要去向 | 判断条件 | 默认动作 | 不应发生 |
|---|---|---|---|
| 继续保留输入 | 信息不完整，但近期可能补充 | 保留原貌，列出缺口 | 为清空 Inbox 强行加工 |
| 进入项目 | 只服务一个当前目标或交付 | 关联项目并注明用途 | 写成跨项目通用结论 |
| 知识卡片 | 能跨项目复用，能帮助判断或行动 | 生成卡片草稿 | 机械摘要、隐藏来源 |
| 外部资料来源 | 主要价值来自外部作者、数据或原文 | 进入 `03-Resources/` 对应集合，保存来源、摘要和使用说明 | 把外部观点写成本人判断 |
| 个人输入来源 | 本人原始观察、表达或素材仍需追溯 | 进入 `03-Resources/02-个人输入/`，保留原貌并连接已形成资产 | 把个人判断写成未经核验的外部事实 |
| 系统历史来源 | 旧方案、项目历史或迁移记录仍有复盘价值 | 进入 `03-Resources/03-系统资料/系统资料/`，注明当前规则入口 | 用历史记录替代当前规则 |
| 更新主题地图 | 会改变现有主题结论、证据或边界 | 提出更新片段和关联位置 | 只追加链接，不说明影响 |
| 待清理隔离 | 已确认有规范副本，或无长期保留价值 | 建议可恢复隔离位置并等待确认 | 把仍需追溯的唯一原件当重复副本处理 |
| 暂停 | 隐私、高风险、来源冲突或职责不明 | 报告风险和所需确认 | 继续加工或扩散原文 |

## 判断优先级

1. 先处理隐私和安全；
2. 再判断是否服务当前项目；
3. 再判断是否具备跨项目复用价值；
4. 再区分外部来源和本人判断；
5. 最后决定进入哪类来源集合、待清理隔离或暂停。

同一输入允许产生多个后续资产，但本次只指定一个主要去向，避免重复写入多个目录。`04-Archive/` 已退役，不再作为路由选项。
