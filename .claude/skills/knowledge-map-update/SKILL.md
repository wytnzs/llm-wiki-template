# 知识库地图更新技能

> 保持根目录 `知识库地图.md` 与全库实际状态同步。
> **不静默删除文件**——发现问题主动报告，列出问题清单和修复建议，由用户决定操作。

---

## 工作流

### 第一步：全库扫描与统计

按顺序扫描并统计：

| 目录 | 统计项 |
|------|-------|
| `00-Inbox/` | 待处理条目数 |
| `01-Projects/` | 活跃项目数、各项目文件数 |
| `02-Areas/知识卡片/` | 各领域卡片数（保险/认知/商业/个人IP/创作/概念/方法论）、总计 |
| `02-Areas/判例库/` | 判例总数（按险种、按拒赔理由、按疾病分类三视图） |
| `02-Areas/主题聚合/` | 主题地图数 |
| `03-Resources/` | 保险资料、读书笔记、自媒体资料、AI工具参考 各计数 |
| `04-Archive/` | 归档项目数 |
| `.claude/skills/` | skill 数量 |
| `06-选题库/` | 选题卡片数、选题树覆盖 |

### 第二步：健康检查清单

逐项检查以下问题，输出**问题清单**（含严重程度 + 建议修复方式）。

#### 1. 乱码检测
- **检查**：全库文本文件中是否有无法解析的中文字符、编码不一致文件
- **严重度**：高——乱码文件不可读
- **修复建议**：先备份或隔离，尝试恢复编码；删除必须经用户确认

#### 2. 旧路径残留
- **检查**：地图中引用的路径是否已被重构？跨技能引用的路径是否依然有效？
- **严重度**：中——链接失效影响访问
- **修复建议**：更新地图路径，删除已不存在的引用

#### 3. 断链检测
- **检查**：遍历 `[[链接]]` 语法，确认每个链接在知识库中能找到对应文件
- **严重度**：中——断链降低卡片网络价值
- **修复建议**：补充缺失链接目标，或更新引用

#### 4. Inbox 堆积检查
- **检查**：`00-Inbox/` 条目数是否超过 20 条
- **严重度**：低→中（>20条/ >50条分别不同等级）
- **修复建议**：运行「整理收件箱」流程

#### 5. 卡片 YAML 覆盖率
- **检查**：`02-Areas/知识卡片/` 下的卡片，有多少带 YAML 头部（title/tags/created/updated）
- **严重度**：低——影响自动索引精度，不影响阅读
- **修复建议**：逐步为无 YAML 卡片补充元数据

#### 6. 主题地图更新时效
- **检查**：`02-Areas/主题聚合/` 中每个主题地图，最近更新时间是否超过 30 天
- **严重度**：中——长期未更新的主题地图可能反映过时认知
- **修复建议**：阅读主题地图，根据最新判例和卡片更新核心判断

#### 7. 选题来源完整性
- **检查**：`06-选题库/cards/` 下正式选题卡的 YAML 中，是否标注了 source_cards / source_cases 等来源字段
- **严重度**：低→中——缺来源无法追溯资产支撑
- **修复建议**：补充选题卡的来源信息，无来源的考虑移至待挖掘方向

#### 8. content-brief 调用记录
- **检查**：已发布内容的 YAML 头部是否记录了调用的 content-brief 路径
- **严重度**：低——影响后续复盘追溯
- **修复建议**：补充 content-brief 引用

#### 9. 成品内容复盘追溯
- **检查**：已发布内容是否在发布后执行了复盘（复盘记录是否写入复盘库）
- **严重度**：中——不复盘则经验无法沉淀
- **修复建议**：对未复盘的内容执行复盘流程

#### 10. Skill 与目录结构一致性
- **检查**：`.claude/skills/` 引用的路径是否仍然存在？CLAUDE.md 中描述的目录与实际目录结构是否一致？
- **严重度**：高——引用了不存在的路径会导致操作失败
- **修复建议**：更新 skill 或 CLAUDE.md 中的路径引用

#### 11. 对比上次记录值
- **检查**：对比上次地图记录值与实际扫描值，标注差异
- **严重度**：信息性——反映知识库变化趋势
- **修复建议**：无——仅用于追踪

#### 12. 孤立卡片检查
- **检查**：`02-Areas/知识卡片/` 中无任何 `[[链接]]` 的卡片
- **严重度**：低——孤立卡片可能已过时或未融入网络
- **修复建议**：阅读孤立卡片，补充 [[链接]] 或归档

#### 13. skill 完整性检查
- **检查**：`.claude/skills/` 下每个 skill 是否有 SKILL.md
- **严重度**：高——缺 SKILL.md 的 skill 不可用
- **修复建议**：为缺失的目录补充 SKILL.md

#### 14. 判例库链接完整性
- **检查**：`02-Areas/判例库/` 的类型索引中引用的判例文件是否都存在
- **严重度**：中——索引中的幽灵引用
- **修复建议**：更新索引或补充缺失判例文件

### 第三步：写入地图

更新根目录 `知识库地图.md`：
1. **统计面板**：各目录文件/卡片计数
2. **健康状态面板**：问题清单摘要（严重问题数、中等问题数、低问题数）
3. **近期更新**：最近 10 条变更
4. **项目状态**：01-Projects 下各项目进度
5. **选题库概览**：选题总数、按树分布
6. **健康标记**：需关注的问题（带上严重级别和修复建议的概要版）

## 触发词

`更新知识地图` | `同步知识地图` | `刷新索引` | `知识库状态`

## 原则

1. **数字来自实际扫描**，不估算。
2. **发现问题主动报告**，不做静默修复。
3. **问题清单必须包含**：问题描述 / 严重程度 / 建议修复方式。
4. **用户决定如何处理**——只建议不强制，不擅自删除文件。
5. **健康状态面板帮助快速决策**：红色（高）/橙色（中）/黄色（低）三档区分优先级。
