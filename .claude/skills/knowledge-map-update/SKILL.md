# 知识库地图更新 Skill

本库已启用知识库地图。开始前读取根目录 `README.md` 获取实际区域和已启用模块。

## 统一审计入口

更新地图前先运行只读审计脚本，避免把模板占位、待补关系种子、历史归档和真实断链混为一类：

```text
python .claude/skills/knowledge-map-update/scripts/audit_knowledge_base.py --details
```

需要结构化结果时使用 `--json`。脚本只读取文件，不自动修复；所有统计仍需按本 Skill 的口径解释和复核。

## 更新内容

1. 当前统计：各主要区域的 Markdown 和文件数量；知识卡片、主题地图、主判例、判例视图、选题卡和 Skill 数量；
2. 项目状态：只列 `01-Projects/` 中实际存在的项目，并从项目文件读取状态；
3. 健康问题：乱码、缺少必要元数据、失效路径、断链、空 Skill、积压输入和过期主题地图；
4. 最近变化：最多保留 10 条真正影响结构或能力的变化；
5. 下一步：按优先级列出 3 至 5 项可执行工作。

知识卡片健康统计同时记录：可用 `source` 数量、空或缺失 `source` 数量、非空 `description` 数量。不得用自动生成的空值提高覆盖率。

## 扫描规则

- 数字必须来自实际扫描，不沿用旧地图估算；
- YAML 必要字段与关系字段结构覆盖正式运营层：`01-Projects/`、`02-Areas/`、`05-Skills/`、`06-选题库/`、根目录三个系统入口和 `00-Inbox/AI待确认.md`；原始 Inbox 与 `03-Resources/` 来源材料允许保留来源格式；`04-Archive/` 应只剩退役说明；
- 判例主库统计 `按险种/` 中 `type: case_law_entry` 的文件；两个视图单独统计，不能相加当作独立判例；
- 模板、README、概览、法条和维权路径不计入主判例数；
- 知识卡片、主题地图和选题卡按 `type` 统计；
- Skill 只统计实际存在 `SKILL.md` 的目录；
- 双链检查必须区分真实断链、同名歧义、模板占位、待补关系种子和历史旧路径；不能为了让数字归零而创建空卡片或虚假关系；
- 对 `.git/`、`.obsidian/`、缓存、草稿和 `03-Resources/03-系统资料/系统资料/` 中的历史记录按检查目的决定是否跳过，并在报告中说明口径。

## 健康检查

重点检查：

| 级别 | 问题 |
|---|---|
| 高 | 乱码、Skill 缺入口、运行路径不存在、真实数据越过公开边界 |
| 中 | 正式知识文件缺 `title` 或 `type`、关键断链、地图与实际严重不符 |
| 低 | Inbox 积压、主题地图过期、缺推荐字段、孤立卡片 |

`00-Inbox/AI待确认.md` 中未完成事项属于待审核积压，应单独统计；根目录 `知识库变更日志.md` 只检查格式和入口有效性，不把流水条数作为质量指标。

选题库已在本库启用，因此可以检查来源完整性；不得把“未自动生成选题”视为故障。

## 写入原则

- `README.md` 解释系统，`知识库地图.md` 展示当前状态；地图不重复系统说明；
- 不保留冗长历史流水账；
- 不静默删除或修复数据，发现异常先报告；
- 更新日期、统计口径和问题状态必须一致。
