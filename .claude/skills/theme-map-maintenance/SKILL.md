# 主题地图维护 Skill

> 主题地图不回答「这个主题有什么资料」——它回答「当前我对这个主题的认知是什么」。

---

## 核心定位

先读取根目录 README。主题地图默认位于 `02-Areas/主题聚合/`；如果 README 记录了其他位置，以实际映射为准。主题地图把分散的原子知识组织成对一个问题的完整认知。

| 维度 | 知识卡片 | 案例库 | 主题地图 |
|------|---------|-------|---------|
| 粒度 | 原子概念 | 单案事实 | 综合认知 |
| 回答 | 这是什么 | 这个案子怎么了 | 当前我相信什么 |
| 维护方式 | 炼新卡片 | 录入新案例 | 认知升级时同步更新 |

---

## 标准结构

```yaml
---
title: 主题名
type: theme-map
tags: [theme_map]
created: YYYY-MM-DD
updated: YYYY-MM-DD
related_cards: [[卡片1]], [[卡片2]]
related_cases: [案例ID1, 案例ID2]
---
```

正文模块：
1. 当前核心判断
2. 关键概念（关联卡片）
3. 证据来源
4. 已验证方法（适用条件/操作步骤/验证案例/成功率）
5. 适用边界
6. 反例与风险
7. 最近更新记录
8. 待补问题

---

## 创建/更新时机

创建：≥3张卡片围绕同一问题，或≥5个案例同属一主题
更新：新案例推翻/巩固判断，稳定打法形成，边界变化，超过30天未更新

---

## 工作流

### 创建
1. 收集卡片和案例 → 2. 提炼核心判断 → 3. 按标准结构写入 → 4. 建立双向链接

### 更新
1. 读取已有地图 → 2. 确定触发原因 → 3. 只更新变化模块 → 4. 更新记录

### 健康检查
遍历所有地图，输出绿/黄/红三档健康报告。

---

## 原则

1. 不是资料列表，是认知快照
2. 只更新变化的部分
3. 标注确定性程度
4. 保持双向链接有效
5. 定期养护，不频繁扰动
