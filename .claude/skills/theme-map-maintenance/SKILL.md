# 主题地图维护 Skill

> 主题地图不回答“有哪些资料”，而回答“当前对这个问题的综合判断是什么，为什么相信，什么情况下可能失效”。

---

## 核心定位

先读取根目录 README。主题地图默认位于 `02-Areas/主题聚合/`；如果 README 记录了其他位置，以实际映射为准。对用户可以简称“主题页”。

| 维度 | 知识卡片 | 案例库 | 主题地图 |
|------|---------|-------|---------|
| 粒度 | 原子概念 | 单案事实 | 综合认知 |
| 回答 | 这是什么 | 这个案子怎么了 | 当前我相信什么 |
| 维护方式 | 炼新卡片 | 录入新案例 | 认知升级时同步更新 |

---

## 标准结构

```yaml
---
title: 主题名
type: theme_map
description: 一句话说明主题页回答的问题
core_question: 围绕这个主题，需要持续回答的具体问题是什么？
tags: [theme_map]
created: YYYY-MM-DD
updated: YYYY-MM-DD
related_cards: []
related_cases: []
---
```

正文模块：
1. 当前核心判断
2. 关键概念（关联卡片）
3. 证据来源
4. 已验证方法（适用条件/操作步骤/验证案例/成功率）
5. 适用边界
6. 反例与风险
7. 最近更新记录
8. 待补问题

---

## 创建/更新时机

创建：至少 3 张卡片围绕同一问题、至少 5 个案例形成同类模式，或用户明确要求综合当前认知。

更新：新证据推翻或巩固判断、稳定方法形成、边界变化、项目反馈暴露问题，或主题长期未审阅。

`core_question` 必须是具体问题，不能只写领域名称。关键判断应能沿卡片、案例或正文链接回到可靠原始依据；只有目录级概述时列入待补问题。

---

## 工作流

### 创建
1. 收集卡片和案例 → 2. 提炼核心判断 → 3. 按标准结构写入 → 4. 建立双向链接

### 更新
1. 读取已有地图 → 2. 确定触发原因 → 3. 只更新变化模块 → 4. 更新记录

### 健康检查
遍历所有地图，输出绿/黄/红三档健康报告。

## 关系链接准入

新增或保留主题关系前判断：

1. 能否一句话说清目标内容提供的证据、方法、例子、反例或边界；
2. 沿链接过去是否有助于回答问题、作出判断或行动；
3. 是否能够继续回到可靠依据。

三项都是“是”才建立正式关系。只是同领域、同标签、关键词相同或以后可能用到时，只保留标签。

批量关系变更必须先干跑，抽查高相关、边界相关和明显不相关三类样本，再检查 YAML、断链、同名歧义和双向一致性。案例、法律和医疗关键结论链逐条核对证据和适用边界。

当 AI 发现冲突、来源不足、可能误连或需要改变用户核心判断时，不直接改写主题页，先追加到 `00-Inbox/AI待确认.md`。用户确认后再更新主题页、双向关系、日期和重要变更日志。

---

## 原则

1. 不是资料列表，是认知快照
2. 只更新变化的部分
3. 标注确定性程度
4. 保持双向链接有效
5. 有证据或实际使用反馈时再更新，不为更新而更新
