# 主题地图维护 Skill

主题页不回答“有哪些资料”，而回答“当前对这个问题的综合判断是什么，为什么相信，什么情况下可能失效”。主题中心入口默认位于 `02-Areas/主题中心/README.md`，主题页默认位于 `02-Areas/主题中心/主题页/`；`type: theme_map` 保持不变，兼容已有字段和双链。

## 标准结构

```yaml
---
title: 主题名
type: theme_map
description: 一句话说明主题地图回答的问题
core_question: 围绕这个主题，我们需要持续回答什么问题？
status: active
tags: [主题地图]
created: YYYY-MM-DD
updated: YYYY-MM-DD
related_cards: []
related_cases: []
---
```

正文包含：

1. 当前核心判断；
2. 关键概念；
3. 证据来源；
4. 已验证方法；
5. 适用边界；
6. 反例与风险；
7. 最近更新记录；
8. 待补问题。

## 创建与更新

创建条件：至少 3 张卡片围绕同一问题，至少 5 个案例形成同类模式，或用户明确要求综合当前认知。

更新条件：新证据推翻或巩固判断、稳定方法形成、边界变化、项目反馈暴露问题，或主题长期未审阅。

只更新发生变化的部分，并同步 `updated` 和更新记录。重复已有模式的新材料只建立关联，不必改写核心判断。

`core_question` 必须是具体问题，不得只写领域名称。主题页中的关键判断应能沿知识卡片、判例或正文链接继续回到 `03-Resources/` 或其他可靠原始依据；只有目录级概述而没有具体证据时，列入待补问题。`04-Archive/` 已退役，不再作为证据入口。

## 关系链接准入

新增、保留、移除或批量补充主题关系前，完整读取 `../../../05-Skills/知识关系质量评估标准.md`；涉及标签、双链和主题地图分工时，同时读取 `../../../05-Skills/知识关系与标签规范.md`。

日常关系按三问法判断：关系是否能一句话说清、沿过去是否有助于判断或行动、能否回到可靠依据。三项都是“是”才建立正式关系；只是同领域、同标签、关键词相同或以后可能用到时，只保留标签。

主题地图的 `related_cards` 和 `related_cases` 只收录直接回答核心问题，或提供证据、方法、反例、边界和代表性验证的精选入口，不追求覆盖全部同领域资料。批量变更必须先干跑并抽查高相关、边界相关、明显不相关三类样本；保险、法律和医疗的关键结论链逐条核对证据与适用边界。

当 AI 发现冲突、来源不足、可能误连或需要改变用户核心判断时，不直接改写主题页，先追加到 `../../../00-Inbox/AI待确认.md`。确认后再更新主题页、双向关系、日期和更新记录，并在根目录 `知识库变更日志.md` 追加一条重要变化。

## 与判例库的关系

判例是证据；主题地图是对多个证据和知识卡片的综合。引用判例时说明样本范围、反例和确定性，不用单案证明普遍结论。

## 与选题库的关系

本库已启用选题库，但主题地图不自动生成正式选题。发现内容方向时只提出候选，由用户确认后再调用选题 Skill 入库。其他未启用选题库的知识库应完全跳过此步骤。

## 原则

1. 不是资料列表，是认知快照；
2. 只更新变化部分；
3. 区分事实、推断和待验证判断；
4. 保持来源和链接有效；
5. 不为更新而更新，不为内容生产强行制造选题。
