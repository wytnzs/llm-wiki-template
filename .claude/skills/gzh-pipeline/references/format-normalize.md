# 格式归一化 — 万物先转 Markdown，再进写作流程

写作管线只认 Markdown。任何非 Markdown 输入，先按本文件转成 Markdown 草稿，**经用户确认结构后**再进认知引擎。

转换只整理结构，**不得增删改写原文实质内容**。

---

## 输入类型判定

| 输入 | 判定依据 | 处理 |
|------|---------|------|
| **Markdown** | `.md` 后缀，或文本含 `#`/`##`/`![]()` 等标记 | 直接进流程 |
| **Word** | `.docx` 后缀 | 运行 `scripts/extract_docx.py` |
| **PDF** | `.pdf` 后缀 | 用 Read 工具分页读取 + 清噪规则 |
| **纯文本** | `.txt` 或粘贴的无标记大段文字 | 语义推断结构 |
| **网页/富文本** | HTML 标签或剪贴板样式残留 | 剥样式 → 按语义映射 Markdown |
| **老版 Word (.doc)** | `.doc` 后缀 | 无法直接解析，请用户在 Word/WPS 另存为 `.docx` |

---

## Word (.docx)

```bash
# 运行提取脚本
PYTHONIOENCODING=utf-8 python scripts/extract_docx.py 文件.docx -o 文件.md
```

脚本使用 Python 内置 zipfile + XML 解析，**零外部依赖**：

| 格式 | 映射方式 |
|------|---------|
| 标题样式（Heading 1-6 / 标题 1-6） | `#` ~ `######` |
| 粗体 | `**粗体**` |
| 下划线 | `<u>下划线</u>` |
| 编号/项目列表 | `- 列表项` |
| 图片 | 解包到 `images/` 目录，相对路径引用 |
| 表格 | Markdown 表格格式（`\| 列1 \| 列2 \|`） |

**结构确认**：读取生成的 `.md`，展示给用户确认章节结构是否正确，再进入认知引擎。

## PDF

用 Read 工具分页读取（原生支持 PDF），读完按规则清噪再组装 Markdown：

**① 删页眉页脚**

每页重复出现的同一行（书名/章节名/URL）、纯页码行（`12`、`- 12 -`、`Page 12`），在前 3 页和后 3 页识别出模式后，全篇删除。

**② 合并硬断行**

| 行尾特征 | 处理 |
|---------|------|
| 行尾无句末标点（。！？；：）且下一行以小写/中文续接 | 拼成一段 |
| 行尾是句末标点 | 保留分段 |
| 行尾是短横线连字符 `-` | 合并为完整单词（去掉连字符） |

**③ 识别标题**

PDF 转换后丢失字号信息，用纯文本启发式：

| 特征 | 判定 |
|------|------|
| 行长 ≤ 20 字，不以句末标点结尾 | 候选标题 |
| 前后都有空行 | 标题 |
| 带序号前缀（一/二/1./01/第X章） | 标题 → `##` |
| 多个短行连续出现 | 第一个可能是标题，其余是正文 |

**分级**：
- 全文最高频的标题模式 → `##`
- 其下一级子标题（若有）→ `###`
- 文首孤立短行 → `#` 文章标题

**④ 图表占位**

PDF 里的图无法提取，位置留 `【配图：原 PDF 第 N 页图】` 占位，配图阶段会渲染为居中待补板块。

**⑤ 双栏检测**

发现语句前后不接、ABAB 交错模式时，提醒用户该 PDF 可能需要人工整理（双栏 PDF 常见串行错乱）。

## 纯文本（.txt / 无标记粘贴）

### 标题识别启发式

满足越多越可信：

- 行长 ≤ 20 字，且不以句末标点（。！？）结尾
- 前后都是空行（或位于全文/段落群开头）
- 带序号前缀：`一、` `二、` `1.` `01` `第X章/节` `Part N`
- 与后文构成"短行 + 多个长段"的节奏

**分级**：全文最高频的标题模式定为 `##`；其下一级子标题定为 `###`；文首孤立短行定为 `#` 文章标题。

### 标题识别失败时

全文识别不出任何标题 → 按语义把内容切成 3-6 个主题块，为每块自拟一个 ≤12 字的章节标题（要说人话、贴内容，不要"第一部分"这种空壳），并在结构确认时明确告知用户"这些标题是自动起的"。

### 其余映射

| 原始格式 | Markdown |
|---------|---------|
| 空行分段 | 保持分段 |
| `"引号金句"` / 破折号署名 | 开头引言候选 |
| `1. 2. 3.` 或 `·`/`-` 起头的行 | 列表 `- ` |
| 表格状排列（空格/制表符对齐的多行） | 评估是否需转为 Markdown 表格 |

## 网页 / 富文本

| HTML 元素 | Markdown |
|-----------|---------|
| `<h1>`~`<h6>` | `#`~`######` |
| `<strong>`/`<b>` | `**粗体**` |
| `<img src="X">` | `![](X)`（外链 URL 原样保留） |
| `<li>` | `- ` |
| `<blockquote>` | `> ` |
| 样式属性、span 嵌套、`&nbsp;` | 全部剥掉 |

**剪存工具元信息**：原文链接、剪存时间、工具署名等不属于正文，剔除。

---

## 结构确认（必做，全自动模式除外）

归一化完成后、进认知引擎前，把推断结果给用户看一眼：

> 识别出 N 个章节：01 xxx / 02 xxx / …（其中第 X、Y 个标题是自动起的）。图片 M 张。结构对吗？

结构错了后面全错——这一步花 10 秒，拦住整篇返工。用户确认或修正后再进认知引擎。

### 全自动模式

用户说「直接排」「一键」「不用问」时：

1. **跳过结构确认和选主题**，全链自动：推断结构 → 自动匹配主题 → 写作 → 排版 → 校验
2. **交付时补决策说明**：识别出的章节结构、自拟了哪些标题、选了什么主题及理由
3. 用户对任何决策不满意，按点改，不用重来
