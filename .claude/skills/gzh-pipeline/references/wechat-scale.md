# WeChat Scale — 公众号长阅读尺度

公众号不是海报流，而是长阅读场。设计目标是让读者读 3-8 分钟不累，并在每一屏都能抓到结构锚点。

默认母风格：

**克制知识编辑风 + 必要时插入材质化解释图**

## 一、设计目标

公众号的高级感来自：

- 可信：像一份专业内刊，而不是营销单页
- 稳定：同类文章有同类结构
- 耐读：正文承担主要阅读，不用卡片压迫读者
- 可扫读：章节、下划线、提示块帮助读者定位
- 有节制：主色只做锚点，不铺满全文

## 二、两条默认路线

### 1. 深度编辑风

适用：

- 观点文
- 案例拆解
- 行业趋势
- 方法论
- 个人创业复盘

视觉语言：

- 白底 / 米白底 / 极浅灰底
- 深灰、石墨、深蓝或墨黑为主
- 少量橙色或红色做重点
- 编号章节
- 引言卡、目录、结论块
- 正文关键词下划线

结构：

```text
标题
开头引言 / 核心冲突
3 个看点
01 章节
02 章节
03 章节
结论 / END
作者签名 / CTA
```

### 2. 信息密度风

适用：

- 教程
- 工具盘点
- 操作指南
- AI 工作流
- 清单型文章

视觉语言：

- 标签、步骤、提示块更多
- 卡片可以出现，但不要把整篇文章变成卡片堆
- 表格优先于长段解释
- 代码、Prompt、命令必须使用专门代码块
- 每个小节后可配一张解释图或流程卡

结构：

```text
标题
为什么需要这个方法
准备条件 / 适用边界
步骤 01-03
常见坑
可复制清单
结尾行动建议
```

## 三、版式尺度

| 元素 | 建议 |
|---|---|
| 正文宽度 | 以公众号正文容器为准，最大约 677px |
| 正文字号 | 15-16px |
| 正文行高 | 1.75-1.9 |
| 段落长度 | 1-3 句为主，超过 5 句必须拆 |
| 章节间距 | 明显大于段落间距 |
| 强调频率 | 每段 1-3 个关键词，不整段高亮 |
| 最强锚点 | 全文不超过 5 处 |

## 四、颜色尺度

不要把公众号做成多彩海报。默认遵循：

- 80%-90%：黑白灰 / 深浅中性色
- 5%-10%：主题主色，用于章节、标签、下划线
- 0%-5%：警示色或点睛色，用于真正重要的风险/结论

默认推荐：

| 用途 | 色值 |
|---|---|
| 深色正文 | `#1A1A1A` |
| 标题深色 | `#0A1931` 或 `#102542` |
| 辅助文字 | `#5F6D78` / `#86909C` |
| 分割线 | `#E5EAF2` |
| 强调橙 | `#F77F00` |
| 浅背景 | `#F7F9FC` / `#FAFAF7` |

## 五、图片尺度

公众号图片只承担三类任务：

1. 解释一个抽象概念
2. 展示一个流程/结构/对比
3. 重画数据或图表

不要为了好看硬塞图。

配图规则：

- 短文：0-2 张
- 中长文：2-4 张
- 深度文：3-6 张
- 每张图都必须有独立信息
- 图内文字只放短标签，解释放正文里

## 六、封面尺度

公众号封面要同时考虑：

- 文章页头图
- 会话分享卡片
- 历史消息列表
- 可能被裁成方形的安全区

原则：

- 主标题放在中间安全区
- 标题不超过 15 字，长标题拆副标题
- 一张封面只表达一个判断
- 右侧方图不要塞复杂信息
- 不使用过密小字

## 七、质量检查

排版前检查：

- [ ] 本文选的是深度编辑风还是信息密度风？
- [ ] 是否只用了一个主题体系？
- [ ] 主色是否克制，没有满篇乱跳？
- [ ] 每段是否有扫读锚点，但不过度高亮？
- [ ] 图片是否承担解释任务？
- [ ] 法律/保险结论是否保留边界？
- [ ] 复制到公众号后是否仍能保持主要层级？
