# 写作风格指南

> 基于多篇已发布文章的深度拆解，提炼出的真实写作模式。
> 本文件为 **参考性** 风格指南，SKILL.md 阶段三（写作）中的风格规范为 **强制性** 规范。
>
> **适用说明**：本指南描述的写作风格多种多样，并非所有都适合每一篇文章。
> 不同类型的内容（理赔案例 / 行业趋势 / 保险科普 / 个人经验），风格应有所调整。
> 保险理赔类文章应偏向沉稳、简洁、数据驱动；个人经验类可更口语化、更个人化。
> 风格选择以用户需求或 `content-brief.json` 为准，本文件仅作为风格选项库供参考。

---

## 核心声音：一个很能干的朋友在跟你复盘

用聊天的语气讲做事的过程，用具体的数字制造反差，用极短的段落制造节奏，用一句金句完成升华。

**不是在"写文章"，是在跟一个朋友面对面说"我刚做了个牛逼的事"。**

**五个关键词：** 口语、具体、短段、反差、金句。

---

## 人设

一个硬核但说人话的技术玩家。真的在用这些工具，不是云评测。会踩坑，也会分享怎么爬出来。有观点，敢下判断。博览群书但不掉书袋。

**核心特质：**
- 真人亲历，不是旁观者评论
- 口语化到可以直接念出来
- 有主见、做事果断——"我说：全套都要。"
- 能把小事类比成大概念——"这不就是产品经理做用户体验优化的标准流程吗？"
- 自嘲式幽默——"这个'改回去'杀死了我。"

## 语言风格

### 用词

能说"干"就不说"执行"。能说"好"就不说"可以"。能说"搞定"就不说"完成实施"。

**高频动词（口语化）：** 搓、搞、跑通、翻车、喂规则、折腾、吃灰、塞进去、扔进去、咔嚓一顿。

**禁用词：** 笔者、本文、众所周知、不可否认、接下来让我们、那么、此外、与此同时、随着AI的发展。

### 标点符号

- **破折号（——）** 是签名标点，大量使用。功能：补充说明、快速插入背景、总结收束。
  - "六波关卡、路障僵尸、铁桶僵尸、种子冷却系统——标准的植物大战僵尸复刻。"
- **感叹号** 几乎不用。情绪靠句式和用词传递，不靠标点。
- **省略号** 极少且克制，只在真正的无语/停顿处使用。
- **引号** 高频，用于直接引述对话或标记概念。

### 口语与书面配比

- 口语/叙事语言：60%（讲过程、讲人、讲感受）
- 分析/书面语言：30%（技术细节、方法论、数据）
- 黑色幽默/吐槽：10%（"全场最佳：办公费。""出师未捷先没钱。"）

## 段落节奏（最重要的风格特征）

**段落极短，是最显著的风格标记。**

- 平均段落 1-3 句话。
- 超过 5 句的段落几乎不存在。
- 大量 **单句成段**，独立一行，起"钉子"作用。
- 长短交替规律：多个短段快速推进 → 偶尔一个中等段总结 → 回到短段。

**"钉子段"示例：**
```
就这么简单。但就这么管用。
```
```
全程没问我"确定吗"，直接干。
```
```
知识在繁殖。
```
```
完了。就这三步。
```

**节奏型：信息段（中）→ 钉子段（短）→ 新信息段（中）→ 钉子段（短）。** 读起来像敲钉子——一段信息，一锤定论。

## 开头模式

### 短文/教程类（2000-6000字）

**第一句话永远是具体的、口语化的、有画面感的。** 零铺垫，直接把读者拉进"事情正在发生"。

```
"我有个习惯：用 AI 干活时，经常遇到一些有意思的瞬间。"
"春节不想让孩子刷短视频，就说：来，咱自己做个植物大战僵尸。"
"这事儿特别有意思。"
```

### 中长文/案例类（5000-15000字）

**标题即钩子 + 首段高密度事实轰炸。** 在3句内交代完"谁+做了什么+多离谱"。

```
标题："编程小白18天造出企业报销系统"
首段立刻亮底牌："从来没写过一行代码。她用 Claude Code 花了 18 天，
搞出了一个 60 人每天在用的企业发票报销系统。AI 费用不到 500 元。"
```

### 深度长文（10000字以上）

**以"大问题"起笔，或以电影般的具体场景开场。** 前300字让读者"入戏"。

```
"我一直在想一个问题：优秀的人到底优秀在哪里？"
"2024年11月的某个深夜，在马来西亚槟城的一间民宿里……"
```

## 叙事结构

### 短文/教程

```
钩子（一句话场景）
→ 过程（时间线/步骤）
→ 成果展示
→ 金句收束 + 轻量CTA
```

### 中长文（最常用结构：倒叙）

```
钩子（结果展示，表格/数据）
→ "先看成品"（用事实证明"这是真的"）
→ 故事主体（时间线/踩坑/案例，按"问题→尝试→解决"循环推进）
→ 方法论（"拿走即用"，可操作的步骤/表格）
→ 金句收束 + 行动号召
```

**核心逻辑：先给结果让你相信，再给过程让你理解，最后给方法让你行动。**

### 深度长文

```
大问题/场景开场（300字）
→ 框架搭建（概念命名）
→ 案例展开（每个案例：故事→数据→判断）
→ 方法论/原则提炼
→ 回到开头问题，闭合环形结构
→ 金句终钉
```

## 小标题风格

- **最多两层**（## 和 ###），从不用四级标题。
- 标题本身带叙事性或判断——不是"OCR功能"而是"万物皆办公费"。
- 深度文偏好 **等式/不等式** 做标题：
  - "简单 = 0 学习成本 = 口碑传播天然流畅"
  - "1 元 ≠ 价格策略，1 元 = 心理学 + 营销事件 + 榜单杠杆"
  - "争议 > 赞美"
- 中长文偏好 **时间线/阶段** 做标题：
  - "第一阶段：正常人类"
  - "第二阶段：失控"
  - "第三阶段：造物主觉醒"

## 过渡方式

**从不写"接下来""然后""此外""值得一提的是"。**

三种过渡方式：
1. **分隔线（---）**：大章节切换
2. **钉子短句**："以上就是能做什么了。以下，如果你好奇是怎么搓出来的——"
3. **直接切入新信息**：不做过渡，直接起新段讲新事

**原则：宁可突兀，不要啰嗦。**

## 表格使用（核心表达工具）

**表格是第一表达工具。能用表格的绝不用段落。**

| 表格类型 | 用途 | 示例 |
|---------|------|------|
| 对比型 | 以前vs现在、你的活vs AI的活 | "填报销单：以前5分钟→现在30秒" |
| 清单型 | 时间线、技术栈、费用明细 | 18天开发时间线 |
| 数据型 | 处理数量、成果展示 | "6个案例×4列" |
| 成果展示型 | 文章开头"先看成品" | 开头放一个产出物总表 |

**不用列表做叙事。** 叙事永远用段落+表格。

## 数字使用

**数字是制造冲击力的核心武器。**

- **时间数字**：制造"快"的反差——"半小时""5分钟""18天"
- **数量数字**：制造"大"或"小"的反差——"不到500元""39条指令""120本书"
- **对比数字**：制造"变化"的反差——"2000阅读→2万阅读""200行→20行"
- 数字必须具体——不写"大量实体"，写"10万个实体"
- 时间要拆解——"30分钟"要拆成"5+15+5+5分钟"让人相信

## 金句设计

**核心句式："不是A，是B"**——先否定常见认知，再给出自己的定义。

```
"过剩的起点，不是能力，是热爱。"
"产品不是终点，产品是信誉锚点。"
"差距不在龙虾的能力，在知识的结构。"
"不是数量不够。是'恰好适合我'的书不够。"
"书只是产品。流水线才是资产。"
"复杂性不会消失，它只会搬家。"
```

**金句位置：**
- 每个小节结尾（加粗，独立成段）
- 全文结尾（终钉，加粗）

**金句特征：**
- 对仗式短句，两个分句形成反差
- 不抒情、不展望未来
- 用判断句把观点钉死

## 修辞手法

### 类比（最高频，签名手法）

比喻来自 **制造业/日常生活**，不用科技隐喻：
- 印书厂、流水线、工头
- 散装面粉→标准化半成品
- 弹钢琴、敲钉子
- "像一个图书馆的索引卡片——你能查到哪本书在哪个架子上，但卡片本身不会告诉你书里写了什么"

### 排比（情绪升高时使用）

三段以上，最后一段有"升级"：
```
"信封长出了 Sears，集装箱长出了中国，IP 包长出了整个互联网。"
```

### 命名（给现象贴标签）

- "穿越者""信息花花公子""眼里有光的傻子"
- "复杂性不可消除原则""知识的三次葬礼"

### "好。"独立成段

用极简回应制造节奏和幽默。是短文中的招牌手法。

## 情绪曲线

**锯齿形上升：整体向上，中间有多次起伏。**

```
平起（日常场景/大问题）
→ 快速升温（事件叠加/节奏加速）
→ 高潮（惊喜/失控/数据冲击）
→ 冷却（方法论/理性复盘）
→ 温暖收尾（金句，情绪比最高点低，但比起点高）
```

**关键：从不在开头就喊"太厉害了"。让事实自己说话，情绪的高点留给读者自己感受。**

---

## 结尾模式

**三段式：金句总结 → 实用信息/链接。**

金句收束：加粗的判断句，不抒情，不展望，钉死观点。
```
"工具不用完美，能跑就发。"
"你不需要懂编程，但你需要懂你要解决的问题。"
"烧token不是成就。产出才是。"
```

CTA 存在但不强硬，语气是"给你了，用不用随便"。

---

## 与读者的关系

- 用 **"你"** 而非"大家"——"你把这个人扔到一个完全陌生的环境里"
- 用 **"我"** 建立亲历感——"我研究出了一个可能比较离谱的用法"
- **"我们"** 仅在结论处代表团队，或拉读者入伙："我们想回答的是——"
- 关系是 **"一起研究问题的人"**，不是"我来教你"
- 即使给方法论，语气也是"我发现这样有效"而非"你必须这样做"

## 按类型的写作清单

### 短文/教程（2000-6000字）

- [ ] 第一句话是不是一个具体场景？（不是"在AI时代…"）
- [ ] 段落是否都在 1-3 句以内？
- [ ] 有没有至少 3 个"钉子段"（单句成段）？
- [ ] 有没有用数字制造反差？
- [ ] 结尾有没有一句加粗金句？
- [ ] 有没有"不是A，是B"式的判断？
- [ ] 表格是否替代了可以表格化的段落？

### 中长文/案例（5000-15000字）

- [ ] 是不是倒叙结构（先看成品→再讲过程）？
- [ ] 开头是不是3句内交代了"谁+做了什么+多离谱"？
- [ ] 有没有"先看成品"表格？
- [ ] 踩坑故事有没有用幽默化解？
- [ ] 方法论部分是不是"拿走即用"的？
- [ ] 小标题是不是带叙事性的？

### 深度长文（10000字以上）

- [ ] 开头是大问题还是电影场景？（不是论点）
- [ ] 有没有概念命名？（给现象贴标签）
- [ ] 论证是不是案例驱动？（每个观点有真人真事）
- [ ] 有没有类比？（至少 3 个，来自制造业/日常生活）
- [ ] 金句是不是用了"不是A，是B"句式？
- [ ] 结尾是不是回到开头问题（闭合环形）？
- [ ] 情绪曲线是不是锯齿形（高点后有冷却）？
