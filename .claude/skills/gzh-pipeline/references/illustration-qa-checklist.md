# 配图 QA 检查清单

> 配图生成完成后、进入朋友圈阶段前执行。三类检查：**内容侧 → 视觉侧 → 格式侧**。
> 任何一项不通过，对应卡片必须重新生成。


## 一、内容侧检查（每张卡片逐张过）

### 1.1 卡片类型匹配

| 内容特征 | 应该用 | 不应该用 |
|---------|-------|---------|
| 步骤/流程/时序 | 类型 1：流程图 | 网格卡片、柱状图 |
| 多个并列概念/功能点 | 类型 2：网格卡片 | 流程图、对比卡 |
| A vs B / 新旧 / 有无对比 | 类型 3：暗底对比卡 | 流程图、网格卡片 |
| 多维评分/权重/数据对比 | 类型 4：柱状图 | 流程图、分栏布局 |
| 两种视角/投入产出/方案对比 | 类型 5：分栏布局 | 流程图、柱状图 |
| 单句金句/核心判断 | 金句卡（独立行，不加卡片容器） | 任何卡片类型 |

- [ ] 每张卡片类型匹配其所在位置的内容特征
- [ ] 没有"为配图而配图"的装饰卡（该位置不需要配图时就不插）

### 1.2 信息准确性

- [ ] 卡片中的数据、数字、比例与正文一致
- [ ] 金句卡片提取的句子是正文的核心判断句，不是随便摘的过渡句
- [ ] 卡片中的标签/分类名称与正文用词一致（不自己发明叫法）
- [ ] 对比卡片的左右栏方向正确（左旧右新 / 左劣势右优势）

### 1.3 与排版主题的一致性

- [ ] 卡片副标题文字不与正文标题完全重复
- [ ] 卡片传达的核心信息，在正文中确实有对应段落支撑


## 二、视觉侧检查（整体扫）

### 2.1 主题一致性

- [ ] 卡片强调色与当前排版主题一致（深蓝→橙色、暖橙→琥珀、暗色→青/紫/橙）
- [ ] 卡片字体与当前排版主题匹配（深蓝/暗色→无衬线、暖橙→衬线标题+无衬线正文）
- [ ] 下载按钮悬浮色与主题强调色一致

### 2.2 风格多样性

- [ ] 相邻两张卡片不是同一风格或同一配色（避免视觉疲劳）
- [ ] 多张卡片组合中有深底和浅底交替
- [ ] 全文没有连续 3 张以上同底色卡片

### 2.3 视觉规范

- [ ] 没有方格背景（禁止）
- [ ] 没有卡片倾斜（禁止 `transform: rotate()`）
- [ ] 卡片内文字不溢出边界
- [ ] 整篇文章的封面、正文、配卡色调统一，没有"三张皮"


## 三、格式侧检查（复用已有，补全）

- [ ] `_preview.html` 中每张卡片 **不含** `<button class="illu-download-btn">` 行
- [ ] `_preview.html` 页脚已加入指向 `_配图下载.html` 的提示
- [ ] `_配图下载.html` 的 `<head>` 中已引入 html2canvas CDN
- [ ] `_配图下载.html` 的 `</body>` 前已包含 `downloadCard()` 函数
- [ ] `_封面.html` 的 `<head>` 中已引入 html2canvas CDN
- [ ] `_封面.html` 的 `</body>` 前已包含 `downloadCover()` 函数
- [ ] 所有卡片 ID 唯一且连续（illu-1, illu-2, ... illu-N）
- [ ] 配图数量在参考范围内（短文 2-3 / 中等 4-6 / 长文 6-10）


## 四、执行流程

```
配图生成完成
  ↓
逐张过 内容侧（1.1 → 1.2 → 1.3）
  ↓ 不通过 → 修改后重审
  ↓
整体扫 视觉侧（2.1 → 2.2 → 2.3）
  ↓ 不通过 → 调整配色/字体后重审
  ↓
逐项核 格式侧（3）
  ↓ 不通过 → 修复后重审
  ↓ 全部通过
✅ 配图 QA 通过 → 进入朋友圈阶段
```

**QA 输出格式：**

```
📋 配图 QA 报告（共 X 张卡片）

内容侧：✅ 通过（类型匹配 X/X · 信息准确 X/X）
视觉侧：⚠️ 1 项注意 → 第 3 张暗底卡与第 4 张暗底卡连续，建议中间插一张暖底
格式侧：✅ 通过

总体：⚠️ 有 1 项视觉建议，可选修 → 进入朋友圈阶段
```

通过标准：

| 等级 | 条件 | 行为 |
|------|------|------|
| ✅ 通过 | 内容侧全部 ✅ + 格式侧全部 ✅ | 进入朋友圈阶段 |
| ⚠️ 有建议 | 内容侧全部 ✅ + 格式侧全部 ✅ + 视觉侧有"注意" | 可选修，不阻塞流程 |
| ❌ 不通过 | 内容侧或格式侧有 ❌ | 修复后重新 QA |
