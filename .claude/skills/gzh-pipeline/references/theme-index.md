## 选择流程

### 从 content-brief 带过来的选题标签

如果 `drafts/文章定位.json` 或 `style-brief.json` 中已标注 `content_type` 和 `mood`，按以下映射选择：

| 内容类型 × 情绪 | 建议主题 |
|----------------|---------|
| 案例文 × 冷静 | **深蓝商务** |
| 案例文 × 冲击 | **科技暗色** 或 深蓝商务（高对比变体） |
| 工具文 × 利落 | **科技暗色** |
| 观点文 × 冲击 | **科技暗色** |
| 故事文 × 亲和 | **暖橙故事** |
| 教程文 × 利落 | **深蓝商务** 或 **科技暗色** |
| 解读文 × 冷静 | **深蓝商务** |

### 手动选择

用户未指定主题时，列出全部 3 个主题供选择（默认推荐放在第一位）。用户说"直接排""一键排版"时跳过选择，按内容类型自动匹配。

### 选择方式

```
用户说"用深蓝商务" / "用科技暗色" / "用暖橙故事"
或在排版阶段说"1"、"2"、"3" 选择
```

## 主题文件结构规范

每个主题文件包含：

1. **设计变量速查表** — 主色、浅底、深字、标题色、正文色、分割线色、强调色
2. **封面颜色方案** — 基底渐变、柔光色、强调色
3. **正文排版 CSS** — 可直接粘贴使用的完整 CSS（全部 inline 化）
4. **配图卡片强调色** — 卡片模板的 accent 色值
5. **下划线/标记风格** — 关键词标记 CSS
