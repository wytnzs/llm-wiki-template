# 反馈回写 Skill

> 核心原则：每次失败都要让系统变得更强。每一次失败都有一条根因链条。
> 更新文件前先读取根目录 README；表中的路径是默认值，实际位置以 README 为准。

---

## 什么时候触发

| 触发场景 | 说明 |
|---------|------|
| 写出来的内容感觉不对 | 口吻不对、结构不对、用户说"不是这个意思" |
| 选题方向偏差 | 用户说"不想写这个" |
| 案例调用错误 | 用错了案例，或结论不匹配 |
| 知识卡片调用不足 | 内容单薄，缺少框架支撑 |
| 平台适配失败 | 公众号内容直接当短视频脚本用 |
| content-brief 不完整 | 缺材料、缺目标读者、缺禁忌 |

## 根因分类体系

| # | 根因 | 表现 | 建议更新对象 |
|---|------|------|-------------|
| 1 | 知识卡片不足 | 内容单薄、缺少框架 | 02-Areas/知识卡片/ |
| 2 | 主题页不清晰 | 同一主题多次相似失误 | 02-Areas/主题中心/主题页/ |
| 3 | 任务方向不清 | 没有目标、受众或来源 | 当前项目；已启用选题库时可更新选题卡 |
| 4 | content-brief 缺字段 | 缺 target_person / forbidden | content-brief.template.json |
| 5 | 个人风格约束不准 | 口吻不对、风格偏离 | 个人风格卡片 |
| 6 | 平台 skill 规则缺失 | 平台调性适配不到位 | 对应平台 skill |
| 7 | 案例/证据边界不足 | 案例标签不准确、边界不清 | 案例库对应条目 |
| 8 | 生产 skill 流程缺陷 | 漏步骤、顺序错 | 对应 SKILL.md |

## 反馈回写流程

### 第一步：记录失败
在复盘表中追加：日期 | 任务 | 使用 skill | 问题类型 | 根因分类 | 上游文件 | 修正动作

### 第二步：追溯根因
问三组问题：
1. 问题出在哪个层？没素材/选错题/写坏了
2. 根因是单点还是系统？一个内容/多个选题
3. 修正够不够？同样错误还会再犯吗

### 第三步：更新源头
根据根因更新对应的源头文件。

### 第四步：验证
确认同一问题不再出现。

---

## 常见根因对照表

| 问题类型 | 最可能的根因 | 首次修正位置 |
|---------|------------|------------|
| 方向不清 | 任务定义或来源不完整 | 当前项目；选题库已启用时更新对应选题卡 |
| 案例调用错误 | 案例/证据边界不足 | 案例库对应条目 |
| 口吻不对 | 个人风格约束不准 | 个人风格卡片 |
| 结构像文章不像视频 | 平台 skill 规则缺失 | 对应 skill |
| 缺少用户痛点 | 知识卡片不足 | 知识卡片对应领域 |
| 同一主题反复出问题 | 主题页不清晰 | 主题中心对应主题页 |
