# 选题挖掘引擎 Topic Selection

> 选题不是找标题。选题是：**发现一个真实问题 + 找到你独有的解答角度 + 判断你是否有表达欲**。

---

## 零、知识库联动

启动选题前先检查 02-Areas/知识卡片/：
1. 02-Areas/知识卡片/保险/ —— 已沉淀概念哪些还没写成文章？
2. 02-Areas/知识卡片/认知/ —— 能从思维模型找跨界选题吗？
3. 02-Areas/知识卡片/创作/ —— 当前方法论能否产生新角度？
4. 02-Areas/知识卡片/个人IP/ —— IP定位是否需要调整？

逻辑：知识库已有卡片说明有积累，优先变文章。

## 核心原则

1. **先挖掘，后筛选**。不要在挖掘阶段边挖边否。
2. **选题枯竭的本质不是"没东西写"，而是"没在正确的地方找"**。
3. **好选题来自张力**：读者以为的 vs 实际上的、常见的 vs 被忽略的。
4. **选题 = 对的人 x 对的时机 x 对的内容**。

## 一、触发模式

### 模式 A：选题枯竭急救（默认）
全量扫描纺锤两端 9 条路径 → 归入五棵选题树 → 三重筛选 → 输出选题菜单

### 模式 B：定向深耕
以指定主题为中心，输出 3-5 个角度选项

### 模式 C：缺口检查
扫描近期文章 → 对照五棵树统计分布 → 输出缺口报告

---

## 二、纺锤结构：两端的挖掘路径

### A 端：从用户需求出发（5条路径）

| 路径 | 方法 |
|------|------|
| A1 欲望/障碍/恐惧 | 列出用户最常问的 10 个问题、最担心的 5 件事 |
| A2 人物小传法 | 定义具体人物，写 3 个最近的烦恼 |
| A3 对标二次创作 | 找同领域高传播内容，用自己的视角重做 |
| A4 行业动态追踪 | 监管变化、典型判例、数据报告 |
| A5 公共热点 | 找到只有你能提供的独特角度 |

### B 端：从人设特质出发（4条路径）

| 路径 | 方法 |
|------|------|
| B1 重要时刻挖掘 | 列出 10 个影响最大的职业时刻 |
| B2 专业声誉挖掘 | 回顾最近 10 次被咨询的内容 |
| B3 已验证能力挖掘 | 列出最近 5 个案例，提取外行人不知道的细节 |
| B4 品味/偏好注入 | 列出 3 个看不惯、3 个坚信 |

## 三、五棵选题树

| 选题树 | 受众价值 | 举例 |
|--------|---------|------|
| AI工具实操 | "看完就能用" | AI辅助保单整理 |
| 代理人效率与方法论 | "原来可以这样做" | 工作流优化 |
| 理赔案例拆解 | "这个人真的懂" | 拒赔复议案例 |
| 行业趋势与认知 | "比同行先看到" | 政策解读 |
| 个人创业实录 | "这个人是真实的" | 创业复盘 |

---

## 四、三重筛选

1. **相关性** —— 能写出具体用户场景吗？
2. **差异性** —— 能填出"很多人以为 X → 但其实 Y"吗？
3. **表达欲** —— 你有表达冲动吗？

---

## 五、用户选择后的操作

1. **生成 content-brief** —— 写入 .claude/drafts/content-brief.json
2. **更新选题卡** —— 06-选题库/cards/ 标记为已选择
3. **写入未使用选题** —— 追加到 06-选题库/待挖掘方向.md

> content-brief.json 是唯一核心生产交接单。不再使用 delivery-brief.json。

---

## 六、关键约束

1. 本 skill 产出"候选人选"，不替 IP skill 做人设终筛。
2. 五棵选题树穷尽，归不进树就检查内容定位。
3. 三重筛选不可跳过。
4. 选题菜单是建议，不是命令。
5. 时效性标注要诚实。
