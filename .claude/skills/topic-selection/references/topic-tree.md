# 五棵选题树：内容支柱详解

> 本文档是 `SKILL.md` 第三章的详细展开。包含每棵树的定位、子分支、选题特征、覆盖追踪方法。

---

## 选题树总览

五棵选题树是内容体系的骨架。所有选题最终归入其中一棵——如果归不进去，要么是选题超出了定位范围，要么是需要一棵新树（但五棵树已经是穷举的）。

```
                ├─ AI工具实操（核心引擎）
                │
                ├─ 代理人效率与方法论
                │
     "让保险    ├─ 理赔案例拆解（能力背书）
     变简单          │
     让承诺      ├─ 行业趋势与认知
     被兑现"         │
                └─ 个人创业实录（情感连接）
```

五棵树不是孤立的。同一篇文章可以关联多棵树（如一篇理赔案例 + AI工具使用 = 跨 AI工具实操 和 理赔案例拆解），但**按文章的主要价值归属判断树**。

---

## 一、AI工具实操

### 定位

**核心引擎**。展示"这个人真的懂AI、能做出东西"——不只是会用AI写文案，而是把AI工具真正落地到保险业务场景。

### 子分支

```
AI工具实操
├─ 具体工具使用教程
│   ├─ 单工具深度教学（如Claude、飞书多维表格）
│   ├─ 多工具组合工作流（Claude + 飞书 + DeepSeek API）
│   └─ 工具配置/部署（Cloudflare Worker、自动化脚本）
│
├─ AI×保险场景落地
│   ├─ AI辅助理赔（材料整理、条款对照、申诉信草拟）
│   ├─ AI辅助展业（客户分析、需求诊断、方案生成）
│   └─ AI辅助内容生产（选题、写稿、排版）
│
└─ 工具思维与方法论
    ├─ 如何判断一个任务能不能自动化
    ├─ 如何选择和评估AI工具
    └─ 从"用AI"到"搭AI"的进阶路径
```

### 选题特征

- **具体**：有截图、有步骤、有工作流、可以照着做
- **落地**：不讲"AI多重要"，讲"今天打开这个工具，做这件事"
- **有成果**：展示使用前vs使用后的效率对比

### 受益受众

核心人群（28-42岁，做了3年以上的代理人）：最直接的效率提升需求
辐射人群（入行1-2年）：对工具好奇但不知道从哪开始

### 选题示例

| 类型 | 好选题（具体/可操作） | 差选题（抽象/泛泛） |
|------|---------------------|-------------------|
| 教程 | "用Claude 5分钟做一份家庭保单分析报告——附完整提示词" | "AI如何提升保险工作效率" |
| 工作流 | "我的AI理赔工作流：从收到拒赔通知到输出申诉策略只需3步" | "AI赋能保险理赔" |
| 方法 | "判断一个重复劳动能不能自动化的3个标准" | "保险代理人应该拥抱AI" |

## 二、代理人效率与方法论

### 定位

分享工作方法和思维模型。让读者产生"原来可以这样做"的感觉。和 AI工具实操的区别：**这棵树关注的是方法本身，不限于AI工具**。

### 子分支

```
代理人效率与方法论
├─ 工作流优化
│   ├─ 客户跟进系统搭建
│   ├─ 信息管理方法（保单整理、客户档案）
│   └─ 日/周/月效率系统
│
├─ 思维模型
│   ├─ 保险销售/服务的底层逻辑
│   ├─ 从"卖产品"到"做服务"的认知转换
│   └─ 专业化路径设计
│
└─ 内容营销方法
    ├─ 公众号/视频号运营方法论
    ├─ 朋友圈内容策略
    └─ 个人品牌建设路径
```

### 选题特征

- **系统性**：不是碎片技巧，是可复用的方法框架
- **可迁移**：读者看完能用到自己的工作中
- **有洞见**：不只是"怎么做"，还有"为什么这样做"

### 受益受众

核心人群：业绩遇到瓶颈、想提升专业形象的代理人
辐射人群：想转型做保险的其他行业从业者

### 选题示例

| 类型 | 好选题 | 差选题 |
|------|--------|--------|
| 工作流 | "高绩优代理人的客户跟进表长什么样——附模板" | "做好客户管理很重要" |
| 思维模型 | "从'卖保险的'到'帮人解决问题的'——专业化的3个阶梯" | "代理人要专业化" |
| 内容营销 | "朋友圈发了3个月没人回复，问题不在文案——在定位" | "朋友圈要多发内容" |

## 三、理赔案例拆解

### 定位

**能力背书**。用真实案例证明专业实力。这是五棵树中最有说服力的内容类型——说一百遍"我懂理赔"，不如展示一个真实的翻盘案例。

### 子分支

```
理赔案例拆解
├─ 真实拒赔复议/翻盘案例
│   ├─ 疾病险（既往症、如实告知、等待期争议）
│   ├─ 意外险（事故认定、免责条款、伤残鉴定）
│   └─ 医疗险（合理且必要、DRG影响、自费项目）
│
├─ 条款深度解读
│   ├─ 常见拒赔理由的法律分析
│   ├─ "格式条款"vs"协商条款"的实务区分
│   └─ 不同保险公司条款的横向对比
│
└─ 理赔流程实操指南
    ├─ 出险后第一步该做什么（标准动作清单）
    ├─ 理赔材料准备（常见缺漏和补救）
    └─ 和保险公司沟通的策略与话术
```

### 选题特征

- **真实案例**：有具体经过、有处理过程、有最终结果（脱敏处理）
- **信息差**：展示外行人不知道的内部逻辑
- **可操作**：不只是讲故事，有提炼出的方法论

### 受益受众

核心人群：理赔专业能力是短板的代理人（对他们来说这是"刚需"内容）
辐射人群：有保险需求的普通消费者（被内容吸引后可能转化为客户/咨询）

### 选题示例

| 类型 | 好选题 | 差选题 |
|------|--------|--------|
| 案例 | "被拒赔3次的一个案子，翻盘关键不是法律条文——是一个被忽略的细节" | "一个理赔案例分享" |
| 条款 | "保险公司说'不属于保险责任范围'——这句话背后的法律漏洞" | "保险条款很重要" |
| 指南 | "出险后24小时内，这5个动作决定了你能不能拿到钱" | "出险后要及时报案" |

## 四、行业趋势与认知

### 定位

解读行业变化，提供判断框架。让读者觉得"这个人比我先看到趋势"。

### 子分支

```
行业趋势与认知
├─ 政策解读
│   ├─ 监管新规对代理人/客户的影响
│   ├─ 医保改革（DRG/DIP）与商业保险的关系
│   └─ 保险法修订/司法解释的实务影响
│
├─ 市场变化分析
│   ├─ 代理人数量/结构变化
│   ├─ 保险公司经营策略变化
│   └─ 保险科技/互联网保险趋势
│
└─ 代理人转型路径
    ├─ 从"销售导向"到"服务导向"的行业转型
    ├─ 独立代理人/保险事务所模式
    └─ 跨界能力建设（法律/医学/AI）
```

### 选题特征

- **有判断**：不只是"发生了X"，还有"这意味着Y，你应该做Z"
- **有框架**：给读者一个分析类似问题的思维工具
- **不过度预测**：基于事实的分析，不是画大饼

### 受益受众

核心人群：关注行业变化的中高级代理人
辐射人群：保险公司管理人员、保险科技从业者

### 选题示例

| 类型 | 好选题 | 差选题 |
|------|--------|--------|
| 政策 | "DRG改革后医疗险的卖法要彻底改变——不是危言耸听，是数学" | "DRG改革对保险行业的影响" |
| 市场 | "代理人从900万降到200万——不是行业不行了，是旧玩法不行了" | "保险代理人数量下降" |
| 转型 | "未来5年，保险代理人的3条出路——你适合哪一条？" | "代理人要转型升级" |

## 五、个人创业实录

### 定位

**情感连接**。记录真实的创业经历、思考、复盘。让读者觉得"这个人是真实的"——这是建立信任最朴素也最有效的方式。

### 子分支

```
个人创业实录
├─ 创业思考与复盘
│   ├─ 从内勤到创业的心路历程
│   ├─ 关键决策的思考过程（为什么选这条路）
│   └─ 失败/踩坑经历（比成功更有价值）
│
├─ 工具开发记录
│   ├─ 从想法到产品的过程
│   ├─ 技术选型与trade-off
│   └─ 用户反馈与迭代
│
└─ 效率提升实验
    ├─ 日/周/月工作流迭代
    ├─ 工具/系统切换记录
    └─ 时间管理实验
```

### 选题特征

- **真实**：晒过程不只是晒结果，有得有失
- **有反思**：不只是记录发生了什么，还有"我当时怎么想的""现在回头看怎么做更好"
- **有普适性**：虽然是个人经历，但能提炼出对读者有用的认知

### 受益受众

所有人群——这棵树产出的内容建立的是情感连接和信任基础，跨越人群分层。

### 选题示例

| 类型 | 好选题 | 差选题 |
|------|--------|--------|
| 复盘 | "做自媒体3个月，最大的收获不是涨粉——是对'什么是好内容'的理解变了" | "自媒体3个月涨粉XXX" |
| 开发 | "我花了两周做了一个理赔分析工具——结果第一个用户说'挺好，但我用不上'" | "我开发了一个AI工具" |
| 实验 | "试了一个月'每天6点起床写作'——失败报告" | "早起的好处" |

## 覆盖追踪方法

### 如何判断一篇文章属于哪棵树

按**主要价值归属**判断，不是按表面内容：

| 文章内容 | 主要价值 | 归属 |
|---------|---------|------|
| 用Claude搭建了保单分析系统 | 工具可复制 | AI工具实操 |
| 代理人如何用飞书管理200个客户 | 方法论可迁移 | 代理人效率与方法论 |
| 一个拒赔案的完整处理过程 | 能力证明 | 理赔案例拆解 |
| DRG改革对医疗险销售的影响分析 | 认知领先 | 行业趋势与认知 |
| 创业第3个月的数据和反思 | 建立连接 | 个人创业实录 |

### 覆盖追踪表（模板）

每次选题或发布后更新：

```markdown
## 选题树覆盖追踪 — 更新于 YYYY-MM-DD

| 选题树 | 最近一篇 | 日期 | 近30天篇数 | 状态 |
|--------|---------|------|-----------|------|
| AI工具实操 | 《xxx》 | 06-15 | 2 | ✅ 正常 |
| 代理人效率与方法论 | 《xxx》 | 05-28 | 1 | ⚠️ 快满月 |
| 理赔案例拆解 | 《xxx》 | 06-10 | 1 | ✅ 正常 |
| 行业趋势与认知 | — | — | 0 | 🔴 缺口 |
| 个人创业实录 | 《xxx》 | 06-01 | 1 | ✅ 正常 |
```

### 轮转规则

1. **缺口优先**：有树超过一个月未覆盖 → 下次选题优先选这棵树
2. **防止偏科**：某棵树连续 3 篇 → 提示轮转到其他树
3. **自然节奏**：不是严格轮流，但每棵树每月至少要有 1 篇（个人创业实录可放宽到 6 周）
4. **热点例外**：行业趋势类内容如果有重大新闻，可以临时增加频次

### 健康指标

| 指标 | 健康 | 警告 | 危险 |
|------|------|------|------|
| 单树最长空白 | <30天 | 30-45天 | >45天 |
| 单树连续篇数 | ≤2篇 | 3篇 | ≥4篇 |
| 月产出总数 | 4-8篇 | <4篇 | <2篇 |
| AI工具实操占比 | 25-35% | <20% | <10%（核心引擎不能停） |
