# 纺锤两端：9条挖掘路径详解

> 本文档是 `SKILL.md` 第二章的详细展开。每条路径包含：方法、提示词模板、实战示例、常见坑。

---

## 纺锤 A 端：从用户需求出发

A 端的核心逻辑：**用户已经在想问题、遇到困难、感到恐惧——你只需要把这些变成选题。**

---

### A1 — 欲望/障碍/恐惧挖掘

**核心逻辑**：最高效的选题来源是用户已经在想的问题。不要猜，去列出来。

**操作步骤**：
1. 拿出一张纸（或一个空白文档），写三列：欲望 | 障碍 | 恐惧
2. 欲望列：目标用户最想得到什么？（提升效率、提升专业形象、获得客户信任...）
3. 障碍列：什么东西拦在他们和目标之间？（不知道从哪开始、工具太多选不来、怕投错方向...）
4. 恐惧列：他们最怕发生什么？（被AI淘汰、被同行甩开、被客户觉得不专业...）
5. 每个格子里写至少 3 条，然后逐条问：这个能写成一篇文章吗？

**提示词模板**（给AI）：
```
我的目标用户是：[描述]
他们最想实现的3个愿望是什么？
他们最常遇到的3个障碍是什么？
他们最害怕的3件事是什么？
针对每个答案，给出一个可以写成的选题方向。
```

**实战示例**：
- 欲望："我想让客户觉得我和其他代理人不一样" → 选题方向：如何用专业内容建立差异化（而非靠人情）
- 障碍："AI工具太多了，不知道从哪个开始" → 选题方向：保险代理人AI工具选型指南（按场景推荐）
- 恐惧："ChatGPT出来了，我写的文案还不如AI" → 选题方向：AI写不好保险文案的三个原因（人的价值在哪）

**常见坑**：
- 问得太宽："用户想要什么？" → 问具体："用户上周遇到的最头疼的一件事是什么？"
- 自己替用户回答 → 去翻用户的实际留言、私信、评论，用真实需求

### A2 — 人物小传法

**核心逻辑**：抽象的"用户画像"产不出好选题。必须具体到一个人。

**操作步骤**：
1. 定义一个具体的人：姓名、年龄、职业、从业年限、家庭状况、性格特点
2. 写 3 个他最近遇到的烦恼（具体场景，不是抽象问题）
3. 问：如果他坐在你对面，你最想告诉他什么？
4. 把你想告诉他的事，变成一个选题

**提示词模板**（给AI）：
```
请帮我刻画一个具体的用户画像：
- 姓名、年龄、职业、从业年限
- 他最近工作中最困扰的3件事（越具体越好，带上场景）
- 他订阅了哪些公众号/博主？（了解他的信息消费习惯）
- 他打开一篇文章最可能的原因是什么？

基于这个画像，提出3个他会主动点开看的选题。
```

**实战示例**：
- 人物：小王，32岁，保险代理人，从业5年，已婚有娃
- 烦恼1：上个月客户问"买了医疗险还要买重疾险吗"，他讲了一堆但客户明显没听懂
- 烦恼2：公司要求用AI工具提升效率，但他打开Claude不知道让它干什么
- 烦恼3：看到同行在写公众号/拍视频，他觉得自己也该做但不知道怎么开始
- → 选题1：医疗险vs重疾险，一句话讲清楚（别再背条款了）
- → 选题2：Claude第一天该干什么——保险代理人的AI上手清单
- → 选题3：想写公众号但不知道写什么？从你上周被问得最多的问题开始

**常见坑**：
- 人物太模糊（"一个代理人"）→ 必须有名有姓有具体场景
- 烦恼太抽象（"业绩不好"）→ 要挖到具体表现："上个月聊了20个客户，成交2个，剩下的说'再考虑考虑'"

### A3 — 对标二次创作

**核心逻辑**：好选题不在真空中。看到别人写的好内容，用自己的专业背景和经历重做。

**操作步骤**：
1. 找到一篇同领域高传播内容（阅读量高、评论多、被转发）
2. 问自己三个递进问题：
   - 我同意吗？（同意→为什么我写会更好？不同意→分歧点是什么？）
   - 漏了什么？（他因为背景/经历限制没说到的部分）
   - 怎么重做？（用我的案例、我的角度、我的方法重新写）
3. 写出你的版本和原版的三个核心区别

**提示词模板**（给AI）：
```
这是一篇同领域的高传播内容：[标题/链接/摘要]

请帮我分析：
1. 这篇文章的核心观点是什么？受众为什么买账？
2. 如果从"保险理赔后端"（而非销售前端）的角度重写，可以增加什么？
3. 如果从"真的用过AI工具的人"（而非泛泛而谈）的角度重写，可以增加什么？
4. 从以上分析中提炼3个选题方向。
```

**实战示例**：
- 原文："2026年保险代理人必备的5个AI工具"
- 问题：泛泛介绍工具功能，没有工作流整合
- 重做角度："我用这5个工具搭了一套保险代理人的AI工作流——从客户咨询到方案输出只需20分钟"
- 核心区别：原文是工具列表，重做是工作流展示；原文谁都能写，重做只有真的用过才能写

**常见坑**：
- 变成洗稿 → 必须有自己的增量信息（新案例、新方法、新数据）
- 挑了太烂的原内容 → 对标要选传播好的，不是随便一篇
- 只看同行的 → 跨行业的好选题往往更有启发

### A4 — 行业动态追踪

**核心逻辑**：每一次政策变化、典型判例、行业数据更新，都有一群人在问"这对我的影响是什么？"——你的选题就是回答这个问题。

**操作步骤**：
1. 建立信息源列表（监管网站、法院判决数据库、行业报告、36kr/虎嗅保险频道...）
2. 定期扫描（政策类每天、判例类每周、报告类每月）
3. 每看到一个信号，回答三个问题：
   - 这件事对谁有影响？
   - 影响有多大？（颠覆性的/渐进的/可忽略的）
   - 受影响的人现在应该做什么？
4. 把这三个问题的答案变成一个选题方向

**提示词模板**（给AI）：
```
最近有这些行业动态：[描述]

请针对每条动态回答：
1. 受影响的群体是谁？（代理人/客户/保险公司/所有人）
2. 大多数人会怎么误解这个变化？
3. 真正需要关注的3个要点是什么？
4. 现在应该做什么？（具体可操作的建议）

基于以上分析，给出选题方向。
```

**时效窗口判断**：
| 动态类型 | 最佳窗口 | 可写期限 | 超过期限的处理 |
|---------|---------|---------|--------------|
| 监管新规发布 | 24h内 | 3天内 | 如果已有大量解读，除非有独特角度否则放弃 |
| 典型判例公布 | 3天内 | 1周内 | 可以做成案例合集或趋势分析 |
| 行业数据报告 | 1周内 | 2周内 | 和往年数据做对比分析仍有价值 |
| 公司重大事件 | 24h内 | 3天内 | 除非能提炼出普适性的教训 |

**常见坑**：
- 只转述不解读 → 读者自己会看新闻，需要的是解读
- 只讲"是什么"不讲"怎么做" → 信息不是内容，方法论才是
- 过了窗口还在追 → 时效选题过时不候

### A5 — 公共热点 + 独特角度

**核心逻辑**：热点是流量的风口，但99%的热点内容都是同质化的。你需要的是只有你能提供的那个角度。

**操作步骤**：
1. 看到一个公共热点，先问："这和保险/理赔/代理人有关系吗？"
2. 如果有关系，再问："我的专业背景能给这个热点什么独特解读？"
3. 列出 3 个可能的角度，选最"只有我能写"的那个
4. 如果没有独特角度 → 不追。宁可错过，不要凑数。

**角度挖掘框架**：
```
热点事件：[简述]

常规角度（别人怎么写）：
- [角度1] [角度2] [角度3]

我的独特角度（只有我能写）：
- 从理赔后端看：这件事如果发生在保险场景下会怎样？
- 从AI工具看：有什么工具可以避免/解决这种问题？
- 从代理人视角看：客户会因为这件事产生什么新的顾虑？
```

**实战示例**：
- 热点：某明星因病住院，医疗费用引发热议
- 常规角度："明星的医疗险配置分析""高端医疗险科普"
- 独特角度："从病历到理赔——如果明星的住院病历到了保险公司手里，哪些地方可能被拒赔？"（只有懂理赔内部流程的人能写）

**红线**：
- 不要消费悲剧（灾难、重大事故）
- 不要蹭和保险毫无关系的热点
- 不要制造恐慌（"你不买这个保险也会像XXX一样"）

## 纺锤 B 端：从人设特质出发

B 端的核心逻辑：**你经历过的事、你擅长的事、你坚信的事——这些内容别人写不出来。**

---

### B1 — 重要时刻挖掘

**核心逻辑**：你职业生涯中的转折点、至暗时刻、高光时刻——每一个都是天然的故事选题。

**操作步骤**：
1. 列出 10 个对你职业影响最大的时刻（可以是事件、对话、决定）
2. 按情感强度排序：哪个让你印象最深？
3. 针对 Top 3，问：
   - 那个时刻发生了什么？（时间、地点、人物、具体对话）
   - 它为什么重要？（它改变了我什么认知/行为/方向）
   - 读者能从中学到什么？（提炼可迁移的经验）
4. 如果读者能带走的经验是通用的 → 好选题
5. 如果只是个人经历的炫耀 → 换一个

**提示词模板**（给AI）：
```
回顾你的职业生涯，列出10个对你影响最大的时刻。每个时刻写：
- 发生了什么（具体时间、地点、画面）
- 为什么重要（它改变了什么）
- 如果分享出去，读者能学到什么

然后选出3个最适合公开分享的，给出选题角度。
```

**实战示例**：
- 时刻：2020年入职即遇疫情，目睹公司对40岁以上员工系统性裁员
- 学到了什么：如果停滞不前，十几年后就是这个结局
- 选题角度："25岁入行那年，我看到40岁的同事一个个被裁——这件事改变了我对'稳定'的所有理解"
- 读者可获得：对职业安全的清醒认知 + 主动成长的动力

**常见坑**：
- 故事太私人化，读者无感 → 必须有可迁移的经验或认知
- 故事太模糊（"有一段时间我很迷茫"）→ 必须有具体的转折场景
- 只有情绪没有方法 → 好故事 + 好方法 = 好选题

---

### B2 — 专业声誉挖掘

**核心逻辑**：别人因为什么来找你，你就该写什么。这不需要猜，不需要调研——需求已经在敲门了。

**操作步骤**：
1. 回顾最近 10 次被咨询/请教/求助的内容
2. 按主题归类：哪些问题反复出现？
3. 对每个高频主题，展开 3 个角度：是什么（概念澄清）/ 为什么（底层逻辑）/ 怎么做（实操方法）
4. 优先选择"被问了3次以上但还没写过的"主题

**高频主题→选题转换表**：

| 被问的问题 | 表层需求 | 深层需求 | 选题方向 |
|-----------|---------|---------|---------|
| "XX保险能买吗？" | 产品评测 | 怕买错 | 如何判断一个保险产品适不适合你（授人以渔） |
| "被拒赔了怎么办？" | 操作流程 | 觉得无助 | 被拒赔后的24小时：一个标准动作清单 |
| "AI工具哪个好用？" | 工具推荐 | 怕浪费时间试错 | 按工作场景选AI工具（不是按功能） |
| "怎么写公众号/拍视频？" | 方法教程 | 想差异化但不知道怎么做 | 保险代理人做内容，最容易犯的3个定位错误 |

**常见坑**：
- 只回答表面问题 → 要挖到深层需求
- 被问太多次觉得"这还用写吗" → 你觉得简单的，对别人可能是刚需
- 等问题来了才写 → 提前整理高频问题，主动出选题

---

### B3 — 已验证能力挖掘

**核心逻辑**：你做过的案例、解决过的问题，是最硬的内容。不需要虚构、不需要夸大——事实本身就够有力。

**操作步骤**：
1. 列出最近做过的 5 个具体案例/项目
2. 对每个案例问三个问题：
   - 这个案例里有什么是外行人不知道的？（信息差）
   - 中间哪个环节最容易被搞砸？（教训）
   - 如果重新做一遍，哪个动作最关键？（方法论）
3. 选择信息差最大的案例 → 把信息差变成选题

**案例→选题转换公式**：
```
[案例类型] + [常见误解] + [真实处理方式] = 选题

例：
拒赔案例 + 很多人以为只能起诉 + 其实一封申诉信就解决了 = 
"被拒赔了别急着请律师，先做这件事——一封申诉信让保险公司改口"
```

**常见坑**：
- 涉及客户隐私 → 必须脱敏处理（模糊时间、地点、金额、个人信息）
- 讲案例变成炫耀成绩 → 重点放在"读者能学到什么"，不是"我多厉害"
- 只讲结果不讲过程 → 读者要的是中间的判断和动作，不是结果的数字

---

### B4 — 品味/偏好注入

**核心逻辑**：你讨厌什么、坚持什么，比你喜欢什么更能定义你。有立场的选题比中立的选题更有传播力。

**操作步骤**：
1. 写下你在专业领域内的 3 个"看不惯"——什么做法让你觉得不对？
2. 写下 3 个"坚信"——什么原则你从来不妥协？
3. 对每一条问：有什么具体事例能支撑你的判断？
4. 有具体事例支撑的 → 可以写成选题
5. 只有情绪没有依据的 → 继续积累素材

**"看不惯"→选题示例**：
- 看不惯："用恐吓的方式卖保险"
- 支撑事例：见过代理人跟客户说"你不买重疾险，得了癌症就得卖房"，然后客户买了但从此不信任任何保险人
- 选题方向："靠恐吓卖出去的每一份保险，都在透支整个行业的信用"
- 具体切入：不讲道理，讲那个客户后来的故事

**"坚信"→选题示例**：
- 坚信："保险真正的价值在兑现那一刻，不在签单那一刻"
- 支撑事例：帮客户拿到理赔款后，客户说"谢谢你当时坚持让我加了这个附加险"
- 选题方向："卖保险的人都在讲签单，但保险真正的价值发生在几年后——一个理赔案例"

**常见坑**：
- 只有立场没有依据 → 变成了发泄情绪，不是内容创作
- 立场太极端 → 有理有据地表达，不是无差别攻击
- 偏离核心受众 → 你的"看不惯"必须是受众也关心的，不是纯粹的个人喜好

## 挖掘效率提示

1. **一次扫一条路径，不要跳**。A1→A2→A3→A4→A5→B1→B2→B3→B4，按顺序过。跳来跳去会漏。
2. **每条路径的目标是 1-2 个方向**，不是 10 个。9×1.5 = 大约 13-14 个初筛方向，足够了。
3. **先写下来，再评判**。不要在挖掘阶段自我审查——想到什么都记下来，三重筛选是后面的事。
4. **如果某条路径产不出任何方向**，记录下来——这本身就是信号（信息来源有缺口）。

## 联网搜索策略（附录）

> 本附录详细说明 A3/A4/A5 三条路径中联网搜索的具体策略。
> 所有搜索操作通过 **web-access skill**（`Skill("web-access")`）执行。

### 搜索原则

1. **并行优先**：A3/A4/A5 的搜索可以同时进行，不要串行等待
2. **广度优先**：先做 2-3 个宽泛搜索看有哪些方向，再做精准搜索深挖
3. **时效过滤**：优先最近 7 天（热点）、最近 30 天（行业动态）、不限时（对标内容）
4. **最少有效信息**：每个搜索方向收集 2-3 条有效信号即可，不要陷入无限浏览

### A3 对标内容搜索

**目标**：找到同领域的高传播/高质量内容，用你的视角重做。

**搜索查询模板**：

| 搜索方向 | 查询示例 | 目标源 |
|---------|---------|--------|
| 公众号热文 | `保险 AI 工具 效率 2026`（微信搜一搜） | mp.weixin.qq.com |
| 保险科技 | `site:36kr.com 保险科技 OR 保险AI` | 36kr.com |
| 理赔话题 | `拒赔 翻盘 案例 公众号` | 微信公众号 |
| 代理人话题 | `代理人 转型 出路 2026` | 微信公众号/知乎 |
| 小红书热帖 | `保险 避坑 推荐` 按热度排序 | xiaohongshu.com |

**搜索后处理**：
1. 挑 2-3 篇阅读量/互动量最高的 → 分析其核心观点和受众为什么买单
2. 对每篇问：我同意吗？漏了什么？怎么重做？
3. 输出：`原内容 [标题/核心观点] → 重做角度 [你的差异化切入点]`

### A4 行业动态搜索

**目标**：找到最新的监管变化、典型判例、行业数据变化。

**搜索查询模板**：

| 搜索方向 | 查询示例 | 目标源 |
|---------|---------|--------|

### A5 热点话题搜索

**目标**：找到当前正在被广泛讨论的话题，从中挖掘保险/理赔角度的切入点。

**搜索查询模板**：

| 搜索方向 | 查询示例 | 目标源 |
|---------|---------|--------|
| 热搜扫描 | 直接打开微博热搜榜、百度热搜榜 | weibo.com, baidu.com |
| 财经热点 | 访问 36kr 热榜、财新网要闻 | 36kr.com, caixin.com |

| 科技×保险 | `AI 保险 应用 OR 争议 OR 案例` | 36kr, 虎嗅, TechCrunch |

**热点→选题筛选漏斗**：
1. 看到热点 → 问"这和保险/理赔有关系吗？"
   - 无关系 → 舍弃
   - 有关系 → 继续
2. 问"我的受众会在意这个角度吗？"
   - 不在意 → 舍弃
   - 在意 → 继续
3. 问"我有什么独特的角度？别人写不出来的？"
   - 能写出至少 1 个 → 入选
   - 写不出 → 舍弃

**红线（搜索时注意，不要碰）**：
- 灾难/重大事故 → 不蹭悲剧
- 和保险毫无关系的纯娱乐热点 → 不硬拗角度
- 涉及逝者/伤者的热点 → 不消费痛苦
