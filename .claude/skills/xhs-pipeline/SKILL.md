# 小红书内容管线 XHS-Pipeline

> 全流程：微信链接 → 抓取 → 分析 → 设计风格+版面 → 种子模板生成轮播图 → 发布文案
> 视觉体系：Slide · PPT 风格（默认） / Editorial Magazine × E-ink / Swiss International

---

## 知识库联动

写作前必须执行：

1. 搜索 `02-Areas/知识卡片/保险/` —— 提取案例/观点作为素材
2. 搜索 `02-Areas/知识卡片/认知/` —— 融入思维模型
3. 搜索 `02-Areas/知识卡片/创作/` —— 参考写作方法

发布后主动提炼知识卡片：识别核心知识点，建议标题和领域，询问是否写入 02-Areas/知识卡片/。

## 完整工作流

```
🤖 【抓取阶段】→ 微信文章抓取
  ↓
🤖 【分析阶段】→ 文章结构分析
  ↓
🤖 【设计阶段】→ 读取 xhs-scale + 选风格模板 + 主题
  ↓                      → 视觉结构分析（7 种结构匹配）
  ↓                      → 按结构规划卡片序列
  ↓
👤 【确认阶段】→ 用户确认视觉结构 + 卡片序列 + 风格
  ↓
🤖 【生成阶段】→ 从种子模板生成 HTML（含下载工具栏）
  ↓
🤖 【文案阶段】→ 生成发布文案（标题 + 正文 + 标签）
  ↓
🔍 【QA 阶段】→ 内容侧 + 视觉侧 + 格式侧 + 文案检查
  ↓
🤖 【收尾阶段】→ 交付并询问反馈
```

### 第零步：检查上游交接

#### 0.0 读取小红书传播尺度

设计前必须读取 `references/xhs-scale.md`。

小红书不是小号公众号。必须把原文重构成“认知切片”，而不是把公众号正文切碎塞进卡片。

如果 `.claude/drafts/style-brief.json` 存在，优先读取其中的 `route`、`visual_profile`、`image_plan` 和 `platform_plan.xiaohongshu`；没有则按 `xhs-scale.md` 自动决策。

#### 0.1 检查 Content Brief

检查 `.claude/drafts/content-brief.json` 是否存在。

**如果存在**，直接抽取以下核心要素（不需要重新理解全部判例）：
1. **一个核心观点** —— 作为轮播图的中心论点
2. **三个视觉化要素** —— 每条对应一张轮播图卡片
3. **一个冲突场景** —— 用于开头封面制造张力
4. **一个评论区互动点** —— 放到发布文案结尾引导互动

> content-brief.json 是唯一核心生产交接单。不再使用 delivery-brief.json。

**如果不存在**，跳过此步，正常从原文分析。

## 阶段三：设计阶段（风格 + 版面规划）

### 第 1 步：选择风格模式

根据内容调性选择。**默认推荐 Slide 模式**（最通用、最清晰）：

| 模式 | 气质 | 适用内容 |
|------|------|---------|
| **Slide · PPT Style** | 大字标题 + 列表要点，清晰直给，PPT 演讲节奏 | **默认首选**。保险科普、产品对比、政策解读、清单总结 |
| **Editorial Magazine × E-ink** | 杂志深度阅读，温暖纸质感 | 深度分析、个人故事、方法论、金句 |
| **Swiss International** | 精确工程感，数据导向 | 数据报告、实操指南、对比测评、清单 |

选择建议参考 `references/theme-presets.md`。

### 第 2 步：选择主题色

- Slide 默认用 **kraft-paper** 暖牛皮纸配色（`#eedfc7` 底 / `#2a1e13` 字 / `#b85c2e` 强调），通过 `theme-warm` 与 `theme-dark` 两张卡片交替节奏
- Editorial 从 6 种主题中选一种（`ink-classic` / `indigo-porcelain` / `forest-ink` / `kraft-paper` / `dune` / `midnight-ink`）
- Swiss 从 4 种强调色中选一种（`ikb` / `lemon-yellow` / `lemon-green` / `safety-orange`）

详见 `references/theme-presets.md`。

### 第 3 步：分析视觉结构（新增）

在规划具体卡片前，先分析内容的内在逻辑属于哪种视觉结构。读取 `references/visual-structure-mapping.md`：

**快速判断：**

| 问题 | 对应结构 |
|------|---------|
| 讲一个"从坏到好"的转变？ | Before/After |
| 是一组操作步骤或流程？ | Pipeline |
| 围绕一个核心概念展开多个方面？ | Hub-and-Spoke |
| 用数据作为主要叙事手段？ | Data-First |
| 讲持续迭代或反复发生的现象？ | Cycle |
| 拆解一个机制或规则？ | Scientific |
| 展示一个体系的多个层级？ | Layer Stack |

确定视觉结构后，按该结构推荐的卡片序列（见 visual-structure-mapping.md）排卡，而不是直接用固定模板。

### 第 4 步：按视觉结构规划 6-8 张卡片

参照 `references/xhs-scale.md` 和 `references/content-planning.md` 规划页面结构。默认优先 7 页：封面、核心结论、原因解释、误区对比、方法路径、收藏清单、CTA。

只有当素材确实足够且每页都有独立价值时，才扩展到 9-10 张。

**Slide 模式卡片映射（PPT 演讲节奏）：**

| 位置 | 卡片类型 | Slide 组件 | 底色 |
|------|---------|-----------|------|
| 第 1 张 | 封面 | `h1` + `tag` + `body-lg` + `bottom-strip` | warm |
| 第 2 张 | 核心变化 | `h2` + `card-box` 对比（dim + 正常） | warm |
| 第 3 张 | 关键数字 | `dark` 暗底 + `big-num` 大数字（140-180px） | **dark** |
| 第 4-5 张 | 案例/对比 | `card-box` 卡片 + 数字网格 | warm |
| 第 6 张 | 要点列表 | `point-row` 编号列表（01-04） | warm |
| 第 7 张 | 金句/彩蛋 | `dark` 暗底 + `big-num` + `divider` | **dark** |
| 第 8 张 | CTA 收尾 | `card-box` 清单 + `callout` 引导语 | warm |

Slide 关键规则：
- **第 3 张和和第 7 张必须用 dark 暗底**，打断全是暖底的单调
- 每张卡片一句话概要（`body-lg`）组件展开
- 数字用 `big-num`（140px+）或 `mid-num`（88px），强调色突出
- 卡片之间用 `card-box`（左侧 accent 色条）和 `point-row`（大编号 + 标题/说明）
- 禁止 emoji，禁止衬线体——全程无衬线（Noto Sans SC）

**Editorial / Swiss 卡片映射：**

| 位置 | 卡片类型 | Editorial 推荐 recipe | Swiss 推荐 recipe |
|------|---------|---------------------|------------------|
| 第 1 张 | 封面 | M01 / M16 | S01 / S02 |
| 第 2 张 | 结果数据 | M04 | S08 / S09 |
| 第 3-4 张 | 概念解释 | M10 / M11 | S12 |
| 第 5-7 张 | 流程/实战 | M14 / M05 | S10 / S09 |
| 第 8 张 | 亮点/对比 | M08 / M15 | S03 / S11 |
| 第 9 张 | 金句/方法论 | M04 | S05 |
| 第 10 张 | CTA | M07（尾页变体） | S01（CTA 变体） |

每个 recipe 的具体结构见 `references/layout-recipes.md`。

### 第 5 步：用户确认

展示视觉结构 + 卡片序列 + 风格模式给用户确认后再进入生成阶段。

## 阶段四：生成轮播图 HTML

### 操作流程

根据选择的风格模式，从对应的种子模板开始编辑：

```bash
# Slide 模式（默认）：复制 slide 种子模板
# Editorial 模式：复制 editorial 种子模板
# Swiss 模式：复制 swiss 种子模板
```

| 模式 | 种子模板 |
|------|---------|
| **Slide 默认** | `assets/template-slide-xhs.html` |
| Editorial | `assets/template-editorial-xhs.html` |
| Swiss | `assets/template-swiss-xhs.html` |

### 具体步骤

1. 在 `小红书/{年份}/{月份}/` 下创建 HTML 文件：`{主题}-小红书版.html`
2. 从种子模板复制内容到该文件
3. **Slide 模式**：交替使用 `theme-warm`（暖底）和 `theme-dark`（暗底），第 3 张和第 7 张必须用 `theme-dark`
4. Editorial：`<html data-theme="ink-classic">`（改为选中的主题）
5. Swiss：`<html data-accent="ikb">`（改为选中的强调色）
6. 替换占位内容为真实文案
7. 保持原有的 `.toolbar` 下载栏不变（html2canvas + JSZip + FileSaver）
8. **CTA 页不出现微信号**，用评论区互动形式替代（如"评论区回复需要，私信发你"）

### 生成规范

- 保持 1080×1440 尺寸，不用调整
- 严格按 `references/xhs-scale.md` 执行 3:4 瞬时传播尺度：每页只讲一个判断，封面要有冲突，至少一页可收藏
- Slide：无衬线字体（Noto Sans SC），禁止 emoji，禁止衬线体；`content` 内边距 100px/96px；用 `grow` 撑满垂直空间
- Editorial：每张卡片必须包含 `.grain`，封面/金句页必须有 `mag-bg`
- Swiss：不用 grain/mag-bg，可选用 `dot-mat` 等装饰矩形
- 内容填充约 55% 画布高度（见 `references/portrait-fill.md`）
- 字体用 class 控制，不写 inline font-size（见 `references/components.md`）

### 视觉规则速查

| 规则 | Slide | Editorial | Swiss |
|------|-------|-----------|-------|
| 字体 | 纯无衬线 (Noto Sans SC) | 衬线(serif)标题 + 无衬线(sans)正文 | 纯无衬线 (Inter + Noto Sans SC) |
| 背景 | warm 暖底 / dark 暗底交替 | grain + 可用 WebGL 墨迹 | 纯色 + 可用装饰矩形 |
| 卡片 | card-box (左侧 accent 色条) + point-row (编号列表) | 无 card 类（用排版表达层级） | card-fill / card-outlined（互斥） |
| 数字 | big-num (140px+) / mid-num (88px) | data-num (96px) | num-mega (168px) |
| 圆角 | 有（card-box 10px） | 有（图片 6px） | **禁止** |
| emoji | **禁止** | **禁止**（用 Lucide） | **禁止**（用 Lucide） |
| 节奏 | 第 3/7 张必须 dark 底打断 | mag-bg 用于封面+金句页 | 全篇纯色统一 |

完整视觉规则见 `references/style-system.md`。

### 材质解释图嵌入规则

当内容包含抽象概念、流程、对比、数据或 AI 工作流时，优先在封面或关键解释页嵌入材质解释图。

读取 `references/xhs-scale.md` 的“材质解释图规则”：

- 图内只放 3-5 个短中文标签
- 标签 2-5 字为佳
- 不把段落塞进图里
- 图必须解释一个关系，不做纯装饰
- 数据图必须保证数值准确

### 风格多样性（读取共享设计标准）

读取 `../references/visual-design-standards.md`（与 gzh-pipeline 共享），为轮播图注入风格变化：

- **相邻卡片风格交替**：8-10 张卡片中，第 1 张（封面）和第 8 张（CTA）可用相同风格形成首尾呼应，中间卡片每张换风格或至少换配色
- **字体多样性**：Slide 模式默认 Noto Sans SC，但金句页/个人感悟卡可换用 Ma Shan Zheng（手写）或 Noto Serif SC（文学感）
- **禁止清单**：方格背景 / 卡片倾斜 / hover 微动动画 / 过渡效果
- **内容模式**：字面模式（数据卡/条款引用）vs 创意模式（金句卡/CTA 卡）
- **金句字数**：小红书卡片 ≤ 20 字，上下文说明不需要

## 阶段五：生成发布文案

### 小红书标题方法论

标题同时服务于**搜索曝光**（含关键词）+ **点击冲动**（有钩子）。标题不超过 20 字，不能标题党——正文必须兑现标题的承诺。

**标题公式**：`[关键词] + [钩子结构]`

关键词放前面（平台搜索权重更高），钩子结构从以下 8 种中选：

**8 种爆款钩子结构：**

| 结构 | 公式 | 示例 |
|------|------|------|
| **结果前置** | 数字/结局甩前面 | "被拒赔15万，凭一张挂号单拿回来了" |
| **颠覆认知** | "你以为X，其实Y" | "你以为在跟医生聊天，其实在帮保司写拒赔理由" |
| **对比反差** | 两件事放一起 | "合同里最短的一句，拒赔时最好用的一句" |
| **代入视角** | 第一人称，让读者觉得"说的就是我" | "卖了5年医疗险，我从没认真看过这4个字" |
| **反常识/损失** | 做错一步会损失什么 | "出险先别急着报案——做错一步，理赔少一半" |
| **情绪击中** | 直接打到最无力的时刻 | "保险人最无力的时刻：你讲的都对，但客户不听" |
| **对话开头式** | 用一句对话拉进冲突现场 | "保险说一分不赔。我问他：挂号等于确诊吗？" |
| **递进恐惧** | 小事→被忽略→严重后果 | "体检高了一点点，没当回事，客户被拒赔了" |

**标题规范**：标题（含标点）≤ 20 字

**标题自查：**
- 有没有包含搜索关键词？（代理人/理赔/拒赔/核保/条款等）
- 是否 ≤ 20 字（含标点）？（超过就删减）
- 正文有没有兑现标题承诺？

### 文案风格

根据内容类型选择：

#### 个人 IP 风格（真人分享）
- 流水账式真实感 + 具体时间细节 + 口语化表达
- 300-350字，8-10个标签（≤ 10 个）
- 当 content-brief 存在时，按其中的核心观点和角度约束语气和结构

#### 方法论风格（干货总结）
- 结构化拆解 + 干货密度高
- 适合转载公众号文章

**一次产出两样东西：**
1. 小红书轮播图 HTML（基于种子模板，含一键下载工具栏）
2. 小红书发布文案（标题 + 正文 + 标签）

## 阶段六：QA 检查（交付前关卡）

轮播图 HTML + 发布文案生成完成后，执行 `references/xhs-qa-checklist.md` 的四类检查：

**① 内容侧** — 卡片序列合理性 + 信息准确性 + 与 content-brief 一致性
**② 视觉侧** — 风格模式遵守 + 风格多样性 + 视觉规范
**③ 格式侧** — 种子模板路径 / CDN / 文件路径
**④ 发布文案** — 标题 ≤ 20 字 / 文案 300-350 字 / 标签 ≤ 10 个

**QA 输出：**

```
内容侧：✅ 通过（序列合理 X/X · 信息准确 X/X）
视觉侧：✅ 通过（风格遵守 · 多样性 OK）
格式侧：✅ 通过
文案：   ✅ 通过（标题 X 字 · 文案 X 字 · 标签 X 个）
总体：✅ QA 通过 → 交付
```

**通过标准**：内容侧 + 格式侧 + 文案全部 ✅。视觉侧 ⚠️ 可选修。任一 ❌ → 修复后重审。

## 故障处理

| 问题 | 处理 |
|------|------|
| 微信抓取失败 | 提示用户手动复制文章正文 |
| 文章太短（<500字） | 压缩到 5-6 张卡片 |
| 文章太长（>5000字） | 精炼核心，控制 10 张以内 |
| 风格选择困难 | **默认 Slide**（最通用清晰） |
| Editorial 主题选择困难 | 默认 `ink-classic`（最百搭） |
| Swiss 强调色选择困难 | 默认 `ikb`（最专业感） |

## 收尾阶段：交付后闭环

交付后执行以下步骤（每次必须执行，但不强制全部有数据）：

### 1. 更新选题卡

在选题卡中将本次对应的选题状态更新：
- 如果是公众号文章衍生 → 更新状态为"已转小红书"
- 如果是独立选题 → 更新状态为"已发布"

### 2. 沉淀表达资产

如果文案中有特别出彩的标题/开头/结尾/金句，写入 `表达资产库.md` 对应分类。

### 3. 更新内容复盘库

如果已发布且有数据，更新 `内容复盘库.md` 对应日期行。

### 4. 反哺知识库

- 新的判断标准 → `02-Areas/知识卡片/`
- 读者评论中出现的真实问题 → 考虑入库为素材或选题
- 有效的题材 → 同类多产
