# 视觉结构映射 — 内容结构 → 轮播图卡片序列

> 参考 guizang-material-illustration 的 7 种视觉结构，将内容的内在逻辑映射为小红书轮播图的卡片序列。
> 在设计阶段第 2 步（选风格模式）和第 3 步（规划卡片）之间执行：先分析内容属于哪种结构，再匹配卡片序列，最后用风格模式包装。

---

## 一、7 种视觉结构

### 1. 循环型（Cycle）

**适合内容**：反馈回路、习惯养成、复利效应、持续改进、飞轮增长

**结构特征**：A→B→C→A（无起点终点，或首尾相连）

**推荐卡片序列**（7 张）：

| 序号 | 卡片角色 | 内容 |
|------|---------|------|
| 1 | 封面 | 标题 + 冲突场景（"为什么总是重复同一个问题"） |
| 2 | 现状痛点 | 当前卡在哪里 |
| 3 | 循环入口 | 启动循环的关键动作 |
| 4 | 循环步骤 1 | 第一步做了什么 |
| 5 | 循环步骤 2 | 第二步如何反馈 |
| 6 | 结果/复利 | 循环后的效果 |
| 7 | CTA | 从哪开始参与循环 |

### 2. 流水线型（Pipeline）

**适合内容**：操作教程、工作流、理赔流程、步骤指南、工序拆解

**结构特征**：输入 → 步骤1 → 步骤2 → 步骤3 → 输出（有明确起点终点）

**推荐卡片序列**（7 张）：

| 序号 | 卡片角色 | 内容 |
|------|---------|------|
| 1 | 封面 | 标题 + 最终效果 |
| 2 | 输入/准备 | 开始前需要什么 |
| 3 | 步骤 1 | 第一步 → 关键操作 |
| 4 | 步骤 2 | 第二步 → 关键操作 |
| 5 | 步骤 3 | 第三步 → 关键操作 |
| 6 | 输出/结果 | 流程走完得到什么 |
| 7 | CTA/清单 | 核心要点快速回顾 |

### 3. 中心辐射型（Hub-and-Spoke）

**适合内容**：产品对比、多因素分析、一个核心概念展开、能力模型

**结构特征**：一个中心节点连接多个分支，分支间并列

**推荐卡片序列**（8 张）：

| 序号 | 卡片角色 | 内容 |
|------|---------|------|
| 1 | 封面 | 标题 + 中心概念 |
| 2 | 中心概念 | 核心定理/原则/框架一览 |
| 3 | 分支 1 | 第一维度展开 |
| 4 | 分支 2 | 第二维度展开 |
| 5 | 分支 3 | 第三维度展开 |
| 6 | 分支对比/权重 | 各分支的关系或优先级 |
| 7 | 整合总结 | 中心概念如何串联 |
| 8 | CTA | 从哪开始应用 |

### 4. 前后对比型（Before/After）

**适合内容**：理赔翻盘、认知升级、状态转变、效果验证、蜕变故事

**结构特征**：A 状态 → 转变过程 → B 状态

**推荐卡片序列**（7 张）：

| 序号 | 卡片角色 | 内容 |
|------|---------|------|
| 1 | 封面 | 标题 + 冲突（"以为没救了，结果…"） |
| 2 | Before | 转变前的痛点/困境/错误认知 |
| 3 | 转折点 | 发生了什么导致转变 |
| 4 | 转变过程 | 具体做了什么（1-2个关键动作） |
| 5 | After | 转变后的效果/成果 |
| 6 | 方法总结 | 可复用的规律或步骤 |
| 7 | CTA | 你也可以做到 |

### 5. 分层堆叠型（Layer Stack）

**适合内容**：知识体系、架构拆解、能力层级、从入门到精通、系统设计

**结构特征**：底层→中层→顶层，层层依赖

**推荐卡片序列**（7 张）：

| 序号 | 卡片角色 | 内容 |
|------|---------|------|
| 1 | 封面 | 标题 + 全景图 |
| 2 | 底层/基础 | 地基是什么 |
| 3 | 中层/应用 | 在此基础上建什么 |
| 4 | 顶层/高阶 | 最终达到什么 |
| 5 | 层级关系 | 各层如何连接 |
| 6 | 路径建议 | 从哪层开始切入 |
| 7 | CTA | 下一步行动 |

### 6. 数据场景型（Data-First）

**适合内容**：行业报告、趋势分析、数据对比、排行榜、数据盘点

**结构特征**：以数据/数字为核心叙事驱动

**推荐卡片序列**（7 张）：

| 序号 | 卡片角色 | 内容 |
|------|---------|------|
| 1 | 封面 | 标题 + 最冲击的数据 |
| 2 | 数据 1 | 核心发现 1（数字+图示） |
| 3 | 数据 2 | 核心发现 2（数字+图示） |
| 4 | 数据 3 | 核心发现 3（数字+图示） |
| 5 | 对比/趋势 | 数据之间的关系或变化 |
| 6 | 洞察结论 | 数据背后的业务洞察 |
| 7 | CTA | 这个数据对你意味着什么 |

### 7. 机制解析型（Scientific / Mechanism）

**适合内容**：保险条款拆解、法律机制、产品原理、规则解读、底层逻辑

**结构特征**：问题 → 原理 → 各部分 → 如何运作 → 结论

**推荐卡片序列**（8 张）：

| 序号 | 卡片角色 | 内容 |
|------|---------|------|
| 1 | 封面 | 标题 + 核心问题 |
| 2 | 问题场景 | 具体场景引入 |
| 3 | 原理 1 | 核心机制的第一个层面 |
| 4 | 原理 2 | 核心机制的第二个层面 |
| 5 | 各部分关系 | 要素如何互相关联 |
| 6 | 常见误解 | 容易被理解错的地方 |
| 7 | 正确做法 | 基于机制的正确应对 |
| 8 | CTA | 记住核心判断 |

## 二、选择流程

在设计阶段，第 2 步（选风格模式）之后、第 3 步（规划卡片）之前执行：

**① 分析内容 → 匹配视觉结构**

回答以下问题来判断：

| 问题 | 对应结构 |
|------|---------|
| 内容讲的是一个"从无到有"或"从坏到好"的转变？ | **Before/After** |
| 内容是一组操作步骤或流程？ | **Pipeline** |
| 内容围绕一个核心概念展开多个方面？ | **Hub-and-Spoke** |
| 内容用数据作为主要叙事手段？ | **Data-First** |
| 内容讲的是持续迭代或反复发生的现象？ | **Cycle** |
| 内容拆解一个机制或规则？ | **Scientific** |
| 内容展示一个体系的多个层级？ | **Layer Stack** |

**② 用匹配的结构卡片序列替换默认序列**

- 不再直接用"默认 7 页"固定模板
- 而是按视觉结构的推荐序列排卡
- 如果素材不足以填满所有卡片 → 合并或删减，不强求凑满
- 如果素材丰富 → 在推荐序列基础上扩展（如步骤型从 3 步扩到 5 步）

**③ 再按风格模式（Slide/Editorial/Swiss）包装卡片**

选定视觉结构后，用所选风格模式的组件和间距做视觉包装。Slide 模式用其组件系统（card-box / point-row / big-num），Editorial 用其 recipe，Swiss 用其网格。

---

## 三、风格模式 × 视觉结构 适配性

| 视觉结构 | 推荐风格模式 | 原因 |
|---------|-------------|------|
| Cycle | Slide | 节奏感强，需要清晰的"步骤"视觉组件 |
| Pipeline | Slide / Swiss | 步骤明确，Slide 的 point-row 或 Swiss 的网格都适合 |
| Hub-and-Spoke | Slide | 需要并列展示多个分支，card-box 组件最合适 |
| Before/After | Slide / Editorial | 对比需要视觉冲击（Slide 暗底）或叙事感（Editorial 金句） |
| Layer Stack | Slide / Swiss | 层级感需要清晰的视觉分隔，Swiss 网格最精确 |
| Data-First | Swiss / Slide | 数字需要精确展示，Swiss 的 num-mega + 网格最佳 |
| Scientific | Editorial / Slide | 拆解机制需要深度阅读感，Editorial 最合适 |
