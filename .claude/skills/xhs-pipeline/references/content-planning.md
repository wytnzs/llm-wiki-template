# Content Planning — 内容规划

> 从素材到页面拆分的指导

---

## 封面钩子

封面是一组卡片里唯一决定"会不会被点开"的页面。需要同时传递：

1. **内容信号**：读者一眼知道这篇讲什么
2. **情绪信号**：为什么要看
3. **标签信号**：关键词方便截流搜索

### 封面公式

```
[结果/态度] + [关键词] + [可信锚点]
```

例：
- "理赔被拒 3 次，我靠一张挂号单翻案"（结果 + 关键词 + 具体数字）
- "保险合同最短的一句，理赔时最好用的一句"（对比 + 关键词）

### 封面在两种模式下的呈现

- **Editorial**：大衬线标题 + 引语 + 图片区域 + issue-row 元数据
  - 可选 M16 纯图片封面（图片占主导，标题叠放）
- **Swiss**：大无衬线 statement + t-cat + hr-accent + lead
  - 可选 S08 Image Hero（图片 + 叠加标题块 + 底部 3 个数据）

## 页面拆分框架

标准小红书优先一套 6-8 张卡片。内容特别扎实、每页都有独立价值时，再扩展到 9-10 张。

### 第 1 张 · 封面
- 传递核心态度或结论
- 让读者产生"这跟我有关"的认知

### 第 2 张 · 先看结果
- 最抓人的数据或结论前置
- 用 big number / 大字号镇场

### 第 3-4 张 · 概念/背景
- 读者需要知道什么才能理解后续？
- 每张只讲一个概念

### 第 5-7 张 · 实操/流程/对比
- 核心干货区
- 步骤、选择对比、关键参数
- 50% 以上的价值感来自这个区域

### 第 8 张 · 亮点/总结
- 卡片形式的总结
- 要点网格、清单

### 可选第 9 张 · 金句/方法论
- 一句能让人记住的话
- 适合单独做一张深色页（Editorial）或一句话 card（Swiss）

### 最后一张 · CTA
- 评论区互动，不直接放微信号
- 一句引导语
- 给读者一个可回复的问题或可收藏的下一步

## 文案压缩

从正文到卡片文案的压缩路径：

1. **提取关键句**：每段第一句通常包含段落核心
2. **去修饰**：删副词、程度词、比喻
3. **转短句**：长句拆成 15 字以内的短句
4. **大字号原则**：页面上最大的 1-2 行字 = 全页的核心信息
5. **最低字数**：一张卡片正文不超过 80 字（含标题）

### 压缩程度速查

| 原始字数 | 压缩目标 | 可容纳卡片数 |
|---------|---------|------------|
| 500-1000 | 每条 10-30 字 | 6-8 张 |
| 1000-3000 | 每条 10-20 字 | 6-8 张 |
| 3000+ | 精选 5-7 个核心点 | 7-10 张 |
