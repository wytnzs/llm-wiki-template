# 小红书配图 QA 检查清单

> 轮播图生成完成后、交付前执行。三类检查：**内容侧 → 视觉侧 → 格式侧**。
> 任何一项不通过，对应卡片必须重新生成。


## 一、内容侧检查

### 1.1 卡片序列合理性

- [ ] 封面包含冲突/钩子，能让人停下来（不只是标题加装饰）
- [ ] 每张卡片只讲一个判断（小红书 3:4 瞬时传播原则）
- [ ] 全部卡片串联起来能讲完一个完整观点
- [ ] 没有"凑数卡"——如果某张卡片没有独立信息量，删掉
- [ ] CTA 页不出现微信号，用评论区互动替代

### 1.2 信息准确性

- [ ] 卡片的数字/数据与原文一致
- [ ] 金句 ≤ 20 字，且是原文的核心判断句
- [ ] 概念/术语使用准确，没有发明叫法
- [ ] 对比卡片的左右/前后方向正确

### 1.3 与上游交接的一致性

- [ ] 如果存在 `.claude/drafts/content-brief.json`，轮播图覆盖了其中的核心观点和视觉化要素
- [ ] 轮播图的中心论点与 content-brief 一致


## 二、视觉侧检查

### 2.1 风格模式遵守

| 规则 | Slide | Editorial | Swiss |
|------|-------|-----------|-------|
| 字体 | 纯无衬线 Noto Sans SC | 衬线标题+无衬线正文 | 纯无衬线 Inter+Noto Sans SC |
| 第 3/7 张暗底 | ✅ 必须 | — | — |
| grain 纹理 | — | ✅ 每张必须有 | ❌ 禁止 |
| mag-bg | — | ✅ 封面+金句页 | ❌ 禁止 |
| 圆角 | card-box 10px | 图片 6px | ❌ 禁止 |
| emoji | ❌ 禁止 | ❌ 禁止（用 Lucide） | ❌ 禁止（用 Lucide） |

- [ ] 所选风格模式的视觉规则全部遵守
- [ ] 没有混用不同模式的组件（不用 Slide 的 card-box 套 Editorial 的 grain）

### 2.2 风格多样性

- [ ] 相邻两张卡片不是同一底色（Slide 模式暖底/dark 交替）
- [ ] Slide 模式第 3 张和第 7 张是暗底
- [ ] Editorial 模式封面和金句页有 mag-bg
- [ ] 全篇没有连续 3 张同配色卡片

### 2.3 视觉规范

- [ ] 没有方格背景
- [ ] 没有卡片倾斜（transform: rotate ❌）
- [ ] 没有 hover 微动动画（transition: none ❌）
- [ ] 没有 emoji（改用 Lucide 图标）
- [ ] 内容填充约 55% 画布高度
- [ ] 内容不溢出卡片边界

### 2.4 材质解释图规则（若使用）

- [ ] 图内只放 3-5 个短中文标签（2-5 字为佳）
- [ ] 没有整段文字塞进图里
- [ ] 图解释了一个关系，不只是装饰
- [ ] 数据图的数值准确


## 三、格式侧检查

- [ ] 从种子模板开始，不是从零写 HTML
- [ ] 所有内容填入 `.poster.xhs` 内
- [ ] `.toolbar` 下载栏完整（html2canvas + JSZip + FileSaver）
- [ ] 文件路径正确：`小红书/{年份}/{月份}/{主题}-小红书版.html`
- [ ] 文件路径正确：`小红书/{年份}/{月份}/{主题}-小红书文案.md`

## 四、发布文案检查

- [ ] 标题 ≤ 20 字，含核心搜索关键词
- [ ] 标题包含钩子结构（结果前置/颠覆认知/对比反差等）
- [ ] 正文文案 300-350 字
- [ ] 标签 ≤ 10 个，覆盖核心关键词
- [ ] 标题不是标题党——正文兑现了标题承诺


## 五、执行流程

```
轮播图 + 文案生成完成
  ↓
逐张过 内容侧（1.1 → 1.2 → 1.3）
  ↓ 不通过 → 修改后重审
  ↓
整体扫 视觉侧（2.1 → 2.2 → 2.3 → 2.4）
  ↓ 不通过 → 调整后重审
  ↓
逐项核 格式侧（3）+ 文案检查（4）
  ↓ 不通过 → 修复后重审
  ↓ 全部通过
✅ 小红书 QA 通过 → 交付
```

**通过标准**：

| 等级 | 条件 | 行为 |
|------|------|------|
| ✅ 通过 | 内容侧全部 ✅ + 格式侧全部 ✅ | 交付 |
| ⚠️ 有建议 | 内容侧全部 ✅ + 格式侧全部 ✅ + 视觉侧有"注意" | 可选修，不阻塞交付 |
| ❌ 不通过 | 内容侧或格式侧有 ❌ | 修复后重新 QA |
