# 知识库协作规则

基于 LLM Wiki 生产管线 + GBrain 长期记忆思想 + OKF 标准文件格式 + Zettelkasten 卡片笔记法。详细工作流见 `.claude/skills/`。

本库的定位不是普通资料夹，而是“AI 知识资产生产系统”：输入进入 `00-Inbox/`，经 Claude Code 提炼为结构化资产，再服务项目、内容生产和复盘反哺。

## 核心能力

**知识库管理**：记灵感 | 整理收件箱 | 查概念 | 提炼卡片 | 更新主题地图 | 更新知识地图
**内容生产**：写公众号 | 转小红书 | 短视频脚本 | 选题 | 风格简报 | 排版配图 | 朋友圈 | 保单整理

> 知识库管理触发词：`记一下` `整理收件箱` `提炼知识卡片` `更新主题地图` `更新知识地图`
> 内容生产触发词：`出稿` `写公众号` `转小红书` `选题` `风格简报` `审美风格` `排版` `配图` `短视频脚本` `朋友圈` `保单整理`

## 跨项目协同

- **知识地图维护**：新增/修改内容后说"更新知识地图"，自动扫描全库刷新根目录知识库地图.md
- **主题地图维护**：新增知识卡片、判例、选题或复盘后，说"更新主题地图"，把新资产挂接到 `02-Areas/主题聚合/`
- **得到大脑/笔记导入**：导出Markdown→放00-Inbox/→我说"提炼知识卡片"处理
- **自媒体运营 → 知识库**：公众号/小红书文章发布后，提炼核心知识点成卡片写入02-Areas/知识卡片/，然后更新知识地图
- **知识库 → 自媒体运营**：写文章/脚本时，生产 skill 自动读取 02-Areas/知识卡片/个人IP/ 获取风格约束 + 检索相关方法论卡片作为认知引擎
- **记灵感**：任意目录下"记一下：XXX"统一写入 00-Inbox/灵感随手记/

## 原则

原子化 | 双向链接 | 概念为本 | 你存我整 | 输出反哺

## 架构分层

知识库四层分工——每层只做一件事，不越界：

| 层 | 位置 | 职责 | 不做什么 |
|----|------|------|---------|
| 证据层 | `02-Areas/判例库/` | 存事实、裁判观点、攻击角度、证据 | 不写情绪、标题、传播包装 |
| 解释层 | `02-Areas/知识卡片/` | 存框架、判断标准、方法论 | 不存单案事实 |
| 表达约束层 | `02-Areas/知识卡片/个人IP/` | 决定立场、口吻、CTA风格、禁忌 | 不提供事实 |
| 人用技能层 | `05-Skills/` | 存给人看的操作说明、检查清单、模板 | 不替代 Claude Code skill |
| 生产层 | `.claude/skills/` | 存 Claude Code 可执行流程、规则、检查清单 | 不堆判例、观点、素材 |

**一句话：判例库=证据 / 知识卡片=解释 / 主题地图=长期记忆 / IP=表达 / Skill=流程。中间用 content-brief 管"讲什么"，用 style-brief 管"怎么讲和长什么样"，文章 YAML 记录调用来源。**

## 禁止事项

1. **不要把判例库写成文章库**——情绪、标题、包装不进判例文件
2. **不要让 skill 变成资料库**——避免"数据库里一套，skill 里又一套"
3. **不要跳过来源记录**——每篇内容头部 YAML 记录 `调用卡片` / `调用判例`
4. **不要把个案结论写成普遍承诺**——保留法律边界，用"这个案子说明"不用"一定能赔"
