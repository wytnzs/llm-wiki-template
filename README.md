# AI 知识资产生产系统

> 基于 LLM Wiki 生产管线、GBrain 长期记忆思想、Gemini OKF 标准文件格式、Zettelkasten 卡片笔记法和 Claude Code skill 的知识库模板。
> 目标是把零散资料变成可复用、可调用、可生产内容的知识资产。

---

## 这是什么

一套可复用的知识资产生产系统，核心是**五步闭环**：

```
输入 → 提炼 → 连接 → 输出 → 复盘
```

每一步只做一件事：
- **输入**：原始灵感、资料、案例先进入 `00-Inbox/`
- **提炼**：把材料拆成知识卡片、判例条目、选题卡
- **连接**：通过主题地图、双向链接和 YAML 元数据形成长期记忆
- **输出**：服务项目、公众号、小红书、短视频、直播和决策
- **复盘**：把真实反馈写回卡片、选题库和主题地图

## 本仓库不包含

公开到 GitHub 时，本仓库只上传系统设计、方法论、模板、说明和必要脚本，不上传真实业务数据。

- ❌ 判例数据和判例视图文件
- ❌ 个人知识卡片正文
- ❌ 保险资料、原始素材
- ❌ 自媒体产出成品
- ❌ 客户、案件、账号、私域相关信息

## 核心设计

### 五层分工

| 层 | 位置 | 职责 |
|----|------|------|
| 证据层 | 判例库 | 存事实、裁判观点、攻击角度 |
| 解释层 | 知识卡片 | 存框架、判断标准、方法论 |
| 长期记忆层 | 主题聚合 | 围绕问题聚合卡片、案例、项目经验 |
| 表达约束层 | 个人IP卡片 | 决定立场、口吻、CTA |
| 生产层 | skills | 存流程、规则、检查清单 |

### content-brief + style-brief 交接机制

所有内容生产必经两步：

```
物料 → content-brief → style-brief → AI skill → 平台内容 → 复盘反哺
```

- `content-brief`：决定讲什么、凭什么讲、调用哪些知识资产
- `style-brief`：决定谁在讲、怎么讲、用什么视觉路线、需要什么图片

## 使用方式

本系统优先为 Claude Code 设计。安装 Claude Code 后，在项目根目录运行：

```
cd path/to/your-knowledge-base
claude
```

触发词：
- `记一下：XXX` → 记灵感
- `整理收件箱` → 分类提炼
- `提炼知识卡片` → 从素材提取概念
- `出稿` / `写公众号` → 内容生产全流程
