# LLM Wiki - AI 知识资产系统

> 用 Markdown 保存知识，用 Claude Code 完成整理、连接、调用和复盘。

这不是一套必须照搬的文件夹，而是一套最小、可扩展的知识协作标准。

## 核心原则

只统一必要内容：

- `README.md` 是唯一必需入口，说明知识库目的、主要区域和使用方式。
- 知识文件使用 UTF-8 Markdown，保留人的可读性和工具可迁移性。
- `title` 和 `type` 是知识文件的最小识别字段，其他字段按需扩展。
- 目录名称是推荐方案，不是强制规范；已有知识库可以保留原结构。
- 案例库、知识库地图、选题库和内容生产都是可选模块。
- Claude Code 只安装用户明确需要的模块，不读取或上传私人资料。

## 推荐结构

```text
你的知识库/
├── README.md          使用入口
├── 00-Inbox/          尚未处理的输入
├── 01-Projects/       当前正在推进的事情
├── 02-Areas/          可长期复用的知识资产
├── 03-Resources/      外部参考资料
├── 04-Archive/        已完成或暂时不用的内容
└── .claude/skills/    Claude Code 执行流程
```

你可以改名、合并或增加目录。只要在 README 中说明每个区域的职责，Claude Code 就能按实际结构运行。

## 可选模块

| 模块 | 适合谁 | 默认安装 |
|---|---|---|
| 知识库地图 | 文件较多、需要统计和健康检查的用户 | 否，安装前询问 |
| 案例库 | 法律、保险、咨询、研究等案例密集场景 | 否，安装前询问 |
| 选题库 | 需要持续生产文章、视频或课程的用户 | 否，安装前询问 |

选题库当前用于人工选择和 Claude Code 辅助整理，不宣称自动从知识卡片生成选题。

## 快速开始

把下面的指令和本仓库地址一起发给 Claude Code：

```text
请阅读本仓库的 README.md 和 AGENT_SETUP.md，帮我创建或改造一套个人知识库。

开始前先询问：
1. 我是新建知识库，还是整理已有知识库；
2. 我的主要输入、使用场景和输出目标；
3. 是否采用推荐目录，还是保留现有目录；
4. 是否需要知识库地图、案例库、选题库。

未经我确认，不安装可选模块，不移动已有资料，不上传任何私人内容。
```

也可以直接运行 Windows 初始化脚本：

```powershell
.\scripts\init-knowledge-base.ps1 -TargetDir "D:\你的知识库"
```

脚本会逐项询问可选模块，不会默认安装选题库。

## 日常使用

```text
记一下：……

更新知识地图（仅安装知识库地图后可用）
```
