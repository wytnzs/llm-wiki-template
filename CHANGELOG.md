# Changelog

## 2026-07-20 - v1.4.0 统一来源层

- 将 `03-Resources/` 从单一外部资源区升级为统一来源层，区分外部资料、个人原始输入和系统历史
- 不再把 `04-Archive/` 作为推荐的活跃内容角色；处理状态改由导读、标签和连接表达
- 同步更新README、初始化规则、协作Skill、主题维护、知识库地图、维护手册和模板
- 公开模板只发布规则、空白模板、脱敏示例和通用脚本，不包含迁移后的私人资料

## 2026-07-17 - v1.3.0 统一主题中心

- 将主题、聚合、自然语言索引和主题页统一为一个“主题中心”模块
- 主题中心 README 负责先定位，独立主题页负责综合判断，双链负责关系扩展
- 初始化脚本改为生成 `02-Areas/主题中心/README.md` 和 `主题页/` 模板
- 增加主题中心模板与示例，不再创建“主题聚合”和“关系导航”两套入口
- 知识库地图继续作为可选的系统状态模块，不与主题中心混用

## 2026-07-17 - v1.2.0 简化关系、运作系统与治理

- 统一主题、聚合、主题页、关系导航和知识库地图的含义
- 明确目录只表达内容角色，不为每一种关系新增目录
- 日常关系收敛为来源、主题问题和真实项目使用
- 知识卡片与主题页模板增加来源粒度、核心问题和证据边界
- 增加 AI 待确认队列与知识库变更日志模板
- 主题关系批量变更增加干跑、抽查、断链和双向一致性检查
- 明确原始资料只读，案例、法律和医疗关键结论需要人工确认
- 新增知识库运作系统说明，统一输入、资源、资产、主题、项目和反馈的流转关系
- 明确主题、聚合、标签、双链、主题中心和知识库地图的职责边界
- 新增关系三问法、自然语言索引下钻顺序和批量关系质量标准
- 增加 AI 修改权限、协作边界和 GitHub 公开安全边界
- 新增每周、双周、月度维护与审计运行单
- 初始化结构将旧的“主题聚合”目录统一为“主题中心/主题页”，并增加主题中心入口模板
- 同步更新知识库协作 Skill、主题维护 Skill、知识库地图模块和初始化模板

## 2026-07-13 - v1.1.0 最小标准

- README 成为唯一必需入口，不默认创建 index.md
- 目录由强制结构改为推荐角色映射
- 知识库地图、案例库和选题库改为按需安装
- 选题库安装前必须征求用户意见，不宣称自动生成正式选题
- 文档从八份整合为标准、操作指南和人群适配三份
- 初始化脚本会复制模板和 Skill，并支持可选模块
- 知识文件统一以 `title` 和 `type` 为最小字段，优先补充 `description`

## 2026-07-09 - v1.0.0 初始发布

- 首次发布 LLM Wiki 知识资产生产系统模板
- 包含目录建议、模板、公开版 Skill、示例和初始化脚本
