# AI 知识库构建操作指南

## 为什么要建 AI 知识库

### 数据档案 vs 知识资产系统

| 对比项 | 数据档案 | 知识资产系统 |
|--------|---------|-------------|
| 典型形态 | 网盘文件夹 / 收藏夹 / 笔记堆积 | 结构化卡片 + 索引 + 生产管线 |
| 使用场景 | "我记得存过，但找不到了" | "我要写这个话题，直接调出相关卡片" |
| AI 参与 | 聊天时问一次性的问题 | AI 可读写文件，形成长期记忆 |
| 复用性 | 每篇文章从头查资料 | 每次调用已有知识资产 |
| 积累效应 | 越存越乱 | 越用越厚 |

**关键区别：** 数据档案是为了"放着"，知识资产系统是为了"用起来"。

### AI 为什么要读你的知识库

- 没有知识库的 AI：每次会话从零理解你的领域，输出靠通用知识
- 有知识库的 AI：读取你写好的卡片、判例、主题聚合，输出贴合的认知框架
- 有生产流程的 AI：能帮你从收件箱提炼卡片、从卡片写文稿、从文稿回流新认知

## 技术栈

| 组件 | 用途 | 理由 |
|------|------|------|
| **Obsidian** | 人类读写界面 | 本地优先、Markdown 原生、双向链接、图谱、免费 |
| **Markdown** | 存储格式 | AI 最易读的格式之一，Git 友好，无需专用软件打开 |
| **Claude Code** | AI 操作界面 | 直接读写文件系统，执行 skill，自动化流程 |
| **YAML** | 元数据标准化 | AI 可扫描解析，提供结构化上下文 |
| **Git** | 版本管理 | 回滚、备份、历史追踪 |

## 知识资产分类

系统把知识资产分为七类，每类有明确的格式和存放位置：

| 类型 | 含义 | 存放位置 | 典型问题 |
|------|------|---------|---------|
| **概念** | 一个独立概念的定义和关键要点 | 知识卡片 | "这是说什么？" |
| **关系** | 概念之间的连接和对比 | 知识卡片（含链接） | "A 和 B 什么关系？" |
| **判断** | 做决策的标准和框架 | 知识卡片 | "怎么办？依什么判断？" |
| **方法** | 可复用的操作流程 | 知识卡片 / Skills | "怎么做？" |
| **行动** | 具体的待办和行动项 | Projects | "现在做什么？" |
| **证据** | 事实、数据、判例 | 判例库/Resources | "依据是什么？" |
| **表达** | 立场、口吻、风格约束 | 知识卡片（个人IP下） | "怎么说？什么不能说？" |

这七类覆盖了从"是什么"到"怎么做"再到"怎么说"的完整认知链路。

## 11 步从零构建指南

### 第一步：安装必要工具

```text
1. 安装 Obsidian — https://obsidian.md/
2. 安装 Git — https://git-scm.com/
3. 安装 Claude Code — npm install -g @anthropic-ai/claude-code
4. 创建一个空目录作为知识库根目录
5. 用 Obsidian 打开该目录作为本地仓库
```

### 第二步：建立目录结构

```bash
# 创建核心目录
mkdir -p 00-Inbox
mkdir -p 01-Projects
mkdir -p 02-Areas/知识卡片
mkdir -p 02-Areas/主题聚合
mkdir -p 03-Resources
mkdir -p 04-Archive
mkdir -p 05-Skills
mkdir -p 06-选题库/cards
mkdir -p .claude/skills
```

（或直接下载模板仓库，见下文初始化命令）

### 第三步：配置知识库地图

创建 `知识库地图.md` 作为全库首页：

```markdown
# 知识库地图

## 快速入口
- [[00-Inbox/|收件箱]] — 待处理原材料
- [[02-Areas/知识卡片/|知识卡片]] — 概念资产
- [[02-Areas/主题聚合/|主题聚合]] — 议题认知
- [[06-选题库/|选题库]] — 内容选题

## 标签索引
#领域1 #领域2 #知识卡片

## 统计信息
- 知识卡片: N 张
- 主题聚合: N 篇
- 进行项目: N 个
```

### 第四步：定义知识卡片格式

```markdown
---
title: 概念名称
type: KNOWLEDGE_CARD
tags: [领域, 知识卡片]
links: [[关联卡片1]], [[关联卡片2]]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---

## 定义

一句话说清楚这个概念是什么。

## 关键要点

- 要点 1
- 要点 2
- 要点 3

## 判断维度

用在什么场景？帮我做什么决策？

## 案例

一个通用示例（非真实数据），说明概念如何应用。

## 来源

- 来源 1
- 来源 2
```

### 第五步：建立主题聚合格式

```markdown
---
title: 关于 X 的当前认知
type: THEME_MAP
tags: [主题聚合, X]
links: [[相关卡片1]], [[相关卡片2]]
created: YYYY-MM-DD
updated: YYYY-MM-DD
status: 活跃 | 待更新 | 已存档
---

## 当前观点

### 子议题 A
- 我认为：...
- 依据：[[相关卡片]] 或 [[判例]]
- 置信度：高/中/低

### 子议题 B
- 我认为：...
- 依据：...
- 置信度：...

## 待验证

- [ ] 待验证观点 1
- [ ] 待验证观点 2

## 近期变化

- YYYY-MM-DD：新增观点 B
- YYYY-MM-DD：修正观点 A（从 X 调整为 Y）
```

### 第六步：建立判例库（可选）

只有专业服务场景需要此步骤。判例库按三视图架构组织：

```text
判例库/
├── 按类型/     ← 核心维度：按案件性质分类
├── 按原因/     ← 辅助维度：按争议焦点分类
└── 按分类/     ← 辅助维度：按领域分类
```

判例卡片格式：

```markdown
---
title: 案件关键词
type: CASE_ENTRY
tags: [判例, 类型, 原因]
links: [[相关卡片1]], [[相关卡片2]]
created: YYYY-MM-DD
---

## 事实摘要

（简述案件事实，不写情绪和立场）

## 争议焦点

- 焦点 1
- 焦点 2

## 裁判观点

- 法院认为：...
- 裁判要旨：...

## 攻击/防御角度

- 从消费者角度：...
- 从专业机构角度：...

## 来源

- 案号 / 链接
```

### 第七步：记录第一批灵感

在 `00-Inbox/` 中创建几条随手记：

```markdown
# YYYY-MM-DD 随手记
- 灵感 1：...
- 灵感 2：...
- 参考链接：...
```

### 第八步：提炼第一批知识卡片

从收件箱中选择 3-5 个最有价值的概念，提炼成知识卡片：

1. 确定概念边界（一张卡片只讲一个概念）
2. 写定义（一句话）
3. 列关键要点
4. 写判断维度
5. 加通用案例
6. 标注双向链接

### 第九步：建立第一个主题聚合

选一个你最常需要输出看法的议题，创建主题聚合。

### 第十步：配置 Claude Code 技能

将常用操作转化为 skill：

```markdown
# skill: knowledge-card-create

从输入文本中提取关键概念，按知识卡片格式输出。
定位：02-Areas/知识卡片/对应领域/
```

### 第十一步：启动第一个项目

在 `01-Projects/` 中创建第一个项目，用已有知识资产驱动产出：

1. 创建项目目录
2. 写项目计划
3. 检索相关卡片和主题聚合
4. 开始产出
5. 复盘并回馈知识库

## 一键初始化命令

以下命令创建完整的模板目录结构，内置示例文件：

```bash
# 从零创建
mkdir my-knowledge-base && cd my-knowledge-base

# 创建目录
mkdir -p 00-Inbox 01-Projects 02-Areas/知识卡片/{通用领域A,通用领域B} 02-Areas/主题聚合 02-Areas/判例库/{按类型,按原因,按分类} 03-Resources/读书笔记 04-Archive 05-Skills 06-选题库/cards .claude/skills

# 创建首页索引
cat > 知识库地图.md << 'EOF'
# 知识库地图

## 快速入口
- [[00-Inbox/|收件箱]]
- [[02-Areas/知识卡片/|知识卡片]]
- [[02-Areas/主题聚合/|主题聚合]]
- [[06-选题库/|选题库]]

## 标签索引

- 主题聚合: 0
- 项目: 0
EOF

# 初始化 Git
git init
git add .
git commit -m "初始化知识库结构"
```

## 最小可行工作流（7 步）

如果你没有时间全面搭建，可以先跑这个最小闭环：

```text
第1步：建目录（5分钟）
    → 复制上面的初始化命令

第2步：记一个灵感（1分钟）
    → 在 00-Inbox/ 写一条随手记

第3步：提炼一张卡片（10分钟）
    → 从那条灵感中提取一个概念写卡片

第4步：写一篇主题聚合（15分钟）
    → 选一个议题，写出当前观点

第5步：做一个选题卡片（5分钟）
    → 在 06-选题库/cards/ 写一个选题

第6步：出一篇内容（30分钟）
    → 用选题+卡片+主题聚合写一篇短文

第7步：复盘回流（5分钟）
    → 内容生产中的新认知更新到知识库
```

7 步走完，你已经跑通了一条完整的知识资产生产线。后面就是持续积累和优化。
