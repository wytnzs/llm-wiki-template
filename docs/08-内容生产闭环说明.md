# 内容生产闭环说明

## 整体流程

```text
                ┌────────────────────────────────────┐
                │           认知引擎                   │
                │  ┌──────────┐ ┌────────┐ ┌──────┐  │
                │  │ 风格约束  │ │知识卡片│ │判例  │  │
                │  │(IP卡片)  │ │(概念)  │ │(证据)│  │
                │  └──────────┘ └────────┘ └──────┘  │
                └────────────────────────────────────┘
                          │ 自动加载
                          ▼
00-Inbox → 知识卡片 → 主题聚合 → 选题卡片 → 内容简报 → 内容生产 → 复盘
  │          ▲            ▲          │          │          │          │
  │          │            │          │          │          │          │
  └──────────┴────────────┴──────────┴──────────┴──────────┴──────────┘
                                                                            │
              ┌───────── 反馈循环 ──────────────────────────────────────────┘
              │
              ▼
          更新卡片 / 新建卡片 / 更新主题聚合 / 更新知识地图
```

---

## 各环节详解

### 1. 收件箱 → 知识卡片

**输入：** 00-Inbox 中的灵感、笔记、剪藏

**加工步骤：**

```text
原始素材 → 提取核心概念 → 确定概念边界 → 写定义
                                      → 列关键要点
                                      → 写判断维度
                                      → 加通用案例
                                      → 标注链接
```

**注意：** 一条原始素材可能提炼出多张卡片。不要试图把所有信息塞进一张卡片。

### 2. 知识卡片 → 主题聚合

**输入：** 多张相关的知识卡片

**加工步骤：**

```text
知识卡片 A ─┐
知识卡片 B ─┼─→ 选择议题 → 整理当前观点 → 写出主题聚合
知识卡片 C ─┘                 ↓
                       基于卡片中的判断维度，
                       结合个人实践经验形成立场
```

**为什么需要这一步：** 知识卡片是相对客观的概念解释，主题聚合是你个人的当前立场。两者分开，方便 AI 理解"什么是公认的"和"我信什么"。

### 3. 主题聚合 → 选题卡片

**输入：** 主题聚合中的议题

**加工步骤：**

```text
主题聚合中的子议题 → 判断是否有内容价值
                    ├─ 有：创建选题卡片（痛点 + 角度 + 素材）
                    └─ 无：暂存待挖掘方向
```

**选题卡片内容：**

```markdown
---
title: 选题：一句话标题
type: TOPIC_CARD
tags: [选题, 领域, 优先级]
links: [[相关卡片]], [[相关主题聚合]]
status: 未开始
priority: high
---

## 选题描述
## 目标受众
## 核心信息
## 素材准备
## 分发建议
```

### 4. 选题卡片 → 内容简报

**输入：** 选题卡片 + 认知引擎自动加载的素材

**内容简报结构：**

```markdown
# 内容简报：[标题]

## 基础信息
- 选题：[[选题卡片]]
- 形式：公众号 | 小红书 | 短视频 | 朋友圈
- 目标受众：...

## 认知引擎加载
### 风格约束
- 来自：[[个人IP卡片A]]、[[个人IP卡片B]]
- 要点：口吻、CTA 风格、禁忌词

### 知识素材
- 概念卡片：[[卡片A]]、[[卡片B]]
- 判例支撑：[[判例X]]

### 主题立场
- [[主题聚合Y]] 中的相关观点

## 内容要求
- 核心信息（3-5 点）
- 禁止词
- 必须引用的来源

## 分发计划
| 平台 | 形式 | 篇幅 | 发布时间 |
|------|------|------|---------|
| 公众号 | 长文 | 2000 字 | 周一 |
| 小红书 | 图文 | 800 字 | 周一 |
```

**内容简报是生产环节唯一的交接文档。** AI 先读简报知道要什么和不能做什么，再开始写正文。

### 5. 内容简报 → 内容生产

**输出形式：**

| 平台 | 形式 | 特点 |
|------|------|------|
| **公众号** | 长图文 | 深度、系统、有观点 |
| **小红书** | 短图文 | 轻量、有用、有共鸣 |
| **短视频** | 脚本 | 口语化、有节奏、有钩子 |

**生产原则：**

- 同一选题，多平台用同一套核心信息，但**形式不同**
- 公众号详述背景和逻辑
- 小红书提炼要点和行动建议
- 短视频设计钩子、节奏、画面

### 6. 内容生产 → 复盘

内容发布后，进行生产复盘：

```text
复盘要点：
├─ 本次生产调用了哪些卡片？够用吗？
├─ 需要新建什么卡片？
├─ 需要更新什么主题聚合？
├─ 有什么新视角/新案例要入库？
├─ 有什么内容生产流程要优化？
```

### 7. 复盘 → 反馈回流

将复盘结果写回知识库：

```text
复盘发现                    →  操作
──────────────────────────────────────────
缺少某个概念卡片            →  新建知识卡片
某个判断维度需要修正         →  更新知识卡片
某个案例要记录              →  更新判例库
对某个议题有新认识          →  更新主题聚合
某个选题角度好但没写到位    →  更新选题卡片
内容有新的质量标准          →  更新 Skill
```

## 认知引擎

认知引擎不是独立的组件，而是一套**内容生产前的自动化加载机制**：

```text
当 AI 被要求生产内容时：
  1. 读取选题卡片 → 知道要写什么
  2. 读取内容简报 → 知道具体要求
  3. 自动加载风格约束
     → 从知识卡片/个人IP/ 读取口吻、禁忌、CTA 风格
  4. 自动加载知识卡片
     → 从选题卡片的 links 中找到相关卡片
  5. 自动加载判例
     → 从知识卡片 links 展开到判例库
  6. 自动加载主题聚合
     → 从选题卡片的 links 中找到相关主题聚合
```

认知引擎确保 AI 在写内容时，**风格、知识、立场、证据**都在上下文里，不需要用户反复提醒。

## 质量检查清单

内容发布前，对照以下检查项：

### 禁止词检查

- [ ] 是否包含绝对化承诺（"一定能"、"保证"）
- [ ] 是否包含情绪化表述（仅在判例库中禁止，内容中视平台规则而定）
- [ ] 是否违反行业合规要求（特定行业有法律法规禁止的表述）

### 声明级别检查

- [ ] 是否为普遍结论？ → 需要有足够的卡片和判例支撑
- [ ] 是否为个案结论？ → 明确标注"本案中" / "此案表明"
- [ ] 是否为推测/观点？ → 标注"我认为"

### 来源追踪

- [ ] 每项主张能否回溯到来源？
- [ ] 知识卡片有来源吗？
- [ ] 涉及的判例是否可查证？

## 完整生产闭环示意

```text
Day 1:   在 00-Inbox 记了一条灵感
Day 2:   提炼成知识卡片 → 放入 02-Areas/知识卡片/
Day 3:   创建主题聚合 → 放入 02-Areas/主题聚合/
Day 4:   从主题聚合提取选题 → 创建选题卡片
Day 5:   生成内容简报 → AI 自动加载认知引擎 → 写正文
Day 6:   多平台适配生产 → 公众号 / 小红书 / 短视频
Day 7:   复盘 → 回流到知识库
       ↳ 新建 1 张卡片
       ↳ 更新 1 篇主题聚合
       ↳ 更新知识地图
```

**这个闭环走完一次，你的知识资产就多了一层厚度。下次写类似内容时，调用成本几乎为零。**
