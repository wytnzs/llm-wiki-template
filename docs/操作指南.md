# 操作指南

## 第一次配置

把仓库地址和下面的内容发给 Claude Code：

```text
请根据仓库标准整理我的知识库。

我的主要输入：
我的主要用途：
我的主要输出：
这是新建知识库 / 已有知识库：

请先询问我是否采用推荐目录，以及是否需要知识库地图、案例库和选题库。
未经确认不要移动已有资料，不要安装可选模块。
```

## 日常最小闭环

### 记录

```text
记一下：今天想到……
```

先保存原始表达，不急着强行制成卡片。

### 整理

```text
整理收件箱。请区分外部资料、个人原始输入、系统历史、可复用知识和当前项目材料；仍需追溯的原始材料进入03来源层。
```

### 提炼

```text
从这份材料中提炼真正可复用的知识。先判断是否值得单独建文件，再按模板生成。
```

### 调用

```text
查一下：……。请先从主题入口定位，再下钻到知识卡片、案例和原始依据，说明来源和适用边界。
```

有主题页时，按下面的顺序下钻：

```text
问题
-> 主题中心：按自然语言问法定位主题
-> 主题页：读取当前判断和边界
-> 知识卡片/案例：定位方法与证据
-> 原始材料：核验正式引用
-> 项目或输出
```

### 复盘

```text
复盘这次结果。判断问题来自输入、知识、目录、模板还是 Skill，并把修正写回源头。
```

## AI 待确认与变更日志

- `00-Inbox/AI待确认.md` 是轻量审核入口，只记录来源缺口、冲突、关系建议和关键判断变更；
- 用户确认后再更新正式知识、双向关系和日期；
- 根目录 `知识库变更日志.md` 只记录影响结构、规则、正式判断或能力的重要变化；
- 普通润色和一次性内容生产不写入变更日志。

没有足够证据时，AI 应保留原文并提出建议，不自动补写来源、事实或普遍结论。

## 已有知识库整理

1. 先扫描，不立即移动；
2. 识别每个区域的实际职责；
3. 找出重复说明、失效路径、临时文件和空目录；
4. 给出保留、合并、进入来源层、可恢复隔离或删除清单；
5. 经用户确认后执行高影响迁移；
6. 更新 README 和 Claude Code 规则；
7. 检查编码、元数据、链接和隐私边界。

## 关系维护

不要按关键词批量互链。新增关系前确认：关系明确、沿过去有用、能够回到依据。只是同领域的内容使用标签即可。

主题页只保存围绕问题的精选判断和证据，不做资料清单。详细规则见[知识关系与索引规则](知识关系与索引规则.md)。

## 定期维护

建议每周检查编码、链接和新文件元数据，每两周复核来源、主题证据和 Inbox 路由，每月用真实问题抽查检索是否有效。完整运行单见[定期维护与审计](定期维护与审计.md)。

## 使用知识库地图

文件数量较多时可以启用知识库地图。它只记录动态状态：数量、项目、健康问题和下一步事项，不重复 README 中的系统说明。
