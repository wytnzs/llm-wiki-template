# 目录结构说明

## 总览

```text
知识库根目录/
├── 00-Inbox/              ← 原材料暂存区
├── 01-Projects/           ← 进行中的项目
├── 02-Areas/              ← 长期知识资产
│   ├── 知识卡片/          原子概念卡片
│   ├── 主题聚合/          专题观点陈述
│   └── 判例库/            事实与裁判观点（仅特定行业需要）
├── 03-Resources/          ← 外部参考材料
├── 04-Archive/            ← 已完成 / 过期内容
├── 05-Skills/             ← 人类可读流程与模板
├── 06-选题库/             ← 内容选题资产
├── .claude/
│   └── skills/            ← Claude Code 可执行技能
└── 知识库地图.md          ← 全库索引首页
```

---

## 各目录职责

### 00-Inbox/ —— 收件箱

**职责：** 灵感随手记、剪藏内容、聊天记录的原始落点。

- 不做分类、不加工、不排版
- 定期清空——提炼完就删或移至其他目录
- 统一文件名格式：`YYYY-MM-DD-简述.md`

### 01-Projects/ —— 项目区

**职责：** 正在做的事。每个子目录一个项目。

- 一个项目可以是一本书的写作、一个课程开发、一个工具开发
- 每个项目有独立的 `项目计划.md` 和 `笔记/` 子目录
- 项目完成后整体移至 `04-Archive/`

### 02-Areas/ —— 长期资产区

这是知识库的核心。分三类资产：

| 子目录 | 内容 | 格式 |
|--------|------|------|
| `知识卡片/` | 原子化概念卡片 | OKF-like YAML + Markdown |
| `主题聚合/` | 围绕一个主题的当前观点 | OKF-like YAML + Markdown |
| `判例库/` | 事实与裁判观点（可选） | 三视图架构 |

知识卡片按领域分子目录：

```text
知识卡片/
├── 领域A/
├── 领域B/
└── 领域C/
```

每个领域目录下是一张张概念卡片。

主题聚合按主题分：

```text
主题聚合/
├── 关于X的当前认知.md
├── 关于Y的当前认知.md
└── 关于Z的当前认知.md
```

判例库按三个维度分类：

```text
判例库/
├── 按类型/
├── 按原因/
└── 按分类/
```

### 03-Resources/ —— 外部参考

**职责：** 读书笔记、课程笔记、工具文档、原始资料。

- 保留原始结构，尽量不做二次加工
- 需要提取的内容通过提炼知识卡片迁入 `02-Areas/知识卡片/`

### 04-Archive/ —— 归档

**职责：** 已完成项目、过期资料、不再维护的笔记。

- 从 `01-Projects/` 移入的项目保持目录结构不变
- 从 `00-Inbox/` 处理完成后的原始笔记也可移入
- 归档不等于删除——只是不再活跃使用

### 05-Skills/ —— 人类技能

**职责：** 人类的操作流程、模板、检查清单。

- 写给人看的，不是写给 AI 的
- 例如：内容生产 SOP、知识卡片编写指南、判例入库流程
- AI 可执行的部分放在 `.claude/skills/`

### 06-选题库/ —— 选题资产

**职责：** 选题卡片 + 全库索引 + 待挖掘方向。

```text
06-选题库/
├── cards/          选题卡片（一张卡片一个选题）
├── 选题总索引.md    所有选题的索引
└── 待挖掘方向.md   未成熟的候选方向
```

### .claude/skills/ —— AI 技能

**职责：** Claude Code 可直接执行的技能文件。

- 用于自动化：提炼卡片、更新知识地图、内容生产
- 每个 skill 有明确的触发词和职责边界
- 格式遵循 Claude Code skill 规范

### 知识库地图.md

**职责：** 全库首页索引。

- 列出所有活跃目录和关键文件
- 每次新增内容后自动更新
- 是人类和 AI 进入系统的第一入口

## 五层管线概念

目录结构本质上是一条生产管线：

```text
[    第一层    ]    [ 第二层 ]    [  第三层  ]    [  第四层  ]    [ 第五层 ]
输入 / 原材料  →   精炼资产  →   项目驱动  →   产出内容  →   复盘反馈
                      │            │              │
                   02-Areas/    01-Projects/    (外部)
                      │            │              │
00-Inbox/          知识卡片     具体项目       公众号/小红书
03-Resources/      主题聚合                    短视频/课程
                   判例库                      咨询交付
```

| 层 | 对应目录 | 做什么 |
|----|---------|--------|
| **输入层** | `00-Inbox/` `03-Resources/` | 收集原材料 |
| **资产层** | `02-Areas/` `06-选题库/` | 精炼成可复用资产 |
| **项目层** | `01-Projects/` | 用资产驱动具体项目 |
| **产出层** | 项目输出（外部） | 产生内容/产品/服务 |
| **反馈层** | 回流到资产层 | 新认知更新资产 |

每一层只做一件事，不越界。这是系统和碎片化文件夹的根本区别。
