# 内容审美与视觉系统设计

## 核心判断

这套系统采用双轨设计：

1. 私有版：服务本人内容生产，追求最高质量，可以保留个人写作人格。
2. 分享版：服务对外分享，只提供方法、选择器、变量和检查清单，不复制个人表达 DNA。

一句话原则：

**审美系统可以共享，写作人格不能共享。**

## 总体架构

```text
知识卡片 / 判例库 / 选题卡
        ↓
content-brief.json
        ↓
style-brief.json
        ↓
公众号 / 小红书 / 短视频 / 配图
        ↓
发布复盘 / 风格回写 / 知识库更新
```

`content-brief` 解决讲什么、凭什么讲。

`style-brief` 解决谁在讲、怎么讲、长什么样、用什么图。

## 母风格

默认母风格：

**克制知识编辑风 + 材质化解释图**

这套风格的目标不是花哨，而是让内容显得可信、清楚、可传播。

### 视觉关键词

- 克制
- 高级
- 专业
- 清晰
- 有判断力
- 图像服务理解
- 版式服务阅读

## 三条内容路线

| 路线 | 适用内容 | 主要平台 | 视觉特点 |
|---|---|---|---|
| 深度编辑风 | 观点、趋势、方法论、案例拆解 | 公众号 | 少色彩、强结构、编号章节、轻标记 |
| 信息密度风 | 教程、工具盘点、操作指南 | 公众号/小红书 | 标签、步骤、卡片、对比块 |
| 材质解释图 | 抽象概念、图表、封面、轮播核心图 | 小红书/公众号配图 | 米白背景、黑色线稿、灰白材质、短中文标签 |

## 私有版和分享版的边界

| 项目 | 私有版 | 分享版 |
|---|---|---|
| 写作人格 | 可以保留 | 必须移除 |
| 语气习惯 | 可以固化 | 改成选择器 |
| 开头方式 | 可以复用偏好 | 只给结构选项 |
| 视觉系统 | 固定母风格 | 提供多风格模板 |
| 图片提示词 | 可带个人偏好 | 给通用骨架 |
| 案例素材 | 可调用知识库 | 不带私有资料 |

## 质量标准

一篇内容进入生产前，必须回答：

1. 这篇内容走哪条视觉路线？
2. 为什么不是另外两条？
3. 公众号和小红书是否需要不同表达？
4. 是否需要中心解释图？
5. 图像是在解释关系，还是只是装饰？
6. 有没有法律、保险、数据边界风险？
7. 如果分享给别人，哪些部分必须抽象成选择器？
