# 知识库健康检查清单

> 灵感来源：GBrain 的"梦循环"（overnight dream cycle）——定期自动养护知识库。
> 建议频率：每两周执行一次完整检查，每周执行一次快速检查。

---

## 一、乱码检查

**目标**：确保所有 Markdown 文件编码正确（UTF-8），无替换字符。

| 检查项 | 方法 | 修复方式 |
|--------|------|---------|
| 搜索替换字符 `�` | 全局搜索 `�` | 找到原始内容恢复，或按上下文重写 |
| 搜索常见乱码串 | 搜索 `锟斤拷`、`鏁版嵁`、`鈫` | 同上 |
| 编码一致性 | 检查新导入文件的编码 | 统一保存为 UTF-8 无 BOM |

**自动检查**：如果已配置脚本，运行编码检查脚本；未配置时，让 Claude Code 执行“乱码检查”。

**安全规则**：乱码文件先备份或隔离，尝试恢复编码；删除必须经用户确认。

---

## 二、断链检查

**目标**：确保知识卡片之间的 `[[双向链接]]` 都有对应的目标文件。

| 检查项 | 方法 | 修复方式 |
|--------|------|---------|
| 搜索 `[[xxx]]` 是否有对应 `.md` 文件 | 提取所有 `[[链接]]`，逐个验证文件存在 | 创建缺失文件，或删除无效链接 |
| 注意跨目录链接 | 有些链接的完整路径可能因目录调整失效 | 更新链接路径或创建重定向 |

**自动检查**：如果已配置脚本，运行断链检查脚本；未配置时，让 Claude Code 执行“断链检查”。

---

## 三、旧路径检查

**目标**：确保所有引用路径与当前目录结构一致。

| 旧路径 | 当前路径 | 检查位置 |
|--------|---------|---------|
| `01-Wiki/` | `02-Areas/知识卡片/` | .claude/、docs/、templates/ |
| `06-Wiki/` | `02-Areas/知识卡片/` | .claude/、docs/ |
| `03-Projects/` | `01-Projects/` | .claude/、.obsidian/ |
| `04判例库/` | `02-Areas/判例库/` | 脚本、CSV 数据文件 |
| `05-选题库/` | `06-选题库/` | skill 文件、选题卡 |
| `delivery-brief` | 已移除（改用 content-brief） | 所有 skill 和 docs；废弃声明可保留 |

**检查方法**：全局搜索以上旧路径关键词，确认只出现在历史记录或废弃声明中。

---

## 四、Inbox 堆积检查

**目标**：防止 00-Inbox/ 变成死仓库。

| 检查项 | 阈值 | 行动 |
|--------|------|------|
| Inbox 文件数量 | > 20 条需要提醒 | 执行"整理收件箱"流程 |
| 单条素材停留时间 | > 2 周未处理 | 判断是否还有价值，或归档 |
| 已处理未归档 | 原始素材处理后是否移入 04-Archive/ | 清理已处理素材 |

---

## 五、卡片与主题的连接检查

**目标**：确保每张知识卡片都连接到至少一个主题地图。

| 检查项 | 方法 | 修复方式 |
|--------|------|---------|
| 卡片是否出现在主题地图中 | 对照主题地图的"可调用卡片"列表和实际卡片目录 | 补充到对应的主题地图 |
| 卡片是否有 `[[链接]]` | 检查没有链接的孤立卡片 | 建立与相关卡片的关联 |
| 主题地图是否覆盖所有卡片领域 | 检查是否有卡片领域没有对应主题地图 | 新建主题地图或扩充已有地图 |

---

## 六、选题与卡片的连接检查

**目标**：确保每张选题卡都标注了可调用的知识卡片。

| 检查项 | 方法 | 修复方式 |
|--------|------|---------|
| 选题卡是否标注了调用卡片 | 查看 `source_cards` 字段 | 补充调用的卡片路径 |
| 选题卡是否标注了判例/案例 | 查看 `source_cases` 字段 | 补充判例案号或案例路径 |
| 选题卡是否关联主题地图 | 查看 `related_theme` 字段 | 补充对应主题地图 |
| 选题卡是否标注内容阶段 | 查看 `content_stage` 字段 | 标注 idea / brief / writing / done |
| 选题卡是否标记了内容方向 | 查看是否有开头钩子、核心观点 | 补充内容方向字段 |
| 选题卡状态是否更新 | 已发布的选题是否标记为"已发布" | 更新选题卡状态 |

---

## 七、成品内容复盘检查

**目标**：确保每次发布都有复盘记录。

| 检查项 | 方法 | 修复方式 |
|--------|------|---------|
| 已发布文章是否有复盘记录 | 对照 `01-Projects/自媒体运营/` 下的文章和 `内容复盘库.md` | 补充复盘记录 |
| 复盘是否包含数据 | 阅读量、点赞、评论等数据 | 补充数据 |
| 复盘是否包含反哺操作 | 是否产出了新卡片、更新了判例标签 | 补充反哺操作记录 |
| AI输出失败是否有记录 | 查看"AI输出失败复盘"表 | 补充失败记录和根因分析 |

---

```
□ 选题与卡片连接检查

□ 更新知识库地图
```
