# 知识关系与标签规范

## 基本分工

- `type` 回答“这份内容是什么”；
- `tags` 回答“它属于哪些稳定类别”；
- `[[双链]]` 回答“它具体关联哪一份知识”；
- 主题地图回答“综合这些资料后，当前判断是什么及其边界”。

这里的“主题”是需要持续回答的问题，“聚合”是筛选资料的动作，“主题地图”是聚合后形成的主题页。根目录知识库地图只展示系统状态，关系导航只负责进入主题页，三者不承担同一功能。

标签只用于稳定筛选，不把临时关键词、完整书名或一句话结论继续扩充成新的规范标签。双链只连接能说明关系的具体文件，不为了消除孤立页而随机互链。

正式新增或复审关系时，按[[05-Skills/知识关系质量评估标准|知识关系质量评估标准]]的三问法判断：关系是否明确、沿过去是否有用、能否回到依据。普通双链不评分；只有批量变更时才增加抽查和结构审计。

同一概念在两种位置出现时，按以下标准区分：

- “这是一份内容成品”“它发布在公众号”“它涉及保险”属于分类，写入 `tags`；
- “这份内容直接讨论保险+AI”“这张卡片支撑某张主题地图的核心判断”属于语义关系，写入双链；
- “生产这份内容时使用了 AI”只是生产方式，不等于内容主题是“保险+AI”；
- “一张保险卡片可以拿来写内容”不等于它属于“内容生产主题地图”。

## 基础标签

| 内容角色 | 至少包含 |
|---|---|
| Inbox 输入 | `输入` |
| 项目材料 | `项目` |
| 知识卡片 | `知识卡片` + 所属领域 |
| 判例条目 | `判例` + `保险理赔` + 险种 + 争议类型 |
| 判例库概览、法条和维权路径 | `判例库` + `保险理赔` |
| 主题地图 | `主题地图` + 主题领域 |
| 内容成品 | `内容成品` + 平台 |
| 外部资源 | `外部资源` + 资源领域 |
| 归档 | `归档` |
| 选题卡 | `选题` |

原有标签不删除；新增规范标签用于形成稳定筛选口径。

## 关系字段

```yaml
related_theme_maps:
  - "[[保险理赔主题地图]]"
related_cards:
  - "[[如实告知义务]]"
related_projects: []
```

日常只先判断三类关系：

- `source`：这份知识从哪里来；
- `related_theme_maps`：它是否直接帮助回答某个主题问题；
- `related_projects`：它是否已经在某个项目中真实使用。

只有确实需要表达两份具体知识之间的推理关系时，才补充 `links.supports`、`links.contrasts` 或 `links.examples`。`links.applies_to`、`links.derived_methods` 等旧字段继续兼容，但不是新卡片的默认字段。

现有卡片中的 `related_topics` 是早期的“相关知识”字段，不等于 `related_theme_maps`。保留旧值用于兼容，不批量改名；以后需要明确关系时，优先在正文或 `supports`、`contrasts`、`examples` 中说明用途。

知识卡片优先维护来源、主题页和真实使用项目；判例优先连接对应的拒赔理由、疾病视图和可复用知识卡片；项目材料只连接实际使用的资产；内容成品连接其主要依据或主题页。没有明确关系时不补空链接。

### 主题关系准入标准

写入 `related_theme_maps` 或主题地图的 `related_cards` 前，至少满足以下一项：

1. 内容直接回答主题地图的核心问题；
2. 内容为主题判断提供证据、方法、反例或明确边界；
3. 内容是该主题下经过验证、需要再次调用的代表性案例。

如果只能说明“可能有关”“属于同一大领域”或“以后可能用到”，保留标签即可，不建立主题关系。主题地图中的 `related_cards` 是精选下钻入口，不追求覆盖全部同领域卡片。

正式主题地图与知识卡片的关系应保持一致：主题地图移除卡片时，同步清理卡片中的对应 `related_theme_maps`；新增时同步补充。Obsidian 会自动生成反向链接，但 YAML 字段仍不得保留已经失效的关系。

## 关系边界

1. Inbox 保留原貌，未经整理的推断不写成正式关系；
2. 外部资源只代表资料来源，不自动视为用户判断；
3. 判例库“一案一主文件”，按拒赔理由和疾病分类的硬链接已经属于关系视图；
4. 单案不直接改写主题地图的普遍判断；
5. 没有明确关系时允许暂时只有标签，不强制制造双链；
6. 关键结论最终应能回到原始条款、判例、病历或其他证据。

## 批量关系变更检查

批量添加或清理关系时：

1. 先干跑，输出各主题预计新增、保留和移除数量；
2. 每个主题至少人工抽查“高相关、边界相关、明显不相关”三类样本；
3. 检查正文而不只检查标题、标签或文件目录；
4. 完成后验证 YAML、断链、同名歧义和主题地图与卡片关系一致性；
5. 链接数量上升不代表质量提升，优先保证高精度。

批量审计的轻量检查见[[05-Skills/知识关系质量评估标准|知识关系质量评估标准]]。

## AI 调用顺序

```text
任务或问题
-> 主题地图：读取当前判断、边界和待补问题
-> 知识卡片/判例视图：定位方法与证据
-> 原始材料：核验需要正式引用的内容
-> 输出
-> 将可复用反馈写回对应卡片或主题地图
```
