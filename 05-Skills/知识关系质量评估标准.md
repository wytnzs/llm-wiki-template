# 知识关系质量评估标准

> 这是知识库的轻量关系检查，不是给每条链接打分，也不要求把所有内容连接起来。

## 一、先理解这里的“图”

PDF 中的 GraphRAG 是在查询时沿“实体—关系—出处”进行多跳检索，并回到原文验证。

本知识库中的标签、双链和主题地图只是人工维护的轻量导航层：

- 标签负责分类；
- 双链负责指出明确关系；
- 主题地图负责提供围绕问题的精选入口；
- 原始材料负责最终溯源。

两者思路相通，但不是同一种系统。不要为了模仿知识图谱而把所有文件互相连接。

## 二、日常只问三个问题

新增或保留一条关系前，只判断：

1. **关系明确吗？**
   能否用一句话说清目标内容提供了什么证据、方法、例子、反例、边界或实际应用？
2. **沿过去有用吗？**
   AI 或人从当前内容进入目标内容后，是否能更好地回答问题、作出判断或采取行动？
3. **能够回到依据吗？**
   这条关系是否能回到正文、条款、判例、病历、项目记录或其他可靠来源？

### 处理方式

| 判断结果 | 处理 |
|---|---|
| 三项都是“是” | 建立或保留关系 |
| 关系可能存在，但暂时说不清 | 只保留标签，或记为待补关系种子 |
| 任一项明确为“否” | 不建立关系 |

不需要为普通双链评分，也不需要额外填写评估表。

## 三、标签、双链和主题地图怎么选

| 工具 | 用途 | 例子 |
|---|---|---|
| `tags` | 稳定分类和筛选 | 保险、AI、公众号、知识卡片 |
| `[[双链]]` | 两份具体内容之间存在明确关系 | 某判例是某项原则的应用实例 |
| 主题地图 | 围绕一个问题组织当前判断、方法、证据和边界 | 保险理赔主题地图 |

以下情况通常只用标签：

- 同属一个大领域；
- 生产内容时使用了 AI；
- 以后可能拿来写内容；
- 仅仅出现了相同关键词。

主题地图中的链接是精选下钻入口，不是同领域资料清单。

## 四、日常只维护三类入口

一份知识资产不需要填满所有关系字段。日常先维护：

1. **来源入口**：`source` 回答“从哪里来”；
2. **问题入口**：`related_theme_maps` 回答“直接帮助回答哪个主题问题”；
3. **使用入口**：`related_projects` 回答“在哪个项目中真实使用过”。

支持、反例和例子属于按需使用的语义关系。没有明确用途时留空，不因字段存在就补关系。目录只给文件一个稳定位置，不用新增目录表达每一种关系。

## 五、需要时再说明语义关系

能判断关系类型时，优先使用已有字段：

| 字段 | 含义 |
|---|---|
| `links.supports` | 支持某项判断 |
| `links.contrasts` | 提供反例或冲突观点 |
| `links.examples` | 提供具体例子 |
| `related_theme_maps` | 回答主题地图的核心问题 |
| `related_projects` | 项目真实使用了该资产 |

`links.applies_to`、`links.derived_methods` 等旧字段继续兼容，但作为高级可选关系；新卡片可以直接在“使用场景”和“已验证方法”正文中表达，不要求继续扩充 YAML。只有需要机器筛选或明确跨页导航时才使用这些字段。

`related_topics` 是现有卡片中的早期通用关联字段，不表示主题页关系，不得机械迁移为 `related_theme_maps`。

如果只能写“相关”，先不要建立正式关系。

保险、法律和医疗内容还要遵守一条边界：不能因为单个判例、单份病历或单次经验，就建立普遍性结论。

## 六、批量操作才做额外审计

批量添加或清理关系时，在三问法之外增加以下检查：

1. 先干跑，查看预计新增、保留和移除的数量；
2. 抽查高相关、边界相关和明显不相关三类样本；
3. 阅读正文，不只看标题、目录、标签或关键词；
4. 检查 YAML、断链、同名歧义和双向一致性；
5. 如果抽查出现系统性误连，立即收紧规则并重新运行；
6. 链接数量只作观察，不作为质量目标。

保险、法律和医疗中的关键结论链，应逐条核对证据和适用边界。

## 七、什么时候复查

出现以下情况时再复查，不做周期性重复打分：

- AI 沿关系连续返回不相关内容；
- 来源、正文或适用边界发生实质变化；
- 文件被移动、合并、替代或出现同名歧义；
- 新证据推翻了原来的连接依据。

复查后可以保留、改成标签、暂列待补或移除关系。移除关系不等于删除原始内容。

## 八、示例

### 应该连接

`AI任务处理优先级框架 → 保险+AI主题地图`

原因：它直接提供判断保险任务应该使用脚本、工具、工作流还是 Agent 的方法，并能回到卡片正文。

### 不应该连接

`交通事故财产保全内容 → 保险+AI主题地图`

原因：正文没有讨论 AI；即使生产时用了 AI，也只是生产方式。保留保险标签即可。
