# 内容审美与视觉系统操作手册

## 什么时候用

在这些场景使用：

- 写公众号前
- 公众号转小红书前
- 做封面或轮播图前
- 需要配图、解释图、图表美化前
- 要把自己的方法分享给别人前

## 标准流程

```text
先生成 content-brief
再生成 style-brief
最后进入平台生产
```

不要跳过 `style-brief`。否则每篇内容都会重新临场判断风格，长期会变散。

## 私有生产

私有生产的目标是最好效果。

允许使用：

- 本人的表达习惯
- 固定母风格
- 常用开头和收束方式
- 私有读者画像
- 个人判断方式

默认风格：

**克制知识编辑风 + 材质化解释图**

## 对外分享

对外分享时，只分享规范，不分享人格。

应该分享：

- 风格选择器
- 设计变量表
- 平台路由规则
- 图片提示词骨架
- QA 检查清单

不应该分享：

- 私人语气习惯
- 固定口头禅
- 未公开案例
- 私有选题判断
- 个人 IP 卡片原文

## 每次风格决策要问

1. 这篇内容是深度文、教程文，还是传播卡片？
2. 读者最需要的是完整理解，还是一眼抓住？
3. 需要图片解释关系吗？
4. 这篇内容的可信边界在哪里？
5. 如果转成分享版，哪些字段要留空给别人自己填？

## 最小检查清单

发布前检查：

- 主题路线是否只选了一条，没有混搭。
- 公众号是否耐读，小红书是否抓眼。
- 图片是否承担解释任务。
- 图内中文是否短、准、清楚。
- 图表数据是否准确。
- 保险/法律相关表达是否没有承诺结果。
- 分享版是否不会让别人写得像本人。
