# 产品方法论

> 核心理念：**知识资产生产线** = 输入 × 提炼 × 连接 × 输出 × 反哺

> 当前最小标准：README 是唯一必需入口；目录名称可调整；正式知识文件至少包含 `title`、`type`；地图、判例库和选题库均按需启用。

---

## 一、为什么需要一套方法论

大多数人的知识管理处于两个极端：

| 极端 | 表现 | 结果 |
|------|------|------|
| **收集癖** | 见什么存什么，文件夹越建越多 | 资料堆成山，用的时候找不到 |
| **空想家** | 知道要整理，但不知道整理成什么样 | 一直想，一直没动 |

这套方法论的目的是：**让知识从"放着"变成"能用"**。

---

## 二、五层架构

### 第1层：输入层 —— 有意识收集

不是所有信息都值得进入系统。输入层只做三件事：

| 入口 | 放什么 | 规则 |
|------|--------|------|
| Inbox | 临时灵感、网页剪藏、读书笔记草稿 | 只存不整理，降低记录门槛 |
| Resources | 完整的书、课程、PDF原文 | 按来源归档，不动原文 |
| 直接提炼 | 对话中冒出来的认知 | 告诉 AI "记一下" 就写入 |

**关键规则**：输入层是漏斗，不是仓库。东西进来后必须流走。

### 第2层：资产层 —— 加工成知识卡片

这是系统的核心。输入进来的原材料，加工成八种资产类型：

| 类型 | 英文 | 定义 | 举例 |
|------|------|------|------|
| 概念 | concept | 一个清晰定义的思考单位 | "拒赔六大维度" |
| 认知 | cognition | 对世界/问题的新理解 | "注意力是最宝贵的财富" |
| 原则 | principle | 能影响选择的判断标准 | "原子化、双向链接、概念为本" |
| 方法 | method | 可重复使用的处理流程 | "五层抽象金字塔" |
| 行动 | action | 可执行的最小动作 | "理赔前三步检查清单" |
| 证据 | evidence | 判例、事实、数据 | 保险拒赔判例 |
| 主题聚合 | theme | 围绕一个问题组织多张卡片 | "保险理赔主题地图" |
| 表达约束 | expression | 个人IP的语气和边界 | "王永涛风格约束" |

**加工标准**：一张卡片只讲一个概念，能跨场景复用，有明确的判断维度。

### 第3层：项目层 —— 围绕目标组织知识

资产层是"散装"的知识，项目层把它们组装成解决方案。

| 项目类型 | 示例 | 需要的资产 |
|---------|------|-----------|
| 内容生产 | 写一篇公众号文章 | 知识卡片 + 判例 + IP约束 |
| 产品开发 | 做一个理赔工具 | 判例库 + 方法论卡片 |
| 教学课程 | 讲一节理赔课 | 主题聚合 + 知识卡片 |
| 咨询服务 | 帮客户分析案件 | 判例库 + 维权路径 |

**关键规则**：项目有开始有结束。项目结束后，沉淀回资产层。

### 第4层：输出层 —— 内容/决策/产品

| 输出形式 | 说明 |
|---------|------|
| 公众号文章 | 完整逻辑，适合深度分析 |
| 小红书笔记 | 视觉化，适合轻量化传播 |
| 短视频脚本 | 口播，聚焦一个反常识观点 |
| 咨询报告 | 针对具体案件的分析和建议 |
| 决策判断 | 用知识卡片快速定位问题 |

**关键规则**：每一次输出都是对资产层的压力测试——你能产出的质量，取决于资产层的厚度。

### 第5层：反馈层 —— 复盘反哺，持续进化

输出之后，不是结束，而是沉淀：

```
发布 → 记录数据 → 提炼新卡片 → 更新判例标签
     → 记录失败原因 → 更新skill规则
     → 复盘可复用经验
```

**关键规则**：没有反哺的输出是消耗。每一次输出都要让系统变得更强。

## 三、核心流程：content-brief 交接机制

这是整条生产线的"交接单"。不是直接去数据库翻资料，而是先生成一份结构化的内容简报：

```json
{
  "topic": "本文主题",
  "core_viewpoint": "核心观点",
  "selected_cards": ["用到的知识卡片"],
  "selected_cases": ["支撑判例"],
  "forbidden": ["不能说的话"],
  "output_plan": {"公众号": "长文", "小红书": "轮播图", "短视频": "口播"}
}
```

**为什么需要交接单？**
- 每次写内容不用重新翻全库
- 公众号/小红书/短视频之间的素材复用有据可查
- 内容出了问题可以回溯——是判例错了、卡片错了、还是表达错了

## 四、与竞品的差异

| 对比项 | 本系统 | Notion / Obsidian | Roam Research |
|--------|--------|-------------------|---------------|
| 定位 | 内容生产引擎 | 笔记工具 | 笔记工具 |
| 核心 | 资产→输出管线 | 自由编辑 | 双向链接 |
| 输出 | 文章/视频/决策 | 笔记/文档 | 笔记/文档 |
| 管线 | 有标准化流程 | 无 | 无 |
| AI协作 | 深度嵌入（skill自动执行） | 插件级 | 无 |

**一句话差异**：别人是笔记本，我们是工厂。

---

## 五、适合谁用

| 人群 | 为什么需要 |
|------|-----------|
| 保险代理人/经纪人 | 需要持续输出专业内容建立信任，但没时间从零写 |
| 咨询/培训从业者 | 需要沉淀案例和方法论，形成可复用的知识资产 |
| 知识付费创作者 | 需要从素材到成品的标准化流程 |
| 律师/法务 | 需要判例管理和内容产出一体化 |
| 任何需要"持续输出"的知识工作者 | 系统的核心价值是降低输出的门槛 |

---

## 六、不适合谁

- 只想找个地方记笔记的人（用 Notion 就够了）
- 不需要对外输出内容的人（这套系统是为"生产"设计的）
- 不想投入时间学习系统的人（任何工具都有学习成本）
