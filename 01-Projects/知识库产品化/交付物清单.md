# 知识库产品交付物清单

## 一、所有用户必需

### 1. README 入口模板

说明知识库目的、实际区域职责、基本工作流、常用指令和已启用模块。README 是唯一必需的用户入口，不强制创建 `index.md` 或每级目录说明。

### 2. 最小目录建议

```text
00-Inbox/       输入
01-Projects/    当前项目
02-Areas/       长期资产
03-Resources/   统一来源（外部资料、个人输入、系统历史）
.claude/skills/ Claude Code 流程
```

这些名称是推荐映射。已有知识库可以保留原目录，只需在 README 写清楚每个区域承担的角色。归档不是必需的顶层目录；已处理但仍需追溯的材料留在来源层，用状态、标签和连接表达，真正重复或无保留价值的内容另行确认清理。

### 3. 最小文件规范

```yaml
---
title: 文件标题
type: 文件类型
---
```

推荐字段：`description`、`status`、`tags`、`source`、`created`、`updated`。模板应给出字段，不要求旧资料一次性补齐所有推荐字段。

### 4. 核心 Skill

- `knowledge-base-collab`：记录、整理、提炼、查询和来源管理；
- `feedback-writeback`：输出失败后的根因追溯和回写；
- `theme-map-maintenance`：已存在主题地图时维护综合认知，不强制创建选题。

### 5. 安装与检查工具

- Claude Code 初始化规则；
- Windows 初始化脚本；
- 乱码、链接、JSON 和公开边界检查；
- 一条可直接交给 Claude Code 的初始化指令。

## 二、按需选择

| 模块 | 适用场景 | 安装规则 |
|---|---|---|
| 知识库地图 | 文件多，需要动态统计与健康检查 | 安装前询问 |
| 判例库 | 法律、保险、咨询、医学等案例密集工作 | 安装前询问 |
| 选题库 | 持续生产文章、视频、课程 | 安装前询问，默认不装 |

选题 Skill 只能提出候选方向，用户确认后才正式入库。不能把“自动从知识卡片生成正式选题”作为当前卖点。

## 三、公开仓库包含

- README、标准、操作指南和不同人群适配；
- Claude Code 初始化规则；
- 通用模板、脱敏示例和核心 Skill；
- 三个可选模块；
- 初始化与质量检查脚本；
- 许可证和变更记录。

## 四、公开仓库不包含

- 真实判例和客户信息；
- 个人知识卡片、项目状态和内容成品；
- 账号、私域、联系方式和个人风格细节；
- 本地配置、缓存、草稿与历史迁移数据；
- 依赖未公开资料才能运行的私有 Skill。

## 五、服务交付

### 标准安装

1. 了解用户输入、用途和输出；
2. 判断新建还是改造已有知识库；
3. 确认目录映射与可选模块；
4. 初始化核心文件和 Skill；
5. 用一份样例完成输入到复盘的闭环；
6. 交付 README 和健康检查结果。

### 行业适配

在标准安装基础上，定义行业资产类型、案例字段、输出模板和专业边界。行业适配改变内容模型与 Skill，不应随意增加通用核心目录。

### 持续运营

根据真实使用反馈维护 Skill、模板、主题地图和项目定义。优先修复重复出现的问题，再考虑增加新功能。

## 六、验收标准

- [ ] 用户知道内容应先进入哪里、何时成为资产；
- [ ] Claude Code 能从 README 理解实际结构；
- [ ] 可选模块没有未经确认自动安装；
- [ ] 正式新文件包含最小字段；
- [ ] 私人内容没有进入公开仓库；
- [ ] 至少跑通一次“输入、整理、连接、使用、复盘”；
- [ ] 输出失败能定位并修正上游。
