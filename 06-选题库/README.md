# 06-选题库 · Topic Library

> 选题不是一次性灵感，而是可以沉淀、复用、迭代的内容资产。
> 正式选题必须能追溯到知识卡片、主题地图、判例/案例、个人经历或外部资料。

## 主要用途

1. **沉淀正式选题** — 已有资产支撑、可进入 content-brief 的选题。
2. **管理待挖掘方向** — 有表达苗头但暂时缺少资产支撑的方向。
3. **连接内容生产** — 选题卡进入 `.claude/drafts/content-brief.json` 后，再交给平台 skill 写稿。
4. **支持复盘回写** — 已发布选题需要记录效果，并反哺知识卡片、主题地图或 skill。

## 目录结构

- `cards/` — 正式选题卡片（按选题树分组）
- `选题总索引.md` — 全库索引
- `待挖掘方向.md` — 未成熟候选方向，不能直接进入生产
- `_templates/` — 选题卡片模板

## 正式入库标准

正式选题必须同时满足：

1. 有具体目标读者。
2. 有核心冲突或反差。
3. 有明确平台方向。
4. 至少关联一个来源资产：
   - `source_cards`
   - `source_cases`
   - `related_theme`
   - 个人经历或外部资料

无来源资产的内容，只能进入 `待挖掘方向.md`。

## 标准生产链路

```text
主题地图 / 知识卡片 / 判例 / 个人经历
→ 选题卡
→ content-brief
→ 公众号 / 小红书 / 短视频
→ 复盘
→ 更新选题卡状态和上游资产
```
