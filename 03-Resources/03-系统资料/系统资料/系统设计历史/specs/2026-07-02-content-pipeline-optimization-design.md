# 内容管线优化设计

基于知识库系统重构公众号+短视频内容生产体系。

## 一、核心变化

| 删除 | 新增 |
|------|------|
| delivery-brief.json 交付简报机制 | 06-Wiki/个人IP/ 人设DNA卡片 |
| 三角度选题（写给谁/解决什么/建立什么） | 认知引擎：卡片驱动的素材拆解 |
| IP skill 战略层（纺锤选题等） | 心法层：真诚分享的底层操作系统 |

## 二、心法层

```
每次写之前，先做这件事（不跳过的1分钟）：

1. 想一个具体的人
   他是你认识的人——那个被拒赔后不知道怎么办的代理人，
   那个买了保险但从来不敢看条款的朋友，那个刚入行被客户问住的新人。

2. 问自己：今天这件事，我为什么想讲给他听？
   不是"因为我需要发一篇文章"。
   是"他真的需要知道这个"。
   是"我想让他看到我没说清楚的那一面"。

3. 写完一段，检查一下：
   这句话他会听进去，还是划走？
   我是在跟他说话，还是在"输出内容"？
```

## 三、架构

```
心法层（底层OS）
  ↓ 贯穿全程
认知引擎层（01-Wiki卡片 + IP人设）
  ↓ 提供素材 + 思维框架
选题层（2-3个方向，每个标注"他在愁什么"）
  ↓ 用户选
生产层（调研→写作→审校→排版→配图→朋友圈）
  ↓
闭环层（发布→提炼卡片→记录效果）
```

## 五、各 Skill 改造清单

### gzh-pipeline

1. SKILL.md 开头新增「心法层」，在知识库联动之前
2. 删除触发前路由检查（delivery-brief.json 相关全部逻辑）
3. 删除阶段一第〇步（检查上游交付）
4. 新增「认知引擎」步骤：读取个人IP卡片 + 主题相关方法论卡片
5. 选题简化：去掉三角度，改为 2-3 方向 + "他在愁什么"
6. 删除所有 delivery-brief.json 引用和 style_constraints 检查
7. 审校阶段删除"风格一致性检查（当有上游交付时）"
8. 每一节写作增加"卡片重拆"检查点

### short-video-script

1. SKILL.md 开头新增「心法层」
2. 删除第〇步中的"读取IP skill前500行"逻辑，改为读取 06-Wiki/个人IP/
3. 新增「认知引擎」步骤：检索方法论卡片驱动内容框架
4. 增强开头钩子：结合镜像神经元/注意力卡片，自动生成3种钩子供选
5. 增强CTA：从IP卡片读取默认风格，不再每次手写
6. 输出格式增强：加口语节奏标记（停顿//、重音**、加速>）
7. 删除"配合 wangyongtao-ip 使用"相关描述

### wangyongtao-ip

1. 瘦身：只保留战略层能力（纺锤选题、冲突设计）
2. 日常内容生产不再需要先走本 skill
3. 仅深度文章/重要选题时作为战略顾问使用

### knowledge-base-collab

1. 新增「06-Wiki/个人IP/」归类规则
2. 新增"内容发布后提炼卡片"的标准流程
3. 增强"素材自动识别"在内容生产中的触发点
