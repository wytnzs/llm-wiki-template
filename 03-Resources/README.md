# 统一来源层

`03-Resources/` 保存当前知识库需要保留和核验的原始来源。按来源类型分三层，看目录前缀即知是什么：

```
03-Resources/
├── 01-外部资料/         ← 别人写的、别人讲的、从外面收集的
├── 02-个人输入/         ← 你自己写的、录的、想的东西
└── 03-系统资料/         ← 知识库本身怎么演变到今天
```

## 三层的日常用法

| 层 | 什么内容 | 怎么对待它 |
|---|---|---|
| **01-外部资料** | 课程笔记、书籍摘要、行业报告PDF、文章收藏 | 提供外部观点和事实，**不代表你的判断**。引用时需核验时效和准确性 |
| **02-个人输入** | 得到大脑语音转写、工作笔记、灵感记录、读书批注 | 是你的原始表达。可追溯观点从哪里来的，但原始想法不等于最终判断 |
| **03-系统资料** | 旧方案、迁移记录、项目历史 | 只在复盘和恢复时调用。当前规则以现行文件为准 |

## 使用方式

1. 先从[[03-Resources/资源工作台|资源工作台]]判断哪组资料能回答当前问题；
2. 沿集合导读进入具体文件、章节或附件；
3. 只有改变判断、方法或行动的内容才提炼到 `02-Areas/`；
4. 项目真实调用后再记录使用关系；
5. 原始正文保持不变，标签和关系优先维护在集合导读与正式知识资产中。

## 三条底线

- **外部资料不自动代表用户判断**——课程说的不是你说的，报告写的不等于你同意的
- **个人输入不自动证明外部事实**——你的笔记可能有记忆偏差，引用时回到原始文件
- **系统历史不替代当前规则**——旧方案只在复盘时参考，当前规则以现行文件为准
