# 好东西都是总结出来的

一个很奇怪的现象：我用AI干活已经很久了，但AI味儿这东西，你一读就能感觉到——假、绕、不像人话。

更有意思的是，用AI来干掉AI味儿，结果往往比AI味儿还AI味儿。这不是讽刺，这是一个真实的问题。

直到我遇到了content-pipeline。一个不是告诉我"怎么写好"的内容生产管线，而是告诉我"AI写作为什么假"以及"怎么干掉AI味"的东西。

用了两周，我发现一件事：它生成的内容可读性很强。不是"不错"，是"真的能看完"。

然后我发现了一个更值得写的话题：**它背后的元Skill方法论——如何设计一个真正好用的skill。**

这篇文章是我的复盘，不是教程。我会拆解content-pipeline的核心机制，然后总结设计skill的方法论。如果你也在用AI工具做内容，或者你想设计自己的skill，这篇会有用。

---

## 01 三个机制，让AI写出了人话

### 机制一：素材收集——不是创作，是组织

content-pipeline有一个自动记录系统。在对话中会主动识别"有料瞬间"并记录，不需要手动触发。

**识别6种信号：**

| 类型 | 识别信号 | 举个例子 |
|------|---------|---------|
| 踩坑翻车 | 预期≠结果 | "试了三种方案都不行" |
| 意外发现 | 没想到、原来可以 | "居然这样就解决了" |
| 迭代打磨 | 改了多版、从复杂到简洁 | "200行改成20行还能跑" |
| 搞笑时刻 | AI抽风、神奇bug | "它认真地给我写了一堆错的" |
| 突破时刻 | 卡了很久终于通 | "困扰一周的bug终于找到了" |
| 方法沉淀 | 可复用的技巧 | "以后遇到这种情况就这么办" |

触发自动记录时，段落结尾会标记`（✓ 素材+1）`，不打断对话。

为什么这个机制有效？

**第一，真实素材自带质感。** 时间细节、数字、情绪——这些是编不出来的。

**第二，AI不是在创作，而是在组织。** 你给了它半成品，它帮你整理。这个分工更符合AI的能力边界。

**第三，读者感受到的是真实过程。** 不是AI堆砌的套话，是你实际踩坑、实际解决的过程。

content-pipeline的核心逻辑：不是让AI凭空创作，而是帮AI组织真实发生的事。

不是先想主题再编内容，而是先有素材再出稿。

素材是锚点，锚点是真的。

### 机制二：反AI写作——五把刀

这是content-pipeline最核心的创新。

它不是问"怎么写好"，而是问"AI写作为什么假"。

**五把刀：**

| 武器 | 问题 | 解法 |
|------|------|------|
| 禁用词 | AI习惯用"笔者、本文、众所周知" | 明确禁用，口语词替代 |
| 段落极短 | AI习惯三段论长段落 | 强制1-3句，单句成段做"钉子" |
| 数字具体 | AI习惯"很多、不错" | 具体数字+制造反差 |
| 金句句式 | AI写不出有冲击力的句子 | "不是A，是B"句式强制训练 |
| 诚实标注 | AI伪装人类产生违和感 | 强制标注"调研&撰写：AI" |

**禁用词清单：**

❌ 禁用：笔者、本文、众所周知、不可否认、接下来让我们、那么、此外、与此同时

✅ 启用：搓、搞、跑通、翻车、折腾、吃灰、干、搞定

段落极短是最显著的风格标记。平均1-3句话，超过5句的段落几乎不存在。

大量单句成段，独立一行，起"钉子"作用。

就这么简单。但就这么管用。

全程没问我"确定吗"，直接干。

知识在繁殖。

节奏型：信息段（中）→ 钉子段（短）→ 新信息段（中）→ 钉子段（短）。读起来像敲钉子——一段信息，一锤定音。

### 机制三：内容类型判断——不是一套模板打天下

不是一套模板打天下，而是根据内容类型匹配骨架。

**三种内容类型：**

| 类型 | 判断信号 | 框架结构 | 篇幅 |
|------|---------|---------|------|
| 教程类 | "教人安装/使用/配置" | 先看结果→概念→操作→实战→拿走即用 | 2000-4000字 |
| 说明书类 | 工具/产品介绍、知识库发布 | 定义+成果→功能→用法→价值→扩展→收束 | 3000-6000字 |
| 深度长文 | 行业分析、观点输出 | 序言→背景→论述→案例→升华 | 8000-12000字 |

判断逻辑：

```
if "教人做xxx" → 教程框架
if "工具介绍/产品说明" → 说明书框架
if "分析/观点/故事" → 深度长文框架
```

框架匹配不是机械套用，而是让内容结构和读者预期对齐。教程类读者想"学会"，说明书类读者想"理解全貌"，深度长文读者想"得到一个新观点"。

## 02 元Skill方法论：好东西都是总结出来的

以上是content-pipeline的三个机制。但我更想搞清楚的是：这些机制是怎么来的？

答案在元Skill方法论里。

**核心命题：好东西都是总结出来的，不是设计出来的。**

**核心公式：**

```
好东西 = 总结出来 ≠ 设计出来
OTF（边做边总结）+ JIT（小步快跑）+ Bootstrap（知识自举）
```

| 缩写 | 全称 | 含义 | 在content-pipeline中的体现 |
|------|------|------|----------------------------|
| OTF | On-The-Fly | 发现模式立即总结，不等做完 | 写作风格从9篇真实文章逆向工程 |
| JIT | Just-In-Time | 及时交付最小版本，快速迭代 | 三种文章框架的渐进式演化 |
| Bootstrap | 知识自举 | 每步产出成为下一步的燃料 | 素材→文章→排版→配图→分发，链路自然生长 |

**OTF案例：** content-pipeline的写作风格从9篇真实文章逆向工程出来——不是先设计风格再去写，而是先写完再总结规律。

**JIT案例：** 三种文章框架不是一次性设计完的——教程框架先出来，验证有效后，再延伸出说明书框架和深度长文框架。

**Bootstrap案例：** 素材→文章→排版→配图→分发，这条链路不是设计出来的，而是自然生长出来的——前一步的产出自然变成下一步的输入。

好东西 = 总结出来 ≠ 设计出来。这个原则不只适用于skill设计，适用于任何需要不断迭代的工作。

## 03 五层抽象金字塔

这是"总结"的具体操作方法。任何复杂技能都可以压缩到这五层：

```
层5 哲学（本质）← 回答"为什么"
层4 洞察（规律）← 回答"什么更重要"
层3 原则（策略）← 回答"怎么做"
层2 模式（规律）← 回答"重复什么"
层1 案例（事实）← 回答"发生了什么"
```

**content-pipeline的五层结构：**

| 层次 | content-pipeline中的具体内容 |
|------|------------------------------|
| 层1 案例 | 9篇已发布文章、130章史记数据、423个标注错误 |
| 层2 模式 | "段落极短"、"数字制造反差"、"钉子段" |
| 层3 原则 | "口语>书面"、"先结果后过程"、"禁用词机制" |
| 层4 洞察 | "AI味来自伪装真人"、"真实素材>凭空创作" |
| 层5 哲学 | "不是在写文章，是在跟朋友复盘" |

从层1到层5，是从具体到抽象、从事实到本质的过程。

每一层都为下一层提供燃料。案例积累到一定程度，模式浮现；模式验证到一定程度，原则固化；原则背后的规律，就是洞察；洞察再往上，是哲学。

## 04 四步法：设计自己的Skills

基于元Skill方法论，我总结了设计skill的四步法：

**Step 1：冷启动（5个样本）**

不要一开始就设计完整框架。先做5个具体案例，记录遇到的问题。

**Step 2：归纳模式**

这5个案例有什么重复？哪些步骤可以自动化？哪些是人的判断、哪些是规则？

**Step 3：固化骨架**

把模式写成文档，把规则写成检查清单，把重复步骤写成脚本。

**Step 4：迭代收敛**

用新案例检验骨架。发现不适用→调整。连续2轮无新模式→收敛。

**自举路径：**

```
人工执行（0%自动）→ 脚本辅助（20%自动）→ Skill沉淀（60%自动）→ 自动迭代（90%自动）
```

每一步的产出都是下一步的燃料。这就是Bootstrap——知识自举。

## 05 回到开头：content-pipeline为什么有效

回到开头的问题：content-pipeline为什么有效？

不是因为它有完美的设计，而是因为它建立在大量真实实践的基础上。它的每一个机制都来自反馈和迭代，不是闭门造车。

它的核心洞察是：**AI味来自伪装真人。** 所以它的解法不是"怎么伪装得更像"，而是"怎么坦诚地使用AI"。

这对我的启发是：

**设计skill不是设计产品，而是记录实践。** 你需要先做，在做的过程中记录模式，然后把这些模式固化成骨架，最后用新案例迭代优化。

**内化skill不是学会使用，而是理解原理。** 知道为什么有效，比知道怎么用更重要。前者让你能迁移，后者只能让你照搬。

**好东西都是总结出来的，不是设计出来的。** 这个原则不只适用于skill设计，适用于任何需要不断迭代的工作。

工具不用完美，能跑就发。在用的过程中迭代，比想好了再做更有效率。
