五个真实场景

场景一：批量提取理赔材料关键信息

一个工伤案子，客户发来 8 份 PDF——病历、诊断证明、事故认定书、保险合同……加起来上百页。

以前我得一份份打开，手动摘。现在我把所有 PDF 放到一个文件夹，对 Claude Code 说一句话：

"读这个文件夹里的所有文件，提取：就诊日期、诊断结论、事故发生时间、涉及的保险条款编号、保险公司拒赔理由。输出一张表。"

几分钟，表出来了。我再花十几分钟核实一遍——完事。

它不是在帮我打字。是在帮我做"信息提取"这个工序。

场景二：产品条款差异对比

客户有两份百万医疗险，要我帮忙看看哪个更好。以前我是两个 PDF 左右分屏，逐条对着看。

现在："对比这两个产品条款，重点看：免赔额设置、等待期、免责条款、续保条件、特殊门诊覆盖范围。输出差异表，标出谁优谁劣。"

输出来一张表，差异项用颜色标好。我再过一遍关键判断。

它不是在帮我读。是在帮我做"结构化对比"——这正是人脑最不擅长的事。

场景三：写公众号初稿

我的流程不是"让 AI 直接写"。是：先给它素材+大纲+风格约束 → 它出初稿 → 我审——改观点、调语气、删废话、补案例 → 回去让它排版生成 HTML。

每篇文章 AI 参与了约 60% 的工作量。但那 40%——观点、语气、案例选择——决定了这篇文章是我的还是机器的。

AI 写不出一篇像你的文章。但它能帮你省掉 60% 你不用动脑的活。

场景四：数据分析

我想知道最近半年处理的理赔案子里，哪种拒赔理由出现频率最高。以前只能凭印象说"好像既往症拒赔比较多"。

现在把脱敏后的案件记录丢给 Claude Code："分析这些案件，统计拒赔理由分布、保险公司响应时间、成功翻盘的比例。画个图。"

五分钟，数据+图表全出来。

一个代理人如果能用数据说话——你跟客户聊的时候，底气完全不一样。

场景五：法规判例搜索

保险法、司法解释、各地法院判例——这些东西散落在十几个网站里。Claude Code 可以联网搜索，然后自动汇总、分类、生成简报。

比如："搜索 2025-2026 年关于'既往症''如实告知义务'的保险合同纠纷判例，提取案号、争议焦点、法院认定、判决结果。输出表格。"

以前干这个活得一下午。现在十分钟。
