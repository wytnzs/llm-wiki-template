# 做一个保险 AI 工具的实际成本——时间和钱

经常有人问我：你做的那些工具，花了多少钱？

拒赔维权分析、伤残鉴定预估、家庭财务分析——听起来像是要一个团队干半年。

实际上，三个工具加在一起，花了几千块。时间加起来不到三个月。一个人。

这篇文章把成本全拆开。如果你想自己做工具，可以当预算参考。

---

## 三个工具，花了多少

| 工具 | 开发时间 | 现金成本 | 核心能力 |
|------|---------|---------|---------|
| 拒赔维权分析 | ~4 周 | 约 800 元 | 输入拒赔通知书+保单 → 输出抗辩要点 |
| 交通事故伤残鉴定 | ~3 周 | 约 600 元 | 输入伤情+事故描述 → 输出等级预估+赔偿区间 |
| 家庭财务分析 | ~2 周 | 约 300 元 | 输入收入/支出/资产/负债 → 输出保障缺口 |

**总计：约 9 周，1700 元。**

---

## 钱花在哪了

### 大模型 API 调用费（最大头）

我用的技术栈是 Cloudflare Worker + DeepSeek API。DeepSeek 的 API 价格大概是 GPT-4 的十分之一。

开发阶段调用量大——反复测试、调提示词、跑用例。三个工具的 API 费加起来大概 1000 块。

上线之后反而便宜了。一个拒赔分析请求，API 成本大概 0.3-0.8 元。用户量没起来之前，一个月几十块。

> 大模型 API 是最大的开支。但选对模型，成本可控。DeepSeek 的性价比目前没有对手。

### 服务器和域名

Cloudflare Worker 有免费额度。日均几百次调用以内，不花钱。

域名一年几十块。

这部分基本可以忽略。

### 数据采购

伤残鉴定工具最花钱的地方——买《人体损伤致残程度分级》的完整数据和各地赔偿标准表。找了第三方数据商，花了 500 块。

其他两个工具用的都是公开数据：保险法条、司法解释、裁判文书网的判例——免费。

### 设计和其他

没花钱。工具界面就是表单+结果展示，不需要设计。我自己用 Claude Code 写的 CSS。

## 时间花在哪了

钱不是最大的成本。时间才是。

### 第一阶段：搭原型（占总时间 30%）

把想法变成一个能跑的东西。这个阶段最快——因为有 Claude Code 帮我写代码。

一个工具的核心功能，用 Claude Code 大概 2-3 天就能搭出能跑的原型。不会写代码也能做——自然语言描述需求，Claude Code 生成代码。

### 第二阶段：调效果（占总时间 40%）

这个阶段最花时间，也最容易被忽略。

原型搭好了，但出来的结果不够好。比如拒赔分析——原型只能告诉你"保险公司引用了第 XX 条"，但说不清楚"这条在法律上能不能被推翻"。

你得调。反复改提示词框架、补数据、加约束条件、跑测试用例。这个过程没有捷径——就是一遍遍试。

> 做工具最难的不是"做出来"。是"做到能用"。

### 第三阶段：找人试（占总时间 20%）

做出来之后，找几个同行试用。收集反馈——"这个结果不对""这里看不懂""能不能加一个 XX 功能"。

试用的反馈有一半是你没想到的。另一半是你想到了但觉得不重要、实际很重要。

迭代两三轮之后，工具才算"真的能用"。

### 第四阶段：整理和文档（占总时间 10%）

写使用说明、录个简单的演示视频、整理常见问题。

这个最没技术含量，但不做不行。再好的工具，没人知道怎么用，等于没做。

## 你的第一个工具，建议怎么开始

**第一步：选一个你每周至少遇到三次的问题。**

别想做"万能工具"。做一个只解决一件事的工具。你越了解这个问题，做出来的东西越准。

**第二步：先用现有 AI 试。**

把这个问题丢给 DeepSeek 或 Claude，看看它能不能答。如果能答但不够好——说明缺的不是 AI 能力，是场景框架和数据。这就是你的工具的切入点。

**第三步：搭一个最简原型。**

不需要界面好看。一个表单+一个结果框就行。让 Claude Code 帮你写——你现在就可以试。

**第四步：找 5 个人试。**

别找夸你的人。找会用、会挑毛病的人。他们的吐槽比点赞值钱一百倍。

> 第一个工具的目标不是"好用"。是"能跑"。能跑之后，迭代才是真正的开发。

---

## 值不值

有人问我：花 9 周和 1700 块，做三个工具，值吗？

我的回答：**一个工具帮 100 个人，一个人省 30 分钟——就是 3000 分钟。我花 9 周换 3000 分钟，值。**

而且工具是资产。你做完了，它就是你的。明年还在用，后面还在用。

> 理赔帮一个人。工具帮一群人。这就是我为什么一直在做。
