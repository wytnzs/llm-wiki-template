# Claude Code 怎么用在保险场景里——从"聊着玩"到"真正干活"

去年我开始用 Claude Code 的时候，身边没一个同行在用。

他们的反应分两种。一种问"那是什么"。一种问"你不就是卖保险的吗，写代码干嘛"。

一年下来，Claude Code 已经是我日常干活用得最多的工具——不是写代码，是处理保险业务里那些"重复、费脑、容易出错"的事。

这篇文章不讲技术。讲场景。

---

## Claude Code 能干嘛——保险版

Claude Code 和 ChatGPT、豆包最大的区别：它不是聊天窗口。

它是一个能直接读文件、写文件、执行命令、联网搜索的 AI Agent。你给它一个任务，它自己拆解、自己执行、自己检查。

翻译成保险场景，大概这五件事：

| 场景 | 以前怎么做 | Claude Code 怎么做 |
|------|----------|-------------------|
| 批量整理理赔材料 | 一份一份手动翻 | 把 PDF 全丢进文件夹，一句话让它提取关键字段 |
| 比对保单条款差异 | 两个文档对着看，眼睛花 | 丢给它，输出差异表 |
| 写公众号文章 | 打开空白文档发呆 | 给素材+大纲，生成初稿 |
| 做数据分析 | Excel 拉半天 | 说一句话，它写脚本跑 |
| 查法规和判例 | 百度一个个翻 | 联网搜索+自动汇总 |

> 聊天窗口是"问一句答一句"。Agent 是"你说目标，它自己干完"。

---

## 五个真实场景

### 场景一：批量提取理赔材料关键信息

一个工伤案子，客户发来 8 份 PDF——病历、诊断证明、事故认定书、保险合同……加起来上百页。

以前我得一份份打开，手动摘：就诊时间、诊断结论、保险事故日期、条款引用。

现在我把所有 PDF 放到一个文件夹，对 Claude Code 说一句话：

> 读这个文件夹里的所有文件，提取：就诊日期、诊断结论、事故发生时间、涉及的保险条款编号、保险公司拒赔理由。输出一张表。

几分钟，表出来了。我再花十几分钟核实一遍——完事。

**它不是在帮我打字。是在帮我做"信息提取"这个工序。**

### 场景二：产品条款差异对比

客户有两份百万医疗险，要我帮忙看看哪个更好。

以前我是两个 PDF 左右分屏，逐条对着看。费眼、费脑、容易漏。

现在：

> 对比这两个产品条款，重点看：免赔额设置、等待期、免责条款、续保条件、特殊门诊覆盖范围。输出差异表，标出谁优谁劣。

输出来一张表，差异项用颜色标好。我再过一遍关键判断——客户问什么我答什么。

**它不是在帮我读。是在帮我做"结构化对比"——这正是人脑最不擅长的事。**

### 场景三：写公众号初稿

这个你们已经看到结果了。

我的流程不是"让 AI 直接写"。是：

1. 先给它素材+大纲+风格约束
2. 它出初稿
3. 我审——改观点、调语气、删废话、补案例
4. 回去让它排版生成 HTML

我写的每篇文章，AI 参与了约 60% 的工作量。但那 40%——观点、语气、案例选择——决定了这篇文章是我的还是机器的。

> AI 写不出一篇像你的文章。但它能帮你省掉 60% 你不用动脑的活。

### 场景四：数据分析——代理人最该用但最少用的功能

举个真实例子：我想知道最近半年我处理的理赔案子里，哪种拒赔理由出现频率最高。

以前我只能凭印象说"好像既往症拒赔比较多"。

现在我把脱敏后的案件记录丢给 Claude Code：

> 分析这些案件，统计拒赔理由分布、保险公司响应时间、成功翻盘的比例。画个图。

五分钟，数据+图表全出来。

**一个代理人如果能用数据说话——"我经手的案子里，XX 类型拒赔翻盘率最高"——你跟客户聊的时候，底气完全不一样。**

### 场景五：法规判例搜索

保险法、司法解释、各地法院判例——这些东西散落在十几个网站里。

Claude Code 可以联网搜索，然后自动汇总、分类、生成简报。

比如：

> 搜索 2025-2026 年关于"既往症""如实告知义务"的保险合同纠纷判例，提取案号、争议焦点、法院认定、判决结果。输出表格。

以前干这个活得一下午。现在十分钟。

## 怎么开始

不需要会写代码。Claude Code 你用自然语言就能操作。

第一步：装好 Claude Code（网上有教程，十分钟）。

第二步：从一件事开始。不要上来就想"全面改造工作流"。就挑一个场景——比如"帮我对比两份保单"——试一次。

第三步：试完你会发现——原来不是你用不上 AI。是你之前用的 AI 工具，形态不对。

> 聊天窗口让你觉得 AI 只能聊天。Agent 让你意识到，AI 能干活的。

---

## 一个提醒

Claude Code 能读文件、能写文件、能执行命令——这既是它的强项，也是你要注意的地方。

回到那三条红线：**数据先脱敏、决策你来下、AI 输出的东西先审核。**

Agent 帮你省的是"做"的时间。不是"判断"的责任。
