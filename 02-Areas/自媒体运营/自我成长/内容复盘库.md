## 反哺判断

每次发布后执行：

### 1. 是否需要新增知识卡片？

| 发现 | 操作 |
|------|------|
| 文章中有新的判断标准/方法论 | → 提炼卡片 → 写入 02-Areas/知识卡片/ |
| 某个概念值得单独记录 | → 新建概念卡片 |
| 与已有卡片产生新关联 | → 更新已有卡片的 [[链接]] |

### 2. 是否需要补充判例标签？

- 这篇文章用到的判例是否值得增加内容传播标签？
- 判例的典型意义是否需要补充？

### 3. 是否需要沉淀表达资产？

| 有效资产 | 写入位置 |
|---------|---------|
| 有效开头 | 表达资产库.md → 好开头 |
| 有效标题 | 表达资产库.md → 好标题 |
| 有效金句 | 表达资产库.md → 好金句 |
| 有效CTA | 表达资产库.md → 好CTA |

### 4. 选题池状态更新

- 这篇文章的选题 → 更新为"已发布"或"已复盘"
- 评论/私信中出现的用户新问题 → 考虑是否入库为新选题

|------|------|-----------|---------|---------|---------|---------|----------------|

### 问题类型

- 选题不清
- 判例调用错误
- 知识卡片调用不足
- 口吻不对
- 结构像文章不像视频
- 法律边界说满
- 缺少用户痛点
- 缺少最小行动
- 平台适配失败

### 反哺规则

| 问题类型 | 修正位置 |
|---------|---------|
| 选题不清 | 更新选题卡模板 |
| 判例调用错误 | 更新 content-brief 规则 |
| 口吻不对 | 更新 个人IP 卡片 |
| 结构不对 | 更新对应平台 skill |
| 法律边界问题 | 更新判例库/内容管线禁忌规则 |
| 缺少用户痛点 | 检查认知引擎是否加载 |
| 平台适配失败 | 更新平台对应 skill 规则 |
