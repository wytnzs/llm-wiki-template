# 主题中心

> 日常只需要从本页进入。主题是需要持续回答的问题；聚合是筛选资料的动作；主题页是保存当前判断、证据、方法、反例和边界的内容单元。它们共同属于“主题中心”，不再作为三套功能分别维护。

模块结构：

```text
主题中心/
├── README.md      自然语言索引和统一入口
└── 主题页/         每个持续问题一张独立主题页
```

索引负责先找到主题，主题页负责综合判断，双链负责继续扩展，知识卡片和判例负责方法与证据，`03-Resources/` 统一来源层负责最终核验。

## 当前关系边界

- 判例的拒赔理由和疾病关系以硬链接视图为准，不复制案件；
- 主题地图只综合多份证据，不用单案证明普遍结论；
- 外部资源可以链接主题，但不自动代表用户判断；
- 归档和 Inbox 允许暂时没有双链，不为了消除孤立页制造关系；
- 正式引用仍需回到条款、判例、病历或其他原始证据。

## 使用顺序

```text
问题
-> 主题中心：按自然语言问法定位主题
-> 主题页：读取当前判断、边界和待补问题
-> 知识卡片/判例：定位方法与证据
-> 03-Resources/（外部资料/个人输入/系统资料）：核验需要正式引用的原始依据
-> 项目或输出
-> 把可复用反馈写回知识卡片、主题页或 Skill
```
