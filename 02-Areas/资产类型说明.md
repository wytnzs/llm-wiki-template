# 资产类型说明

## 类型清单

| 类型 | 英文名 | 说明 | 示例 |
|------|--------|------|------|
| 概念 | concept | 一个清晰定义的思考单位 | “拒赔六大维度” |
| 认知 | cognition | 对世界或问题的新理解 | “注意力是最宝贵的财富” |
| 原则 | principle | 能影响选择的判断标准 | “原子化、双向链接、概念为本” |
| 方法 | method | 可重复使用的处理流程 | “五层抽象金字塔” |
| 行动 | action | 可执行的最小动作 | “理赔前三步检查清单” |
| 主题聚合 | theme | 围绕一个问题沉淀当前判断、证据、方法和边界 | “知识管理主题地图” |
| 证据 | evidence | 判例、事实、数据、原文材料 | 判例库中的每个判例 |
| 表达约束 | expression | 个人IP、语气、禁忌、CTA | 个人IP卡片 |
| 选题 | topic | 可进入内容生产的表达入口 | “普通人为什么做知识库总失败” |

## 判断标准

- **只服务某一类案件的事实** → 留在判例库（证据层）
- **能跨场景复用的抽象框架** → 提炼为知识卡片（解释层/方法层）
- **决定表达方式和立场的内容** → 写入个人IP卡片（表达约束层）
- **围绕某个问题形成稳定判断的内容** → 进入主题聚合（长期记忆层）
- **能转化为文章、短视频、直播的内容** → 进入选题库（内容入口层）

## YAML 类型建议

新增文件优先使用以下 `type`：

| 文件 | type |
|------|------|
| 知识卡片 | knowledge_card |
| 主题地图 | theme_map |
| 选题卡 | topic_card |
| 项目状态 | project_status |
| 复盘记录 | review |

## 成熟度标记

| 标记 | 含义 |
|------|------|
| seed | 刚萌芽，需要验证 |
| growing | 有实践支撑，正在完善 |
| stable | 经过多次验证，可稳定复用 |
