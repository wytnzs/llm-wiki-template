# 02-Areas 长期资产层

> 02-Areas 不是资料仓库，而是长期资产层。
> 它负责把原始输入加工成可复用资产：概念资产 → 关系资产 → 判断资产 → 方法资产 → 行动资产 → 输出资产。

## 目录

| 目录 | 用途 |
|------|------|
| 知识卡片/ | 解释层：原子化概念、判断标准、方法论、行动清单 |
| 判例库/ | 证据层：保险拒赔判例三视图架构（按险种/按拒赔理由/按疾病分类） |
| 主题聚合/ | 长期记忆层：围绕一个问题沉淀当前核心判断、证据、方法和边界 |

## 资产类型

见 `资产类型说明.md`。

## 运行方式

02-Areas 是内容生产和项目决策的上游。标准调用链路：

```text
00-Inbox / 03-Resources
→ 02-Areas/知识卡片
→ 02-Areas/主题聚合
→ 06-选题库
→ content-brief
→ 输出内容
→ 复盘回写
```

## 原则

1. 每一份资产都必须回答“它能帮我做什么判断、决策或行动”。
2. 新增知识卡片、主题地图、选题卡应尽量带 YAML 元数据。
3. 资产之间用 `[[双向链接]]` 建立关系网络。
4. 判例库只存事实和证据，不写传播包装。
5. 主题聚合不是资料列表，而是当前版本的长期记忆。
6. 定期清理过时资产，保持资产层活跃。
