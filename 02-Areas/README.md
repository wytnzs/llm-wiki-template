# 02-Areas 长期资产层

> 02-Areas 不是资料仓库，而是长期资产层。
> 它负责把原始输入加工成可复用资产：概念资产 → 关系资产 → 判断资产 → 方法资产 → 行动资产 → 输出资产。

## 目录

| 目录 | 用途 |
|------|------|
| 知识卡片/ | 解释层：原子化概念、判断标准、方法论、行动清单 |
| 判例库/ | 证据层：保险拒赔判例三视图架构（按险种/按拒赔理由/按疾病分类） |
| 主题聚合/ | 综合认知层：保存各主题页的当前判断、证据、方法、反例和边界；目录名为兼容现有链接保持不变 |

## 资产类型

见 `资产类型说明.md`。

## 运行方式

02-Areas 是内容生产和项目决策的上游。标准调用链路：

```text
00-Inbox / 03-Resources
→ 02-Areas/知识卡片
→ 02-Areas/主题聚合
→ 当前项目 / content-brief
→ 06-选题库（本库已启用，按需使用）
→ 输出内容
→ 复盘回写
```

## 主题、聚合和主题页

- 主题是需要持续回答的问题；
- 聚合是筛选相关资料的动作；
- 主题地图是聚合形成的主题页，不是资料清单，也不是新的原始内容副本；
- `02-Areas/关系导航.md` 只负责进入不同主题页；
- 根目录 `知识库地图.md` 只负责展示全库状态。

主题页供人和 AI 先读取当前判断，再下钻到知识卡片、判例和 `03-Resources/`、`04-Archive/` 中的原始依据。

## 原则

1. 每一份资产都必须回答“它能帮我做什么判断、决策或行动”。
2. 新增正式资产至少包含 `title` 和 `type`；新建或实际更新的知识卡片优先补充 `description`、可核验来源和失效边界。
3. 资产之间按[[05-Skills/知识关系与标签规范|双向链接规范]]建立关系网络。
4. 判例库只存事实和证据，不写传播包装。
5. 主题页不是资料列表，而是当前版本的综合认知；`core_question` 必须写成一个能够回答的问题。
6. 定期清理过时资产，保持资产层活跃。
7. 选题库是本库已启用的可选模块，候选方向经确认后才正式入库。

## 日常最小关系

知识卡片不需要填满所有关系字段，只优先维护：

1. `source`：从哪里来；
2. `related_theme_maps`：直接帮助回答哪个主题问题；
3. `related_projects`：在哪个项目中真实使用过。

支持、反例和例子等关系只在能够说清用途并回到依据时补充。没有明确关系时允许只有标签。
