# Codex 协作规则

开始任务前先阅读根目录 `README.md`。README 是知识库目的、目录职责和已启用模块的唯一说明入口。

## 协作定位

- Codex 默认负责需求判断、设计、执行、检查和返工闭环，不只输出操作建议；
- `.claude/skills/` 是共享 Skill 的源目录，不在多个入口分别维护同名 Skill；
- 不让两个 Agent 同时修改同一成品；需要第二个 Agent 时，优先让其独立审查，再由主执行者统一落实；
- 工作区已有修改默认属于用户或其他协作者，不回滚、不覆盖无关改动。

## 基本流程

```text
输入 -> 整理 -> 连接 -> 使用 -> 检查 -> 复盘
```

1. 先判断内容属于输入、项目、资产、资源还是归档；
2. 保留用户原始表达，只有可跨项目复用并帮助判断、决策或行动的内容才进入资产层；
3. 区分用户判断、外部资料和案例事实，记录来源和适用边界；
4. 查询时按“主题页 -> 知识卡片或案例 -> 原始依据”下钻；
5. 任务匹配现有 Skill 时先读取对应 `SKILL.md`；
6. 输出失败时追溯到输入、知识、项目定义、模板或 Skill，并把可复用修正写回源头。

## 关系规则

- 标签负责稳定分类；
- 双链只表达明确、可用、可追溯的具体关系；
- 主题页保存围绕一个持续问题的当前判断、证据和边界；
- 聚合是筛选资料的动作，不单独形成知识类型；
- 不为消灭孤立页、增加关系数量或填满字段制造链接。

## 写入与安全边界

- 不静默删除、覆盖或批量迁移；
- 正式知识文件至少包含 `title` 和 `type`；
- 原始材料默认只读，来源不确定时不猜测；
- 不把单个案例外推成普遍承诺；
- 案例、合同、病历和法律或医疗关键结论只能提出建议，确认后写入；
- 提交、推送和对外发送必须获得明确授权；
- 公开仓库只发布规则、模板、示例、Skill 和通用脚本，不上传真实卡片、案例、客户资料、项目材料、内容成品、凭证或个人配置。

初始化后的知识库从以下入口读取详细说明：

- `05-Skills/知识库运作系统与协作边界.md`
- `05-Skills/知识关系与索引规则.md`
- `05-Skills/定期维护与审计.md`

在模板仓库中，对应源文件位于 `docs/`。
