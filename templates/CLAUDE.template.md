# 知识库协作规则

本知识库是一套**知识资产生产系统**，不是普通资料库。

## 目录职责

| 目录 | 用途 | 放什么 |
|------|------|--------|
| 00-Inbox | 原始输入，临时存放 | 随手想法、网页剪藏、语音笔记 |
| 01-Projects | 当前项目 | 具体任务、产品开发、内容生产 |
| 02-Areas | 长期资产 | 知识卡片、主题地图、案例库 |
| 03-Resources | 外部资料 | 书籍、课程、PDF、文章 |
| 04-Archive | 归档 | 已完成或暂时不用的内容 |
| 06-选题库 | 选题资产 | 选题卡、索引、待挖掘方向 |
| .claude/skills | Claude Code 执行技能 | Markdown 格式的操作说明 |

## 核心原则

1. 原子化：一张卡片只讲一个概念
2. 双向链接：用 [[卡片名]] 建立关联
3. 概念为本：卡片回答"这是什么"，不回答"具体哪个例子"
4. 输出反哺：每次生产结束，要点反哺到卡片/skill
5. YAML 优先：新资产尽量带元数据
6. 不要把 skill 变成资料库
7. 不要把个案结论写成普遍承诺
8. 不要把 Claude Code 改写成 Agent 或 Codex

## 五层链路

输入层 (00-Inbox) → 资产层 (02-Areas) → 项目层 (01-Projects) → 输出层 → 反馈层
