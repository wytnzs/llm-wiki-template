# 知识库协作规则

先阅读根目录 `README.md`。README 是知识库目的、目录职责和已启用模块的唯一说明入口。

## 基本流程

```text
输入 -> 整理 -> 连接 -> 使用 -> 检查 -> 复盘
```

## 推荐目录映射

| 角色 | 默认位置 |
|---|---|
| 输入 | `00-Inbox/` |
| 项目 | `01-Projects/` |
| 资产 | `02-Areas/` |
| 资源 | `03-Resources/` |
| 归档 | `04-Archive/` |

如果 README 中记录了不同路径，以 README 为准。

## 文件规范

- 正式知识文件至少包含 `title` 和 `type`；
- 推荐补充 `description`、`status`、`tags`、`created`、`updated`、`source`；
- 原始材料不强制立刻改造成知识卡片；
- 不把事实、观点和操作流程混在同一层；
- 输出失败时追溯并修正上游，而不是只修改一次结果。

## 关系和检索

- 标签用于稳定分类，双链只表达明确的具体关系；
- 主题页围绕一个持续问题保存当前判断、证据和边界；
- 聚合是筛选资料的动作，不单独创建知识类型；
- 日常优先维护来源、主题问题和真实使用项目；
- 检索顺序为“主题页 -> 知识卡片或案例 -> 原始依据”；
- 不为消灭孤立页、提高链接数量或填满字段制造关系。

## 可选模块

只有 README 明确标记已启用时，才使用知识库地图、案例库或选题库。不要擅自创建未启用模块。

## 安全边界

- 不上传私人资料；
- 不静默删除或批量迁移；
- 原始材料默认只读，来源不确定时不猜测；
- 案例事实、合同、病历和法律或医疗结论只提出建议，确认后写入；
- 提交、推送和对外发送必须获得明确授权；
- 不把 Claude Code 改写成 Agent 或 Codex。
