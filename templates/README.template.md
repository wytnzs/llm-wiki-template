# 我的知识库

## 目的

用一句话说明这个知识库帮助你解决什么问题。

## 主要区域

| 区域 | 实际位置 | 职责 |
|---|---|---|
| 输入 | `00-Inbox/` | 尚未处理的内容 |
| 项目 | `01-Projects/` | 当前正在推进的事情 |
| 资产 | `02-Areas/` | 可以长期复用的知识 |
| 主题中心 | `02-Areas/主题中心/` | 自然语言索引和主题页的统一入口 |
| 来源 | `03-Resources/` | 外部资料、个人原始输入和系统历史 |

目录名称可以调整；调整后同步修改这张表和 `CLAUDE.md`。

目录只表达内容角色。来源、主题问题和项目使用通过字段或链接表达，不为每一种关系新增一级目录。

## 主题中心

- 主题中心：统一承接自然语言索引和主题入口；
- 主题：需要持续回答的问题；
- 聚合：围绕问题筛选资料的动作，不单独形成目录；
- 主题页：保存当前综合判断、证据、方法、反例和边界；
- 知识库地图：启用后展示系统动态状态，不保存主题判断。

查询顺序：`自然语言问题 -> 主题中心 -> 主题页 -> 知识卡片/案例 -> 原始资料`。

## 基本流程

```text
输入 -> 整理 -> 连接 -> 使用 -> 检查 -> 复盘
```

## 常用指令

- `记一下：……`
- `整理收件箱`
- `提炼知识卡片`
- `查一下：……`
- `复盘这次结果`

知识资产日常只优先维护 `source`、`related_theme_maps` 和 `related_projects`。关系说不清时只保留标签，不强行补链。

来源类型和处理状态分开表达：来源类型区分外部资料、个人输入和系统历史；“待筛选、已提炼、已使用”通过导读、标签和连接表达，不反复搬动原文件。

检索顺序：主题页 -> 知识卡片或案例 -> 原始依据。标签负责分类，双链表达具体关系，主题页保存当前综合判断，知识库地图只记录运行状态。

新增关系前确认：关系明确、沿过去有用、能够回到依据。不为消灭孤立页制造链接。
