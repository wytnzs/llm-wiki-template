# 内容选题挖掘 Skill

> 选题不是凭空想出来的，而是从需求和定位的交集中长出来的。
> 本 Skill 是可选模块。只有用户明确启用选题库后才运行，输出候选，不自动入库。

---

## 一、核心定位

本 skill 负责从两个方向挖掘候选选题：

1. **需求端**：目标用户的痛点、问题、焦虑
2. **定位端**：自己的知识积累、经验判断、独特视角

两者的交集就是好选题。

---

## 二、双端挖掘

### 需求端挖掘

从以下渠道收集用户需求信号：
- 客户/用户经常问的问题
- 社交媒体评论区的高频疑问
- 行业论坛/社群的讨论热点
- 竞品内容下的用户反馈

### 定位端挖掘

从自身知识资产中提取选题潜力：
- 02-Areas/知识卡片/ —— 哪些概念可以延展出内容？
- 02-Areas/案例库/ —— 哪些案例值得讲述？
- 02-Areas/主题聚合/ —— 哪些判断需要传播？
- 个人经历 —— 哪些经验有分享价值？

---

## 三、选题树结构

| 选题树 | 适合什么内容 |
|--------|------------|
| 案例拆解 | 具体问题分析、实战经验 |
| 工具实操 | 效率工具、操作教程 |
| 方法论 | 可复用框架、思维模型 |
| 行业趋势 | 政策解读、方向判断 |
| 个人实录 | 真实经历、认知升级 |

---

## 四、筛选过滤

生成候选后，按以下标准过滤：

1. **相关性**：是否和你的定位相关？
2. **差异性**：是否和别人说的不一样？
3. **表达欲**：你有没有真正想说的？

三项都满足 → 输出给 topic-library
缺失任一项 → 先标记待定

---

## 五、与 topic-library 的关系

topic-selection 产生"候选"，topic-library 做"入库"。

```
topic-selection → 候选选题列表
    ↓
topic-library → 查重 → 评分 → 溯源 → 入库 / 待挖掘
```

本 skill 不需要写文件，只输出候选列表给用户确认。

---

## 六、输出格式

每次挖掘结束后输出：

```markdown
## 候选选题清单

### 推荐入库（有资产、有痛点、有差异）
1. {选题名} — {理由} — {支撑资产}

### 待定（需用户判断）
1. {选题名} — {缺什么}

### 来源汇总
- 需求端：{来源数}
- 定位端：{卡片/案例引用数}
```
